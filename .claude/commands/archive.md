# 작업 아카이브

이번 세션에서 수행한 작업 내역을 세 개의 트래킹 문서에 반영합니다.

## 대상 파일

1. `docs/migration/changelog.md` — 변경 기록 (AS-IS → TO-BE 형식)
2. `docs/migration/comparison.md` — Bettermode ↔ Next.js 비교표
3. `docs/ux-issues.md` — UX 이슈 트래커 (체크리스트)

## 실행 절차

### 1단계: 변경사항 수집

먼저 이번 세션에서 어떤 작업을 했는지 대화 내역을 분석합니다.
변경된 파일, 수정 이유, 개선 내용을 정리합니다.

### 2단계: changelog.md 업데이트

`docs/migration/changelog.md` 파일을 읽고, 맨 아래 주석 위에 새 섹션을 추가합니다.

형식:
```markdown
## YYYY-MM-DD | 작업 제목

### 페이지/컴포넌트명
- AS-IS: 기존 상태
- TO-BE: 변경 후 상태
- 개선: 핵심 개선 포인트
```

주의:
- 이미 기록된 변경사항은 중복 추가하지 않습니다.
- 날짜는 오늘 날짜를 사용합니다.
- 여러 페이지/컴포넌트를 변경했으면 각각 별도 ### 섹션으로 작성합니다.

### 3단계: comparison.md 업데이트

`docs/migration/comparison.md` 파일을 읽고, 변경된 페이지의 비교표를 업데이트합니다.

주의:
- 기존 항목의 To-Be 컬럼을 최신 상태로 수정합니다.
- 새로운 화면이 추가되었으면 비교표 섹션을 추가합니다.
- "핵심 수치 변화" 섹션도 변경이 있으면 업데이트합니다.
- 마지막 줄의 `*마지막 업데이트:` 날짜를 오늘로 변경합니다.

### 4단계: ux-issues.md 업데이트

`docs/ux-issues.md` 파일을 읽고 다음을 반영합니다:

- 이번 작업으로 해결된 이슈: `- [ ]` → `- [x]` + 해결 방법 메모 + 날짜
- 이번 작업 중 새로 발견한 이슈: 해당 페이지 섹션에 `- [ ]` 추가

### 5단계: 결과 보고

업데이트한 내용을 간단히 요약합니다:
- changelog: 추가한 섹션 수
- comparison: 수정한 항목
- ux-issues: 해결 표시한 이슈 수, 새로 추가한 이슈 수
