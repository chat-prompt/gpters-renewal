## 1. Reddit (reddit.com)

### 1.1 네비게이션 구조 및 IA

| 레벨 | 구성 |
|------|------|
| **글로벌 네비** | 홈, Popular(폐지 예정), All, 채팅, 알림, 프로필 |
| **피드 선택** | Home(구독 기반), Popular(전체 인기), All(모든 서브레딧) |
| **서브레딧 내부** | 포스트 목록, About/Rules, Wiki, Flair 필터 |
| **포스트 상세** | 본문, 투표, 댓글 트리, 공유, 저장, 크로스포스트 |

**핵심 특징**: 2단계 IA 구조. 글로벌 레벨(전체 Reddit)과 서브레딧 레벨(개별 커뮤니티)이 독립적으로 운영되며, 각 서브레딧이 자체 규칙/Flair/위키를 보유한다.

**최근 변화 (2025-2026)**: r/popular를 폐지하고 머신러닝 기반 개인화 피드로 대체. CEO Steve Huffman은 "r/popular는 Reddit의 다양한 문화를 대변하지 못한다"며 신규 유저를 개인화 경험으로 직접 연결하는 방향을 선택.

### 1.2 콘텐츠 분류 체계

**서브레딧 (Subreddit) 시스템**
- 주제/관심사 기반의 독립 커뮤니티 단위 (r/MachineLearning, r/ChatGPT 등)
- 각 서브레딧이 자체 규칙, 모더레이터, 디자인 커스터마이징 보유
- 사용자가 자유롭게 서브레딧 생성 가능

**Flair 시스템**
- **Post Flair**: 게시물을 카테고리화하는 태그 (예: Discussion, Tutorial, News)
  - 모더레이터가 설정, 일부 서브레딧에서 필수 지정
  - Flair별 필터링으로 콘텐츠 탐색 가능
- **User Flair**: 사용자 전문성/역할 표시 (예: AI Engineer, Beginner)
  - 커뮤니티 내 신뢰도와 맥락 제공

**콘텐츠 정렬 옵션**
| 정렬 | 설명 |
|------|------|
| Hot | 최근 투표 + 시간 가중치 (기본값) |
| New | 최신순 |
| Top | 높은 투표 수 (기간별 필터: 오늘/주/월/년/전체) |
| Rising | 빠르게 투표 올라가는 게시물 |
| Controversial | 찬반이 비슷한 게시물 |

### 1.3 유저 온보딩 / 신규 유저 경험

- **비로그인 경험**: 인기 콘텐츠 피드를 바로 노출하여 가치를 즉시 보여줌
- **관심사 선택**: 가입 시 관심 주제/서브레딧 선택 화면 제공
- **개인화 피드**: 가입 직후부터 관심사 기반 추천 피드 생성 (2025년부터 ML 클러스터링 강화)
- **단계적 참여 유도**: 읽기 → 투표 → 댓글 → 포스트 순으로 참여 장벽을 낮춤
- **2025 신기능**: 삭제된 게시물에 "다른 커뮤니티에 포스트하기" 버튼 제공, 관련 서브레딧 자동 추천

### 1.4 콘텐츠 디스커버리 방법

1. **개인화 홈 피드**: 구독 서브레딧 + ML 추천 기반
2. **검색**: 키워드/서브레딧/사용자 검색
3. **Flair 필터링**: 서브레딧 내에서 특정 카테고리만 보기
4. **크로스포스트**: 동일 콘텐츠를 여러 서브레딧에 공유
5. **트렌딩 알림**: 구독 서브레딧의 인기 게시물 푸시
6. **커스텀 피드**: 여러 서브레딧을 묶어 맞춤 피드 생성
7. **ML 기반 추천**: 사용자 행동(투표, 댓글, 체류시간) 분석 → 새 서브레딧/콘텐츠 추천

### 1.5 전환 동선

- **가입 유도**: 투표/댓글 시도 시 로그인 프롬프트 → 자연스러운 가입 전환
- **Reddit Premium**: 광고 제거, 아바타 커스터마이징, 코인 지급
- **Awards**: 유저 간 게시물에 보상 부여 (표현 + 가시성 상승)
- **Karma 시스템**: 활동량의 수치화 → 참여 동기 부여

### 1.6 UX 강점 / 약점

| 강점 | 약점 |
|------|------|
| 투표 기반 큐레이션으로 고품질 콘텐츠 자연 부상 | 신규 유저에게 서브레딧 구조가 복잡 |
| Flair로 체계적 콘텐츠 분류 + 필터링 | 시각적 디자인이 텍스트 중심으로 단조로움 |
| 댓글 트리 구조로 깊이 있는 토론 가능 | 모바일 UX가 웹 대비 제한적 |
| 커뮤니티 자치(모더레이션)로 품질 유지 | 검색 기능이 Google 대비 약함 |
| 다양한 정렬 옵션으로 니즈별 탐색 | "파워 모더레이터" 집중 문제 (2026년 5개 제한 도입) |
| 개인화 피드로 관련 콘텐츠 발견 용이 | 초기 참여 장벽 (Karma 부족 시 게시 제한) |

### 1.7 GPTers에 적용 가능한 인사이트

1. **투표 기반 큐레이션**: 게시물에 좋아요/투표 시스템 도입으로 커뮤니티가 직접 양질 콘텐츠 선별
2. **Flair/태그 시스템**: 게시물에 카테고리 태그(Flair) 필수 적용 → 태그별 필터링 지원
3. **다양한 정렬 옵션**: 인기순/최신순/주간Top 등 다양한 정렬로 콘텐츠 디스커버리 강화
4. **단계적 참여 유도**: 읽기(비회원) → 좋아요(회원) → 댓글(회원) → 글쓰기(활동회원) 구조
5. **개인화 피드**: 관심 분야(AI도구별, 난이도별) 선택 기반 맞춤 콘텐츠 추천
6. **커뮤니티 자치**: 스터디별/주제별 공간에 운영진(모더레이터) 권한 부여

## 2. 모두의연구소 (modulabs.co.kr)

### 2.1 네비게이션 구조 및 IA

**GNB (Global Navigation Bar) 구조**

| 메뉴 | 설명 |
|------|------|
| **캠프** | 유료 교육 프로그램 (부트캠프, 직장인 캠프) |
| **온라인강의** | 비동기 온라인 강의 (왕초보~심화) |
| **커뮤니티** | 라운지, 모두모임, LAB, 페이퍼샵, 테크포임팩트 |
| **무료세미나** | 진입장벽 낮춘 무료 행사 |
| **기업교육** | B2B 교육 프로그램 |
| **블로그** | 콘텐츠 마케팅/SEO용 블로그 |

**특징**: 드롭다운 메뉴로 "AI 엔지니어", "AI 리서처" 등 구체적 프로그램에 직접 접근 가능. 정형교육(캠프)과 비정형학습(커뮤니티)이 GNB 레벨에서 명확히 분리.

### 2.2 콘텐츠 분류 체계

**분류 축 1: 형식별**
| 형식 | 기간 | 대상 |
|------|------|------|
| 부트캠프 (동기식) | 6개월 | 취업준비생, 전환희망자 |
| 직장인캠프 | 2-3개월 | 현직자 업스킬 |
| 온라인강의 (비동기) | 자율 | 전체 |
| 세미나 (하이브리드) | 단발 | 전체 |

**분류 축 2: 대상별**
- 취업준비생 → AI 엔지니어 부트캠프
- 연구자 → AI 리서처 부트캠프
- 직장인 → LLM 데이터분석, 서비스기획
- 일반인 → 무료세미나, 온라인강의

**분류 축 3: 커뮤니티 계층**
```
라운지 (일반 커뮤니티)
├── 모두연소식
├── 자유게시판
├── 모두모임 둘러보기
│
모두모임 (피어러닝)
├── 스터디: 학습 목표 설정 + 지식 공유
├── 북클럽: 책 기반 관점 나눔
├── 챌린지: 목표 설정 + 공동 도전
└── 세미나: AI/SW 전문 강의
│
LAB (열린 연구실)
├── 다오랩
├── Microsoft Fabric LAB
└── Tech for People LAB
│
페이퍼샵 (논문 지원)
테크포임팩트 (임팩트 기술)
```

### 2.3 유저 온보딩 / 신규 유저 경험

- **메인 배너 10개**: 다양한 유저 페르소나 타게팅 (학생→AI엔지니어, 직장인→LLM데이터분석)
- **서비스 아이콘 카드**: 12개 프로그램을 시각적으로 한눈에 표시
- **무료 세미나**: 가장 낮은 진입장벽의 참여 기회
- **NEW 배지**: 최신 콘텐츠 강조
- **모집 상태 표시**: "모집중", "모집완료", "진행중" 상태 배지로 정보 투명성 확보
- **팝업 리마인더**: "작성 중인 지원서" 안내로 재방문 유도

### 2.4 콘텐츠 디스커버리 방법

1. **메인 배너**: CTA가 명확한 슬라이드 배너
2. **서비스 카드 그리드**: 12개 프로그램 시각적 진입점
3. **KDT 교육 일정**: 일정 투명성으로 의사결정 지원
4. **온라인 강의 할인율**: 90% 할인 등 가격 즉시 노출
5. **모두모임 상태 표시**: 모집중/모집완료 상태로 참여 기회 인지
6. **누적 조회수 TOP7**: 인기 콘텐츠 하이라이트

### 2.5 전환 동선

```
인식 → 메인배너, GNB 접근
  ↓
관심 → 서비스카드 클릭, 프로그램 상세
  ↓
고려 → 무료세미나 참석, 온라인강의 체험
  ↓
전환 → 캠프 지원, 온라인강의 구매
```

**전환 장애요소**:
- 프로그램 종류 과다로 의사결정 지연
- 프로그램 간 비교 기능 부재
- 학습 경로 가이드 부족 (어떤 과정을 먼저 들을지)

### 2.6 UX 강점 / 약점

| 강점 | 약점 |
|------|------|
| 명확한 CTA 버튼 ("바로시작하기", "지금도전하기") | 프로그램 정보가 산재되어 있음 |
| 시각적 계층화 (배너→카드→리스트) | 프로그램 비교 기능 부재 |
| 상태 배지로 정보 명확성 확보 | 필터링/검색 기능 미약 |
| "함께 성장" 메시지로 커뮤니티 감성 전달 | 모바일 정보 탐색 번거로움 |
| 교육+커뮤니티 이중 모델 | 학습경로 가이드 부족 |
| 피어러닝 구조(캡틴-크루) | 커뮤니티-정형교육 연계 약함 |

### 2.7 GPTers에 적용 가능한 인사이트

1. **피어러닝 구조**: 캡틴(스터디장)-크루(스터디원) 역할 분담 모델을 GPTers 스터디에 적용
2. **모임 유형 다양화**: 스터디/북클럽/챌린지/세미나 등 참여 유형을 명확히 구분
3. **상태 배지 시스템**: 모집중/진행중/완료 상태를 시각적으로 표시하여 참여 기회 인지 향상
4. **무료 진입점**: 무료 세미나/이벤트로 신규 유저의 진입 장벽을 낮추는 퍼널 구축
5. **커뮤니티 계층 구조**: 자유게시판(라운지) → 주제별 모임(모두모임) → 심화 연구(LAB) 3단계 구조
6. **인기 콘텐츠 하이라이트**: 누적 조회수 TOP 등 데이터 기반 콘텐츠 추천

## 3. 패스트캠퍼스 (fastcampus.co.kr)

### 3.1 네비게이션 구조 및 IA

**헤더 구성**
- **좌측**: 로고 (홈 링크)
- **중앙**: 검색바 (콘텐츠 검색 강조)
- **우측**: 로그인, 커뮤니티, 기업교육 링크

**카테고리 메가메뉴 (9개 대분류)**

| 카테고리 | 서브카테고리 예시 |
|----------|-------------------|
| AI TECH | LLM, RAG, 딥러닝, 자율주행, 컴퓨터 비전 |
| AI CREATIVE | 2D/3D 이미지, 영상 생성 |
| AI/업무생산성 | AI 생산성, 마케팅, 엑셀 |
| 개발/데이터 | 백엔드, 프론트엔드, DevOps |
| 디자인 | UX/UI, 그래픽/브랜딩 |
| 영상/3D | 블렌더, 모션그래픽, VFX |
| 금융/투자 | 부동산, 재무회계 |
| 드로잉/일러스트 | 캐릭터, 웹툰 |
| 비즈니스/기획 | PM/PO, 창업, 리더십 |

각 대카테고리 아래 3-12개의 서브카테고리 + "이벤트", "추천코스" 특수 섹션 포함.

### 3.2 콘텐츠 분류 체계

- **대분류**: 직군/관심사 기반 9개 카테고리
- **서브분류**: 기술 스택/세부 분야별 세분화
- **특수 분류**: 이벤트, 추천코스, 패키지 상품
- **아이콘+텍스트 조합**: 카테고리 시각적 구분

### 3.3 유저 온보딩 / 신규 유저 경험

- **프로모션 배너 회전**: 할인/이벤트 정보를 최상단에 노출
- **무료 세미나 배너**: "지금 신청할 수 있는 무료 세미나 확인하기"
- **학생 할인**: Adobe 학생 패키지 65% 할인 등 타겟 프로모션
- **실시간 인기 강의 TOP5**: 사회적 증거(Social Proof) 활용
- **월간 인기 검색어 / 급상승 검색어**: 트렌드 기반 탐색 유도
- **신뢰도 구축**: 올해의 브랜드 대상 배지, 원격평생교육원 등록번호

### 3.4 콘텐츠 디스커버리 방법

1. **중앙 검색바**: 키워드 검색 강조 배치

5. **인기/급상승 검색어**: 트렌드 기반 탐색 유도

### 3.5 전환 동선 (가입, 구매)

**강의 상세페이지 구조 (핵심 전환 페이지)**

## 4. LinkedIn (linkedin.com)

> 다혜 인터뷰에서 제시된 핵심 비전 — **"AI계의 LinkedIn"** — 을 구체화하기 위해 LinkedIn의 프로필/셀프브랜딩/활동 피드 구조를 집중 분석한다.

### 4.1 네비게이션 구조 및 IA

| 레벨 | 구성 |
|------|------|
| **글로벌 네비** | 홈(피드), 네트워크, 채용정보, 메시징, 알림, 프로필 |
| **프로필 내부** | 소개, 활동, 경험, 학력, 스킬, 추천, 자격증, 프로젝트 |
| **콘텐츠 영역** | 피드(포스트/아티클), 그룹, 이벤트, 뉴스레터 |
| **학습 영역** | LinkedIn Learning (강의, 학습 경로, 수료증) |

**핵심 특징**: 프로필이 플랫폼의 중심축. 모든 활동(포스트, 댓글, 학습 수료, 스킬 인증)이 프로필에 자동으로 누적되어 "살아있는 이력서" 역할을 수행한다.

### 4.2 프로필/셀프브랜딩 시스템

**프로필 구조 (핵심 분석 대상)**

```
프로필 페이지
├── 헤더: 배경 이미지 + 프로필 사진 + 이름/직함/소속
├── 소개(About): 자유 형식 자기소개 (2,600자)
├── 활동(Activity): 최근 포스트, 댓글, 반응 모아보기
├── 경험(Experience): 직무 이력 (회사, 역할, 기간, 설명)
├── 학력(Education): 학력 사항
├── 자격증(Licenses & Certifications): 외부 인증 + LinkedIn Learning 수료증
├── 스킬(Skills): 최대 50개 스킬 + 타인의 endorsement 수
├── 추천(Recommendations): 동료/상사의 서술형 추천
├── 프로젝트(Projects): 참여 프로젝트 + 협업자 태그
└── 관심사(Interests): 팔로우하는 인플루언서/회사/그룹
```

**셀프브랜딩 핵심 메커니즘**
- **스킬 endorsement**: 타인이 특정 스킬에 +1 보증 → 스킬별 endorsement 수가 전문성 지표로 기능
- **수료증 자동 반영**: LinkedIn Learning 수료 시 프로필에 자동 추가
- **활동 피드 노출**: 포스트/아티클 작성 → 프로필 "활동" 섹션에 자동 정리 → 방문자가 전문성 판단 가능
- **Open to Work / Hiring 배지**: 구직/구인 상태를 프로필 사진에 시각적으로 표시
- **프로필 강도(Profile Strength)**: 항목 완성도에 따라 "All-Star" 등급까지 단계별 피드백 → 프로필 완성 유도

### 4.3 활동 피드 시스템

**피드 콘텐츠 유형**
| 유형 | 형식 | 비고 |
|------|------|------|
| **포스트** | 텍스트 + 이미지/문서/폴/영상 (최대 3,000자) | 가볍고 빠른 공유 |
| **아티클** | 장문 블로그 형식 (제한 없음) | 전문성 어필, SEO 노출 |
| **리포스트** | 타인의 콘텐츠 + 본인 코멘트 | 큐레이터 역할 |
| **뉴스레터** | 구독 기반 정기 발행 | 인플루언서/기업용 |
| **이벤트** | 온/오프라인 이벤트 | 참석/관심 표시 |

**피드 알고리즘 특징**
- 1촌 활동, 팔로우 대상, 관심 주제 기반 개인화
- "좋아요"보다 댓글이 있는 포스트에 가중치 부여 (인게이지먼트 우선)
- 포스트 발행 후 1시간 내 초기 반응이 확산 범위 결정

### 4.4 LinkedIn Learning 통합

- 강의 수료 → 프로필 "자격증" 섹션에 자동 추가
- 스킬 평가(Skill Assessments) → 합격 시 프로필에 "Verified" 배지
- 학습 경로(Learning Path) 제공 → 직무별 큐레이션된 강의 묶음
- 기업 관리자 대시보드 → 팀원 학습 현황 추적

### 4.5 구인/구직 통합

- 프로필이 곧 이력서 → 별도 이력서 없이 "간편 지원(Easy Apply)"
- 채용 공고에 "이 회사에 아는 사람" 표시 → 네트워크 활용
- 스킬 매칭: 보유 스킬과 채용 공고 요구 스킬 자동 비교
- 리크루터 검색: 스킬, 경험, 활동 기반으로 인재 서칭

### 4.6 UX 강점 / 약점

| 강점 | 약점 |
|------|------|
| 프로필이 전문성의 통합 허브 역할 | 피드가 자기홍보/영업 콘텐츠로 포화 |
| 활동이 자동으로 프로필에 누적 | 포스트 품질 관리 메커니즘 약함 (투표 없음) |
| 스킬 endorsement로 객관적 전문성 표시 | endorsement가 형식적으로 남발됨 |
| 학습 수료 → 프로필 자동 반영 연계 | 무료 기능 제한 (Premium 유료화) |
| 가벼운 포스트 ~ 깊은 아티클까지 다양한 형식 | 커뮤니티(그룹) 기능이 피드 대비 약함 |
| 프로필 강도 피드백으로 완성 유도 | 소규모/니치 커뮤니티에는 부적합한 구조 |

### 4.7 GPTers에 적용 가능한 인사이트

1. **프로필 = AI 포트폴리오**: 스터디 수료, AI 활용사례 게시글, 발표 기록이 자동으로 마이페이지에 누적되는 구조 (다혜 비전: "프로필을 보고 이 사람이 AI 잘 쓰는 인재라고 느낄 수 있도록")
2. **스킬 endorsement 변형**: AI 도구별 활용 역량을 타인이 보증하는 시스템 (예: "Claude 활용 +12 endorsement")
3. **활동 자동 요약**: AI 활용사례가 자동 요약되어 프로필에 반영 (다혜 비전 핵심)
4. **가벼운 포스트 + 깊은 아티클 이원화**: 피드형 가벼운 공유와 구조화된 사례글을 분리하여 참여 장벽 해소
5. **학습 이력 통합**: 스터디 수강 내역, 수료증이 프로필에 자동 노출
6. **프로필 완성도 게이미피케이션**: 프로필 강도 단계별 피드백으로 정보 입력 유도

## 5. 요즘IT (yozm.wishket.com)

> 위시켓(Wishket)이 운영하는 IT 실무자 대상 기술 콘텐츠 매거진. "99만 IT 실무자"를 대상으로 한다. 콘텐츠 매거진 + 프로덕트 리뷰 + AI Q&A + 커뮤니티(놀이터)를 결합한 복합 모델.

### 5.1 네비게이션 구조 및 IA

| 메뉴 | 설명 |
|------|------|
| **콘텐츠** | 메인 매거진 피드 (카테고리별 탐색) |
| **프로덕트 밸리** | IT 프로덕트 리뷰/평가 플랫폼 (Product Hunt형) |
| **요즘 작가들** | 기고 작가 프로필 목록 |
| **컬렉션** | 독자/에디터가 만든 글 모음 큐레이션 |
| **물어봐** | AI 캐릭터 '요고' 기반 Q&A (베타) |
| **놀이터** | 미니게임/퀴즈형 참여 콘텐츠 |

콘텐츠 하위 카테고리: 개발 / AI / IT서비스 / 기획 / 디자인 / 비즈니스 / 프로덕트 / 커리어 / 트렌드 / 스타트업 (10개) + "새로 나온" / "인기" (정렬 2개)

### 5.2 비회원 → 회원 전환 전략 (핵심)

**전문 공개 + 상호작용 게이팅** — 페이월 없이 모든 글 본문 100% 무료 공개. 댓글/스크랩/형광펜만 로그인 필요.

| 위치 | 장치 | 문구 |
|------|------|------|
| 콘텐츠 목록 중간 | 인라인 CTA 배너 | "지금 회원가입하고 실무 꿀팁을 스크랩해 보세요" |
| 글 상세 내 | 형광펜 유도 | "요즘IT 멤버가 되어 형광펜 해보세요" |
| 댓글 영역 | 로그인 리다이렉트 | 댓글 작성 시도 시 로그인 페이지 이동 |
| 하단 | 뉴스레터 구독 CTA | "무료로 구독하기" + 이메일 입력 |

**전환 동선**:
```
비회원 → 글 전문 읽기 (무료)
  → 스크랩/형광펜/댓글 시도 → "회원가입하고 스크랩해보세요"
  → 회원 → 뉴스레터 구독 → 정기 재방문
  → 프로덕트 리뷰/Q&A 참여 → 능동적 기여자
  → 작가 지원 → 콘텐츠 생산자
```

### 5.3 독자층 정보 표시 기능 (핵심 차별점)

각 글마다 **읽은 독자의 직군/경력 분포를 시각화**하는 독특한 기능:

- 글별로 "이 글을 읽은 독자" 직군 TOP 3 + 경력 분포 공개
- 예: AI 에이전트 글 → 풀스택 개발자(1위), PM(2위), 백엔드 개발자(3위) + 경력 0~1년 77명, 2~4년 39명...
- 독자에게 "나와 비슷한 사람들이 이 글을 읽고 있다"는 사회적 증거(Social Proof)
- 작가에게 "누가 내 글을 읽는지" 피드백

### 5.4 형광펜(하이라이트) 기능

- 글 본문에서 중요 문장을 하이라이트 (회원 전용)
- 하이라이트 데이터를 메인 페이지 "독자들의 성장을 이끈 문장" 섹션에 재활용
- 독자 참여 + 콘텐츠 큐레이션을 동시 해결

### 5.5 글 상세 페이지 UX

- 읽는 시간 표시 (분 단위)
- 3열 레이아웃: 본문 | 우측 사이드바(작가 프로필, 다른 콘텐츠, 광고)

### 5.8 GPTers에 적용 가능한 인사이트

1. **독자 직군/경력 통계**: GPTers에서는 "이 사례를 읽은 AI 활용자: 마케터 35%, 개발자 28%" 형태로 사회적 증거 제공
2. **전문 공개 + 상호작용 게이팅**: 본문 100% 공개로 SEO 극대화, 스크랩/댓글만 회원 제한
3. **형광펜 → 메인 재활용**: "이번 주 가장 많이 밑줄 친 AI 팁"으로 메인 노출
4. **컬렉션**: "AI 마케팅 활용 모음" 등 주제별 사례 큐레이션
5. **읽는 시간 표시**: 가벼운 글(3분)과 심층 사례(15분) 구분
6. **문맥 기반 CTA**: 사용자 행동 맥락에 맞는 가입 유도 문구
7. **뉴스레터 퍼널**: 회원가입과 별도로 이메일 구독을 독립 전환점으로 운영

### 6.2 GPTers만의 차별점 제안

#### 1. AI 이력서/포트폴리오 시스템 — "AI계의 LinkedIn" 핵심
- Reddit, 모두의연구소, 패스트캠퍼스, **심지어 LinkedIn조차** AI 도구 활용 역량에 특화된 포트폴리오를 제공하지 않음
- 다혜 비전: "마이페이지를 셀프브랜딩 포트폴리오로 만들기. AI 활용사례가 자동 요약되어 프로필에 반영. 프로필을 보고 '이 사람 AI 잘 쓰는 인재'라고 느낄 수 있도록."
- GPTers 차별점:
  - 스터디 수강 내역 + AI 활용사례 게시글 + 발표 기록이 자동으로 누적되는 "AI 포트폴리오 마이페이지"
  - AI 활용사례가 **자동 요약**되어 프로필에 반영 (수동 정리 불필요)
  - 각 활동에 기여도/역할/결과물이 기록되어 외부 공유 가능
  - LinkedIn의 "프로필 = 이력서" 모델을 AI 도구 활용 영역에 특화 적용

#### 2. AI 도구 특화 커뮤니티 구조
- Reddit은 범용, 모두의연구소는 AI/ML 기술 연구 중심, 패스트캠퍼스는 강의 판매 중심
- GPTers 차별점: **AI 도구 활용 사례**에 특화된 콘텐츠 구조
  - 도구별 게시판 (ChatGPT, Claude, Gemini, Midjourney 등)
  - "이렇게 활용했다" 사례 중심 포맷
  - 비개발자도 쉽게 이해 가능한 난이도 표시

#### 3. 콘텐츠-스터디 연결 브릿지
- 타 사이트들은 콘텐츠(게시물)와 교육(스터디/강의)이 별개 섹션
- GPTers 차별점: 게시물 하단에 **관련 스터디 추천**, 스터디 페이지에 **관련 사례글 모음**
  - "이 글이 도움이 되셨다면 → 관련 스터디 알아보기" 자연스러운 전환
  - 스터디 커리큘럼 옆에 "이전 기수 수료생 사례글" 연결

#### 4. SEO + LLM 최적화 듀얼 전략
- 패스트캠퍼스는 SSR 기반 SEO를 하지만, LLM 대화 노출 최적화는 미비
- GPTers 차별점: 구글 SEO + ChatGPT/Perplexity 등 AI 검색 동시 최적화
  - 구조화된 데이터(Schema.org), 메타 태그, 시맨틱 HTML
  - AI 검색 봇 친화적 콘텐츠 구조

#### 5. 피드형 가벼운 포스트 — 벤치마크 사이트들이 못 푸는 과제
- 현재 GPTers의 사례게시글은 "과제 제출용"처럼 무겁다는 피드백 (다혜, 재호 인터뷰)
- Reddit은 텍스트 위주로 시각적 매력 부족, LinkedIn은 자기홍보성 포스트로 포화, 모두의연구소/패스트캠퍼스는 피드 자체가 없음
- GPTers 차별점: **AI 활용 자랑/공유에 최적화된 경량 피드 포스트**
  - "자랑 포스트": AI로 만든 서비스/프로덕트를 간단히 공유 (스크린샷 + 한 줄 설명)
  - 누구나 게시글 올리기 (현재는 스터디 참여자만 가능 → 개방)
  - 피드형 포스트(가벼운 공유)와 사례게시글(구조화된 심층 콘텐츠)을 이원화

#### 6. 게이미피케이션 — 포인트/뱃지/기여 시스템
- Reddit은 Karma 시스템이 있지만 단순 수치에 불과, LinkedIn은 endorsement가 형식적으로 남발
- GPTers 차별점: **기여 기반 포인트 + 의미 있는 뱃지 시스템**
  - 기여 정의: AI 활용사례 게시글 작성, 무료 AI토크 개설, 모각AI 리딩
  - 포인트 제도: 기여 활동별 포인트 적립 → 등급/혜택 연동
  - 뱃지 제도: 현재 기준 없이 남발되지만 동기부여 효과 큼 (재호 인터뷰) → 명확한 기준 + 희소성 관리 필요
  - 포인트/뱃지가 프로필(AI 이력서)에 반영되어 전문성 시각화의 일부로 기능

#### 7. 채팅방 ↔ 게시판 양방향 통합 (26년 2Q~ 계획)
- 벤치마크 사이트 중 채팅과 게시판을 유기적으로 연결한 사례 없음 (Reddit 채팅은 커뮤니티와 분리, LinkedIn 메시징은 1:1 위주)
- GPTers 차별점: **채팅방 ↔ 게시판 양방향 채널**
  - 채팅방에서 나온 인사이트 → 게시글로 정리/발행
  - 게시글 토론 → 채팅방에서 실시간 이어가기
  - 채팅방에서 AI교육 구매 전환 퍼널 (자연스러운 추천)
  - 모각AI(온/오프라인 코워킹 세션)를 채팅방 기반으로 운영

#### 8. 구인/구직 게시판 (중장기)
- LinkedIn은 구인/구직의 글로벌 표준이지만 AI 활용 인재 특화는 아님
- GPTers 차별점: AI 활용 역량 기반 구인/구직
  - AI 포트폴리오 프로필이 곧 지원서 → "AI 잘 쓰는 인재"를 찾는 기업과 연결
  - 스터디 수료 이력, 활용사례, 뱃지가 객관적 역량 지표로 기능
  - 우선순위: 낮음 (포트폴리오/커뮤니티 안정화 후 검토)

```
│  │ → 뱃지 시스템 (명확한 기준 + 희소성)        │            │
│  │ → 등급/레벨 (기여 포인트 누적 기반)         │            │
│  │ → 포인트/뱃지 → 프로필에 반영              │            │
│  │ → 다음 스터디/모각AI 추천 (재전환 루프)     │            │
│  │ → 구인/구직 연결 (중장기)                   │            │
│  └──────────────────────────────────────────┘            │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

**핵심 원리: 각 단계가 다음 단계로의 자연스러운 브릿지 역할 + 5단계 게이미피케이션이 전체 순환 동력**

| 단계 | 핵심 지표 | 벤치마크 참조 | 전환 트리거 |
|------|----------|--------------|------------|
| 1단계: 콘텐츠 접근 | 페이지뷰, 체류시간 | Reddit (투표, Flair, 정렬) | "더 많은 콘텐츠를 보려면 가입하세요" |
| 2단계: 커뮤니티 참여 | DAU, 게시물/댓글 수, 피드 포스트 수 | 모두의연구소 (피어러닝) + LinkedIn (피드) | "관련 스터디로 더 깊이 배워보세요" |
| 3단계: 교육 참여 | 스터디 전환율, 수료율 | 패스트캠퍼스 (상세페이지, CTA) | "수료 이력을 AI 이력서에 추가하세요" |
| 4단계: 포트폴리오 | 프로필 완성도, 외부 공유 수 | LinkedIn (프로필, endorsement) | "뱃지를 모아 AI 전문가 등급을 달성하세요" |
| 5단계: 게이미피케이션 | 포인트/뱃지 수, 재참여율 | Reddit (Karma) + LinkedIn (endorsement) | "새 스터디 기수가 열렸습니다" / "AI토크 리딩 보상" |

### 6.4 카테고리 분류 체계 제안

현재 GPTers의 약 30개 카테고리 (분야별/도구별 혼합)를 다음과 같이 재구성:

**1차 분류 (GNB 레벨)**
| 메뉴 | 설명 |
|------|------|
| 홈 | 개인화 피드 + 인기 콘텐츠 |
| 커뮤니티 | AI 도구별/주제별 게시판 |
| 스터디 | 모집중/진행중 스터디 목록 |
| 이벤트 | 세미나, 챌린지, 모임 |
| AI 이력서 | 내 포트폴리오 페이지 |

**2차 분류 (태그/Flair 레벨) - Reddit 참조**
- **AI 도구**: ChatGPT, Claude, Gemini, Midjourney, Cursor 등
- **콘텐츠 유형**: 사례, 튜토리얼, 뉴스, 질문, 토론
- **난이도**: 입문, 중급, 고급
- **분야**: 마케팅, 개발, 디자인, 비즈니스, 교육

→ 1차 분류는 최소화(5개), 2차 분류(태그)로 세분화하여 "중구난방 카테고리" 문제 해결
