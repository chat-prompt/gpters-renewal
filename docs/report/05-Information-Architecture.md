> 작성일: 2026-02-25
> 업데이트: 2026-02-25 (다혜/재호 인터뷰 반영)

## 1. 설계 원칙

본 IA는 다음 원칙을 기반으로 설계되었다.

| 원칙 | 설명 | 근거 |
|------|------|------|
| **3클릭 도달** | 모든 핵심 콘텐츠에 3클릭 이내 접근 | 인터뷰: 깊은 네비게이션으로 사용자가 "여러 번 헤맴" |
| **분류 단순화** | 30개 카테고리 → 6개 대분류 + 태그 | 인터뷰: "중구난방인 카테고리", 분야별/도구별 혼재 |
| **역할 기반 동선** | 방문자/회원/스터디원/운영자별 명확한 경로 | 신규 사용자 "뭐가 이렇게 많지?" 반응 해소 |
| **SEO 우선 URL** | 의미 있는 한글 slug, 정적 경로 | 마케터: "SEO에 잘 걸리게 하는 것이 주요 업무" |
| **전환 최적화** | 콘텐츠 소비 → 회원가입 → 스터디 참여 퍼널 | 마케터: 게시글→회원가입, 상세→구매 전환 병목 |
| **참여 진입장벽 최소화** | 피드 포스트(가벼운 글) + 사례 게시글(깊은 글) 2트랙 | 다혜: "현재 게시글이 과제 제출용, 가벼운 포스트 필요" |
| **기여 가시화** | 포인트/뱃지 시스템으로 활동 동기부여 | 다혜: "기여 정의 + 포인트 + 뱃지 체계화" |
| **메시지 신뢰성** | DM을 자동 메시지와 분리, 전용 메시지함 | 다혜: "DM이 자동메시지에 묻혀 중요한 메시지 누락" |
| **아이콘 최소화** | 메뉴 이모지/아이콘 최소화 또는 제거 | 재호: "이모지가 오히려 혼란" |

## 2. 글로벌 네비게이션

### 2.1 메인 메뉴 (GNB) - 7개 항목

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│  [Logo]   탐색    스터디    커뮤니티    AI 이력서    메시지   [검색]  [로그인]    │
└──────────────────────────────────────────────────────────────────────────────────┘
```

| 순서 | 메뉴 | 설명 | 설계 근거 |
|------|------|------|-----------|
| 1 | **탐색** | 콘텐츠 디스커버리 허브 (피드 포스트, 사례 게시글, 카테고리, 인기글) | 핵심 요구사항: "관심사/수준 맞춤 콘텐츠 발견". 피드 포스트를 통한 가벼운 진입 포함 |
| 2 | **스터디** | 모집 중/진행 중 스터디 목록, 상세 | 마케터: 상세→구매 전환 개선 필요 |
| 3 | **커뮤니티** | 자유 게시판, Q&A, 이벤트, 모각AI, AI토크 | 커뮤니티 성격 강화. 다혜: 모각AI, AI토크(누구나 개설) 추가 |
| 4 | **AI 이력서** | 내 프로필, AI 활용 요약, 포인트/뱃지, 포트폴리오 | 다혜: "AI계의 LinkedIn" 비전, 셀프브랜딩 프로필 + AI 활용 자동 요약 |
| 5 | **메시지** (아이콘) | 개인 메시지함 (자동/수동 분리) | 다혜: "DM이 자동메시지에 묻혀 중요한 메시지 누락" → 전용 메시지함 |
| 6 | **검색** (아이콘) | 통합 검색 (게시글 + 스터디 + 사용자) | 인터뷰: "필요한 카테고리를 검색으로 찾는 경향" |
| 7 | **로그인/프로필** | 인증 + 내 스터디 바로가기 | 인터뷰: 신규 참여자 "스터디를 들으려면 어디로?" 해소 |

### 2.2 로그인 후 프로필 드롭다운

```
┌──────────────────┐
│  내 스터디        │  ← 가장 상위 (신규 참여자 FAQ 해소)
│  내 글 / 댓글     │
│  AI 이력서        │
│  포인트 / 뱃지    │  ← NEW: 기여 기반 포인트/뱃지 확인
│  메시지           │  ← NEW: DM 메시지함 바로가기
│  알림 설정        │
│  ──────────────  │
│  관리자 대시보드   │  ← 운영자 권한 있을 때만 노출
│  ──────────────  │
│  로그아웃         │
└──────────────────┘
```

### 2.3 모바일 네비게이션

```
┌──────────────────────────────────────────────┐
│  [Logo]              [메시지] [검색] [메뉴]   │
└──────────────────────────────────────────────┘
하단 탭바:
┌──────┬──────┬──────┬──────┬──────┐
│  홈  │ 탐색 │ 스터디│커뮤니│ 내정보│
└──────┴──────┴──────┴──────┴──────┘
```

> **아이콘 가이드라인**: 재호 인터뷰에서 "이모지가 오히려 혼란을 준다"는 피드백 반영, 메뉴 아이콘은 범용적인 시스템 아이콘(예: 메시지 말풍선, 검색 돋보기)만 사용하고 이모지는 배제한다.

## 3. 페이지 계층 (사이트맵)

### 3.1 전체 사이트맵 트리

```
홈 (/)
├── 탐색 (/explore)
│   ├── 전체 피드 (/explore/feed)
│   ├── 카테고리별 (/explore/category/{slug})
│   │   ├── AI 활용법 (/explore/category/ai-usage)
│   │   ├── 프롬프트 (/explore/category/prompt)
│   │   ├── 자동화/노코드 (/explore/category/automation)
│   │   ├── 개발/코딩 (/explore/category/dev)
│   │   ├── 비즈니스/마케팅 (/explore/category/business)
│   │   └── AI 뉴스 (/explore/category/news)
│   ├── 인기글 (/explore/trending)
│   └── 게시글 상세 (/explore/{post-slug})
│
├── 스터디 (/study)
│   ├── 모집 중 (/study?status=recruiting)
│   ├── 진행 중 (/study?status=ongoing)
│   ├── 스터디 상세 (/study/{study-slug})
│   │   └── 주차별 커리큘럼 (페이지 내 섹션)
│   └── 내 스터디 (/study/my)
│       └── LMS 연결 (외부 링크)
│
├── 커뮤니티 (/community)
│   ├── 피드 (/community/feed)                          ← NEW: 가벼운 포스트 (자랑글, 짧은 공유)
│   ├── 자유 게시판 (/community/free)
│   ├── Q&A (/community/qna)
│   ├── 이벤트/모임 (/community/events)
│   ├── 모각AI (/community/mogak-ai)                    ← NEW: 온/오프라인 공동작업 세션
│   ├── AI토크 (/community/ai-talk)                     ← NEW: 누구나 개설 가능한 세미나
│   └── 게시글 상세 (/community/{board}/{post-slug})
│
├── 채용 (/jobs)                                         ← NEW (하반기 이후, 낮은 우선순위)
│   ├── 구인 (/jobs/hiring)
│   └── 구직 (/jobs/seeking)
│
├── AI 이력서 (/profile/{username})
│   ├── 개요 (기본 탭) — AI 활용 자동 요약 포함
│   ├── 스터디 이력 (/profile/{username}/studies)
│   ├── 작성글 (/profile/{username}/posts)
│   ├── 포트폴리오 (/profile/{username}/portfolio)
│   └── 포인트/뱃지 (/profile/{username}/badges)        ← NEW: 기여 기반 포인트/뱃지
│
├── 메시지 (/messages)                                   ← NEW: DM 시스템 재설계
│   ├── 받은 메시지 (/messages/inbox)
│   ├── 보낸 메시지 (/messages/sent)
│   └── 자동 알림 (/messages/notifications)              ← 자동 메시지 분리
│
├── 채팅 (/chat)                                         ← NEW (2026년 2Q 이후)
│   └── 채팅방 목록 (/chat/rooms)
│       └── 게시판 ↔ 채팅방 양방향 채널
│
├── 검색 (/search?q={query})
│
├── 인증
│   ├── 로그인 (/auth/login)
│   ├── 회원가입 (/auth/register)
│   └── 비밀번호 재설정 (/auth/reset-password)
│
├── 글쓰기 (/write)
│   ├── 피드 포스트 (/write?type=feed)                   ← NEW: 가벼운 글쓰기
│   ├── 사례 게시글 (/write?type=case)                   ← 기존 상세 게시글
│   └── 임시저장 목록 (/write/drafts)
│
└── 관리자 (/admin)
    ├── 대시보드 (/admin)
    ├── 게시글 관리 (/admin/posts)
    ├── 스터디 관리 (/admin/studies)
    ├── 회원 관리 (/admin/users)
    ├── 홈 배너 관리 (/admin/banners)
    ├── 카테고리/태그 관리 (/admin/taxonomy)
    ├── 상품/쿠폰 관리 (/admin/products)                 ← NEW: Retool 의존 제거
    ├── 뱃지/포인트 관리 (/admin/badges)                 ← NEW: 기여 기준/뱃지 설정
    └── DM/알림 관리 (/admin/messaging)                  ← NEW: 메시지 템플릿, 자동 알림 관리
```

> **사이트맵 변경 사유**: 다혜 인터뷰에서 커뮤니티 활성화를 위해 가벼운 피드 포스트, 모각AI, AI토크 등 참여 진입장벽을 낮추는 공간이 필요하다는 점이 강조되었다. 또한 DM 시스템의 구조적 문제(자동 메시지에 묻히는 중요 메시지)를 해결하기 위해 전용 메시지함을 신설한다. 재호 인터뷰에서 Retool에 의존하던 상품/쿠폰 관리를 어드민 내장으로 전환하고, 뱃지/포인트와 DM/알림 관리 기능을 관리자 영역에 추가한다.

### 3.2 페이지 깊이 분석

| 깊이 | 페이지 예시 | 도달 경로 |
|------|------------|----------|
| 0단계 | 홈 | 직접 |
| 1단계 | 탐색, 스터디, 커뮤니티, AI 이력서, 메시지, 채용 | GNB 클릭 |
| 2단계 | 카테고리별 피드, 스터디 상세, 게시글 상세, 피드 포스트, 모각AI, AI토크, 채팅방 | 1단계 → 항목 클릭 |
| 3단계 | 프로필 탭 (스터디 이력, 포트폴리오, 뱃지), 관리자 하위 (상품/쿠폰, 뱃지/포인트) | 프로필/관리자 → 탭/메뉴 전환 |

**최대 깊이: 3단계** - 신규 추가 페이지(피드, 모각AI, AI토크, 메시지 등) 포함 모든 콘텐츠가 3클릭 이내 접근 가능

## 4. 콘텐츠 분류 체계

### 4.1 현황 문제점

- 약 30개 카테고리가 분야별(콘텐츠/마케팅), 도구별(클로드, 코파일럿, 제미나이) 기준 혼재
- 자주 쓰는 카테고리는 3~4개, 나머지는 사실상 미사용
- 사용자가 "어떤 스페이스에 올려야 할지 고민"하는 상황 발생
- **게시글 형식이 "과제 제출용"처럼 무거움** — 가벼운 공유를 하고 싶어도 진입장벽이 높음 (다혜)
- **스터디원만 게시글 작성 가능** — 일반 회원의 참여 기회 차단 (다혜)
- **관리자가 콘텐츠 수정 시 제한이 많음** — 스페이스와 UI 매핑이 불명확 (재호)

### 4.2 콘텐츠 유형 구분 (NEW)

현재 사이트에서는 "사례 게시글" 하나의 포맷만 존재하여, 가벼운 공유를 원하는 사용자에게 심리적 진입장벽이 높다. 다혜 인터뷰에서 "피드 형태의 가벼운 포스트"와 "자랑 포스트(AI로 만든 서비스/프로덕트)" 등 다양한 콘텐츠 유형이 필요하다는 의견이 강하게 제기되었다.

| 유형 | 설명 | 형식 | 진입장벽 | 작성 권한 |
|------|------|------|----------|-----------|
| **피드 포스트** | 가벼운 짧은 공유, 자랑글 (AI로 만든 서비스 등) | 짧은 텍스트 + 이미지/링크 | 낮음 | 모든 회원 |
| **사례 게시글** | 상세한 AI 활용 사례 (현재 포맷) | 긴 글 (구조화된 리치 텍스트) | 높음 | 모든 회원 (기존: 스터디원만) |
| **AI토크** | 누구나 개설 가능한 세미나/발표 | 이벤트 (일시 + 장소/링크 + 설명) | 중간 | 모든 회원 |
| **모각AI** | 온/오프라인 공동작업 세션 | 이벤트 (일시 + 장소/링크) | 낮음 | 모든 회원 |

> **핵심 변경**: 기존에는 스터디원만 게시글을 작성할 수 있었으나, 누구나 게시글을 올릴 수 있도록 개방한다 (다혜). 피드 포스트는 SNS처럼 부담 없이 올릴 수 있는 형식으로, "과제 제출용"이 아닌 일상적 AI 활용 공유를 촉진한다.

### 4.3 새로운 분류: 카테고리(6개) + 태그 시스템

#### 대분류 카테고리 (6개, 상호배타적)

| 카테고리 | slug | 설명 | 현재 대응 스페이스 |
|----------|------|------|--------------------|
| **AI 활용법** | ai-usage | AI 도구 사용 사례, 팁, 튜토리얼 | 사례 게시글, 바이브 코딩 등 |
| **프롬프트** | prompt | 프롬프트 엔지니어링, 공유 | 프롬프트 관련 게시판 |
| **자동화/노코드** | automation | n8n, Make, Zapier 등 자동화 | 자동화 관련 게시판 |
| **개발/코딩** | dev | AI 기반 개발, 코파일럿 등 | 코파일럿, 개발 관련 게시판 |
| **비즈니스/마케팅** | business | AI 비즈니스 적용, 마케팅 | 콘텐츠/마케팅 게시판 |
| **AI 뉴스** | news | 최신 AI 소식, 트렌드 | 최신 AI 소식 |

#### 태그 시스템 (다중 선택, 교차 분류)

```
도구 태그:      #ChatGPT  #Claude  #Gemini  #Copilot  #Midjourney  #Cursor  ...
난이도 태그:    #입문  #중급  #고급
형식 태그:      #튜토리얼  #사례  #리뷰  #토론  #질문
```

#### 분류 개선 전후 비교

```
[기존] 30개 혼합 카테고리 (분야+도구 혼재)
  → 사용자: "어디에 올려야 하지?" / "검색으로 찾아야 한다"

[개선] 6개 카테고리 + N개 태그 조합
  → 카테고리: "이 글의 주제는 무엇?" (1개 필수 선택)
  → 태그: "어떤 도구/난이도?" (다중 선택, 선택적)
  → 글쓰기 시 명확한 기준, 탐색 시 유연한 필터링
```

### 4.4 카테고리-태그 매핑 예시

| 게시글 제목 | 카테고리 | 태그 |

## 5. 유저 플로우

### 5.1 신규 방문자 → 회원가입

```
[구글 검색/SNS]
       │
       ▼
  게시글 상세 페이지 (SEO 랜딩)
       │
       ├── 글 읽기 (가독성 높은 레이아웃)
       │
       ├── 하단 CTA: "더 많은 AI 활용법 보기"
       │   └── → 탐색 페이지 (피드)
       │
       ├── 사이드 CTA: "이 글이 도움되었다면 가입하세요"
       │   └── → 회원가입 (간편 소셜 로그인)
       │
       └── 관련 스터디 배너: "이 주제를 깊이 배우세요"
           └── → 스터디 상세 페이지
```

**설계 근거**: 마케터 인터뷰에서 "게시글 확인 후 회원가입으로 이어지는 것이 더딤"이라는 이탈 포인트 지적. 게시글 내에 회원가입 유도 장치를 자연스럽게 배치.

### 5.2 콘텐츠 탐색 (회원)

```
  홈 (개인화 피드)
       │
       ├── 추천 콘텐츠 (관심 카테고리 기반)
       ├── 인기 게시글 (투표/조회수 기반)
       └── 최신 글
              │
              ▼
     탐색 (/explore)
       │
       ├── 카테고리 필터 (6개 중 선택)
       ├── 태그 필터 (도구, 난이도, 형식)
       ├── 정렬 (최신 / 인기 / 추천)
       │
       └── 게시글 클릭
              │
              ▼
        게시글 상세
              │
              ├── 투표 (upvote/downvote)
              ├── 댓글
              ├── 북마크
              └── 관련 글 추천
```

### 5.3 스터디 참여

```
  스터디 메뉴 클릭 (GNB)
       │
       ▼
  스터디 목록 (모집 중 우선 노출)
       │
       ├── 카테고리/난이도/기간 필터
       │
       └── 스터디 카드 클릭
              │
              ▼
     스터디 상세 페이지
       │
       ├── 히어로: 스터디명 + 핵심 정보 (기간, 인원, 가격)
       ├── 커리큘럼: 주차별 내용
       ├── 스터디장 프로필
       ├── 수료생 후기
       ├── FAQ
       │
       └── CTA: "스터디 신청하기" (Sticky)
              │
              ▼
         결제/신청 완료
              │
              ▼
       내 스터디 → LMS 연결
```

**설계 근거**: 마케터 인터뷰 "상품 상세 페이지까지는 가지만 전환(구매)이 잘 안 됨" → 패스트캠퍼스 참고하여 전환 최적화된 상세 페이지 설계. Sticky CTA 버튼, 사회적 증거(수료생 후기) 배치.

### 5.4 글 작성

```
  GNB "글쓰기" 버튼 (로그인 필수)
       │
       ▼
   글쓰기 에디터 (/write)
       │
       ├── 카테고리 선택 (6개 중 1개 필수)
       ├── 태그 입력 (자동완성)
       ├── 제목 입력
       ├── 본문 작성 (리치 텍스트 에디터)
       │
       ├── [임시저장] → 자동 저장 (30초) + 수동 저장
       │   └── 임시저장 목록 (/write/drafts)
       │
       └── [발행] → 게시글 상세 페이지로 이동
```

**설계 근거**: 인터뷰 "임시저장 기능이 없어서 작성 중인 내용이 날아질 수 있다" → 자동 저장 기능 필수. "외부 페이지에서 작성 완료 후 복사 붙여넣기" 우회 프로세스 해소.

### 5.5 관리자 운영

```
  프로필 드롭다운 → 관리자 대시보드
       │
       ▼
   관리자 대시보드 (/admin)
       │
       ├── 요약 (오늘 신규 가입, 게시글, 스터디 현황)
       │
       ├── 게시글 관리 (/admin/posts)
       │   ├── 검색 + 카테고리 필터
       │   ├── 일괄 작업 (이동, 삭제, 공개/비공개)
       │   └── 게시글별 빠른 수정
       │
       ├── 스터디 관리 (/admin/studies)
       │   ├── 기수별 목록
       │   ├── 상태 변경 (작성중 → 모집중 → 진행중 → 완료)
       │   ├── 최종제출 토글 (목록에서 바로 조작)
       │   └── 게시판 이동 (드래그앤드롭 또는 원클릭)
       │
       ├── 홈 배너 관리 (/admin/banners)
       │   ├── 배너 순서 변경 (드래그앤드롭)
       │   └── 배너 등록/수정 (이미지 + 링크)
       │
       ├── 상품/쿠폰 관리 (/admin/products)         ← NEW
       │   ├── 상품 목록 + CRUD
       │   ├── 쿠폰 발급/관리
       │   └── (Retool 의존 제거)
       │
       ├── 뱃지/포인트 관리 (/admin/badges)          ← NEW
       │   ├── 포인트 적립 기준 설정
       │   ├── 뱃지 종류/조건 관리
       │   └── 회원별 포인트/뱃지 현황
       │
       ├── DM/알림 관리 (/admin/messaging)           ← NEW
       │   ├── 자동 메시지 템플릿 관리
       │   ├── 알림 규칙 설정
       │   └── 발송 이력 조회
       │
       └── 회원/카테고리/태그 관리
```

**설계 근거**: 인터뷰 "관리 → 디자인 스튜디오 → 컬렉션 및 공간 → ... → 6단계" 복잡성 해소. 자주 쓰는 기능(게시글 관리, 스터디 상태 변경, 배너 수정)을 최소 클릭으로 접근 가능하게 설계. "최종제출 토글"을 목록에서 바로 조작 가능하게 하여 스크롤 깊이 제거. 재호 인터뷰에서 "어드민 스페이스와 프론트 UI 매핑이 불명확"하다는 지적 반영, 관리자 UI에서 프론트 페이지와의 연결을 명확히 표시한다.

### 5.6 가벼운 포스트 작성 (피드형) ← NEW

```
  커뮤니티 > 피드 탭 (/community/feed)
       │
       ▼
  [+ 포스트 작성] 버튼 (진입장벽 최소화)
       │
       ▼
   인라인 작성 폼 (페이지 이동 없음)
       │
       ├── 짧은 텍스트 입력 (최대 500자)
       ├── 이미지/스크린샷 첨부 (선택)
       ├── 링크 첨부 + OG 미리보기 (선택)
       ├── 카테고리 태그 (선택)
       │
       └── [게시] → 피드에 즉시 노출
```

**설계 근거**: 다혜 인터뷰 "현재 게시글이 과제 제출용, 가벼운 포스트가 필요하다"는 피드백 반영. SNS(X/Twitter, LinkedIn)처럼 부담 없이 AI 활용 경험을 공유할 수 있는 진입장벽 최소화 동선. 특히 "자랑 포스트" (AI로 만든 서비스/프로덕트 공유)를 촉진한다.

### 5.7 DM/메시지 플로우 ← NEW

```
  GNB 메시지 아이콘 (읽지 않은 수 표시)
       │
       ▼
   메시지함 (/messages)
       │
       ├── [받은 메시지] 탭 (기본)
       │   ├── 개인 메시지 (수동 DM)     ← 우선 노출
       │   └── 자동 알림 (시스템 메시지)   ← 별도 분리
       │
       ├── [보낸 메시지] 탭
       │
       └── 메시지 상세
              │
              ├── 답장하기
              └── 프로필 보기 → AI 이력서
```

**설계 근거**: 다혜 인터뷰 "DM이 자동메시지에 묻혀 중요한 메시지를 놓친다"는 문제점 해결. 수동 DM과 자동 알림을 명확히 분리하여, 개인 간 소통이 시스템 메시지에 묻히지 않도록 한다.

### 6.3 SEO 고려사항

- **게시글 URL은 최상위 `/posts/{slug}`로 배치**: 카테고리 변경 시에도 URL이 변하지 않음 (SEO 안정성)
- **한글 slug**: 검색 엔진 스니펫에서 한글 키워드 노출 → CTR 향상
- **canonical URL 설정**: 동일 콘텐츠가 여러 경로로 접근 가능할 때 정규 URL 지정
- **구조화된 데이터 (JSON-LD)**: Article, Course, Person 스키마 적용으로 리치 스니펫 노출
- **sitemap.xml 자동 생성**: Next.js의 정적/동적 페이지 자동 크롤링 지원

### 6.4 LLM 최적화

- Open Graph + meta description 충실히 작성
- 게시글 본문의 시맨틱 HTML 구조 (h1-h3 계층)
- 구조화된 FAQ 스키마로 AI 답변 소스 노출 가능성 확보

## 7. 주요 설계 결정 요약

| 결정 | 선택 | 대안 | 선택 이유 |
|------|------|------|-----------|
| 메뉴 수 | 7개 (탐색/스터디/커뮤니티/AI이력서/메시지/검색/인증) | 기존 6개 유지 | 메시지 기능의 독립적 중요성 (다혜: DM 문제), Miller의 법칙 범위 내 |
| 카테고리 수 | 6개 대분류 | 30개 유지 or 10개 | 인터뷰: 3~4개만 자주 사용, 6개면 한 화면 노출 가능 |
| 분류 방식 | 카테고리(주제) + 태그(도구/난이도) | 카테고리만 | 분야별/도구별 혼재 문제 해결, 교차 필터링 가능 |
| 콘텐츠 유형 | 피드 포스트 + 사례 게시글 2트랙 | 사례 게시글만 (현행) | 다혜: "현재 게시글이 과제 제출용, 가벼운 포스트 필요" |
| 글쓰기 권한 | 모든 회원 게시 가능 | 스터디원만 (현행) | 다혜: "누구나 게시글 올리기" — 참여 진입장벽 해소 |
| DM 시스템 | 전용 메시지함 (자동/수동 분리) | 기존 통합 DM 유지 | 다혜: "DM이 자동메시지에 묻혀 중요한 메시지 누락" |
| 상품/쿠폰 관리 | 어드민 내장 | Retool 의존 유지 | 재호: "상품/쿠폰이 Retool에 의존" → 운영 자립성 확보 |
| 메뉴 아이콘 | 시스템 아이콘 최소 사용, 이모지 배제 | 이모지 유지 | 재호: "이모지가 오히려 혼란" |
| 포인트/뱃지 | 기여 기반 포인트 + 뱃지 시스템 | 없음 (현행) | 다혜: "동기부여 효과 큼, 기준 체계화 필요" |
| 게시글 URL | `/posts/{slug}` (flat) | `/explore/category/{cat}/{slug}` | 카테고리 변경 시 URL 불변, SEO 안정성 |
