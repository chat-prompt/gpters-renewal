## 1. Nielsen 10가지 휴리스틱 평가

### H1. 시스템 상태의 가시성 (Visibility of System Status)
**점수: 2/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 임시저장 상태 피드백 없음 | Critical | 게시글 작성 중 저장 상태를 알 수 없고, 페이지 이탈 시 경고 없이 내용 소실 |
| DM 알림-리디렉션 불일치 | Critical | 이메일 알림 클릭 시 해당 DM으로 이동하지 않음 → 중요 메시지 완전 누락 |
| 스터디 진행 상태 불명확 | Major | 현재 모집 중인 스터디가 홈에서 잘 드러나지 않음 |
| 관리 작업 진행 상태 | Minor | 관리 탭에서 깊은 네비게이션 중 현재 위치 파악 어려움 |

**인터뷰 근거**:
- 신연권: "임시저장 기능이 없어서 작성 중인 내용이 날아질 수 있다"
- 신연권: "스터디 모집이 진행되고 있는지는 잘 안 보일 수 있을 것 같다"
- 다혜: 자동 DM이 받은함을 범람시키고, 이메일 알림에서 DM으로의 리디렉션이 작동하지 않아 실제 중요한 DM이 완전히 묻힘

---

### H2. 시스템과 현실 세계의 일치 (Match Between System and Real World)
**점수: 3/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| "스페이스/컬렉션" 용어 혼란 | Major | Bettermode 플랫폼 용어가 일반 사용자에게 직관적이지 않음 |
| 카테고리 명명 불일치 | Major | 도구명(클로드, 코파일럿)과 분야명(콘텐츠/마케팅)이 같은 분류 수준에 혼재 |
| "최종제출 토글" | Minor | 스터디장에게 직관적이지 않은 관리 용어 |

**인터뷰 근거**:
- 신연권: "카테고리의 분류 기준이 모호하다. 어떤 것은 분야별이고, 어떤 것은 도구별이어서 찾기 어렵다"
- 소연: "글을 올릴 때 어떤 스페이스에 올려야 할지 고민될 때가 있음"

---

### H3. 사용자 제어와 자유 (User Control and Freedom)
**점수: 1/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 임시저장/되돌리기 없음 | Critical | 게시글 작성 중 실수로 나가면 전체 내용 소실, 복구 불가 |
| 멤버 게시글 공간이동 불가 | Critical | 멤버가 자기 게시글 공간 변경 불가, 삭제 후 재게시해야 함 → 웹훅 2회 발동 연쇄 문제 |
| 게시글 이동의 어려움 | Major | 잘못된 스페이스에 올린 글을 옮기려면 관리자 도움 필요 |
| 편집 중 취소/되돌리기 부재 | Major | 편집 히스토리나 버전 관리 기능 없음 |
| 북마크(찜) 기능 실종 | Major | 기존 mark 기능이 사라져 찜해놓은 게시물 접근 불가 |

**인터뷰 근거**:
- 신연권: "게시글 작성 중 페이지에서 나가면 내용이 날아간다. 이 때문에 외부 페이지에서 작성을 완료한 후 복사 붙여넣기 방식으로 사이트에 올린다"
- 다혜: 멤버가 게시글을 다른 공간으로 이동할 수 없어 삭제 후 재게시 → 웹훅이 2번 발동되어 n8n 이중 실행 연쇄 장애 발생
- 다혜 (유저 불만): 북마크(찜) 기능이 사라져 기존에 저장해두었던 게시물에 접근 불가

---

### H4. 일관성과 표준 (Consistency and Standards)
**점수: 2/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 디자인 일관성 부족 | Critical | 페이지마다 레이아웃, 스타일, 컴포넌트가 상이 |
| 카테고리 분류 원칙 불일치 | Major | 8개 컬렉션이 각각 다른 분류 기준(도구/분야/시간/행동) 사용 |
| 내비게이션 패턴 불일치 | Major | 헤더 4개, 사이드바 빠른링크 4개, 컬렉션 8개 등 다중 내비게이션 체계 |
| 이모지 남발 | Major | 모든 메뉴 앞에 이모지가 붙어 시각적 혼란을 가중시킴 |
| 외부/내부 링크 구분 없음 | Minor | 스터디장 지원, 기업교육이 외부 링크인지 사전 표시 없음 |

**인터뷰 근거**:
- 소연: "디자인 일관성이 부족하게 느껴짐", "브랜딩이 더 필요하다고 느낌"
- 신연권: 카테고리 분류 기준 혼재
- 재호: 모든 메뉴에 이모지가 붙어 있어 탐색에 도움이 되기보다 오히려 혼란을 줌

---

### H5. 오류 예방 (Error Prevention)
**점수: 1/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 작성 중 이탈 방지 없음 | Critical | 게시글 작성 중 페이지 이탈 시 확인 다이얼로그 없음 |
| 스페이스 선택 가이드 없음 | Major | 글 작성 시 어떤 스페이스가 적합한지 안내 없음 |
| 관리 작업 실수 방지 부족 | Major | 깊은 네비게이션에서 잘못된 항목 수정 가능성 높음 |
| 게시/수정 버튼 위치 불일치 | Major | 게시 버튼이 최하단, 수정 버튼이 최상단 → 긴 글에서 스크롤 혼란 |
| 스터디 종류 오기입 | Major | 스크롤 시 스터디 종류 선택 영역이 뷰포트 밖으로 벗어나 잘못된 종류 선택 |

**인터뷰 근거**:
- 신연권: 임시저장 없어 내용 날아감 → 외부 도구 우회
- 소연: 스페이스 선택 시 고민
- 다혜 (유저 불만): 게시 버튼은 최하단, 수정 버튼은 최상단에 위치하여 긴 글 작성/수정 시 불필요한 스크롤 발생
- 다혜 (유저 불만): 스크롤 시 스터디 종류 선택 필드가 시야에서 사라져 잘못된 스터디 종류를 선택하는 오류 빈발

---

### H6. 인식 vs 회상 최소화 (Recognition Rather Than Recall)
**점수: 2/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 스터디 접근 경로 기억 의존 | Critical | 프로필→내 AI스터디→LMS 경로를 기억해야 함 |
| 카테고리 위치 기억 의존 | Major | 30개 카테고리에서 원하는 것을 찾으려면 검색에 의존 |
| 관리 경로 기억 의존 | Major | 홈 수정 7단계 경로를 기억해야 함 |
| 자주 쓰는 기능 즐겨찾기 없음 | Minor | 자주 사용하는 3~4개 카테고리에 빠른 접근 방법 없음 |

**인터뷰 근거**:
- 신연권: "스터디를 들으려면 어디로 들어가야 하나요?" 빈번한 질문
- 신연권: "필요한 카테고리를 찾을 때 검색으로 찾는 경향이 있다"
- 신연권: 관리 탭 경로 7단계 이상

---

### H7. 유연성과 효율성 (Flexibility and Efficiency of Use)
**점수: 1/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| Admin 스페이스-UI 매핑 불명확 | Critical | 원하는 UI 변경을 위해 어떤 스페이스를 수정해야 하는지 찾기 어려움 |
| 숙련 사용자 단축 경로 없음 | Major | 자주 사용하는 기능에 대한 단축키나 즐겨찾기 없음 |
| 관리자 대시보드 부재 | Major | 운영자가 일상 업무를 한 곳에서 처리할 수 없음 |
| 검색 필터링 제한 | Major | 카테고리/태그/기간 등 복합 필터링 부족 |
| 콘텐츠 수정 제약 | Major | 섹션 타이틀/문구를 자유롭게 수정할 수 없음 |
| 상품/쿠폰 관리 외부 의존 | Major | 상품·쿠폰 관리를 위해 Retool이라는 별도 도구 필요 |
| 개인화 기능 부재 | Minor | 관심 분야/수준에 따른 콘텐츠 추천 없음 |
| 댓글 전체 펼치기 불가 | Minor | "1개 더 많은 답글 보기"를 일일이 클릭해야 함 |

**인터뷰 근거**:
- 신연권: 11개 외부 도구에 의존 → 사이트 자체 효율성 부족
- 방향성.md: "내 수준에 맞는, 내가 지금 관심있는 다른 콘텐츠를 발견하기 어렵다"
- 재호: Admin에서 원하는 UI를 변경하려고 해도 어떤 스페이스를 수정해야 하는지 직관적으로 알 수 없음
- 재호: 섹션 타이틀이나 문구를 자유롭게 수정할 수 없어 콘텐츠 운영에 제약
- 재호: 상품/쿠폰 관리가 Retool에 의존하여 관리 포인트가 분산됨
- 다혜 (유저 불만): 댓글 답글을 한꺼번에 펼칠 수 없어 긴 댓글 스레드 탐색이 비효율적

### H8. 미학적이고 미니멀한 디자인 (Aesthetic and Minimalist Design)
**점수: 2/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 홈 화면 정보 과밀 | Critical | 4개 섹션(웨비나, 최신, 인기, 베스트) + 배너 + 사이드바가 동시 노출 |
| 게시글 가독성 부족 | Critical | 본문 타이포그래피, 여백, 이미지 처리 등이 타 사이트 대비 부족 |
| 사이드바 항목 과다 | Major | 8개 컬렉션 + 4개 빠른링크 + 추가 섹션이 한 사이드바에 밀집 |
| 모바일 최적화 부족 | Major | 모바일 가독성, UI 친화성 부족 |
| 영상 콘텐츠 색인 부재 | Major | 수백 개 AI캠프 영상에서 원하는 콘텐츠를 찾을 수 없음 (인덱스/목차 없음) |

**인터뷰 근거**:
- 신연권: 신규 회원이 "뭐가 이렇게 많지?" 반응
- 소연: "게시글 가독성이 좋지 않다", "모바일도 친화적이지 않음"
- 다혜 (유저 불만): 수백 개 AI캠프 영상 중 원하는 것을 찾으려면 하나하나 열어봐야 함, 검색이나 색인이 전혀 없음

---

### H9. 오류 인식, 진단, 복구 지원 (Help Users Recognize, Diagnose, and Recover from Errors)
**점수: 3/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 서버 오류 시 안내 부족 | Major | 간헐적 서버 다운 시 사용자에게 명확한 안내/대안 부족 |
| 작성 실패 시 복구 불가 | Major | 게시글 작성 중 오류 발생 시 작성 내용 복구 방법 없음 |

**인터뷰 근거**:
- 신연권: "가끔씩 서버가 터지는 일이 있다"

---

### H10. 도움말과 문서 (Help and Documentation)
**점수: 3/5**

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 사용가이드 접근성 | Major | 사이드바에 존재하지만, 신규 사용자가 자발적으로 찾아볼 가능성 낮음 |
| 맥락적 도움말 부재 | Major | 각 기능/페이지에서 즉각적인 도움말 제공 없음 |
| 온보딩 플로우 없음 | Minor | 가입 직후 사이트 활용법 안내 없음 |

**인터뷰 근거**:
- 신연권: 신규 참여자 "스터디 어디서 듣나요?" → 사용가이드가 있어도 찾지 못함

## 6. 결론

현재 GPTers 사이트는 Nielsen 휴리스틱 평균 1.9/5로, **전반적으로 미흡한 사용성** 수준이다. 특히 **사용자 제어와 자유(H3)**, **오류 예방(H5)**, **유연성과 효율성(H7)** 영역이 1/5로 가장 낮으며, 이는 임시저장 부재, 게시글 이동 불가, DM 시스템 미작동, Admin UI의 구조적 결함이 주요 원인이다.

4명의 인터뷰이(신연권, 소연, 재호, 다혜)와 유저 불만 게시글 24건에서 교차 검증된 증거를 통해, Critical 이슈 10건 중 다수가 **콘텐츠 작성/소비 경험**, **운영 도구의 구조적 한계**, **커뮤니케이션 시스템 장애**와 직결되어 있음이 확인되었다. 리뉴얼의 최우선 과제는 다음과 같다:

1. **콘텐츠 가독성과 작성 경험 혁신** (임시저장, 에디터, 타이포그래피, 버튼 위치 일관성)
2. **정보 구조 단순화** (카테고리 통폐합, 점진적 공개, 역할 기반 홈)
3. **운영 도구 자립** (Admin UI 직관화, 상품/쿠폰 관리 내재화, Retool 의존 탈피)
4. **커뮤니케이션 시스템 정상화** (DM 시스템 재설계, 알림-리디렉션 수정)
5. **전환 퍼널 최적화** (SEO 유입 → 가입 → 스터디 구매)
6. **디자인 일관성 확보** (디자인 시스템 기반 리빌드, 이모지 가이드라인)

Next.js + shadcn 기반 리뉴얼은 이러한 구조적 문제를 근본적으로 해결할 수 있는 기회이며, 디자인 시스템(`@gpters-internal/ui`)의 토큰 기반 접근이 H4(일관성) 문제 해결의 핵심이 될 것이다. 특히 4명의 인터뷰에서 공통적으로 확인된 **Bettermode 플랫폼의 근본적 제약**(DM 시스템, 게시글 이동, Admin UI, 콘텐츠 수정 자유도)은 단순 UI 개선이 아닌 **플랫폼 전환의 정당성**을 강력히 뒷받침한다.
