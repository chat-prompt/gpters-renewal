# GPTers 사이트 리뉴얼 제안

> 2026.02.25 | 미팅용

---

## 1. 왜 리뉴얼해야 하는가

### 현재 GPTers의 3가지 핵심 문제

운영팀(연권, 소연) 인터뷰를 통해 확인된 구조적 문제입니다.

---

### 문제 1: 콘텐츠를 발견할 수 없다

> "게시글이 관심이 있어 들어왔는데, 내 수준에 맞는 다른 콘텐츠를 발견하기 어렵다" - 방향성 문서

**현황**
- 카테고리가 약 30개인데, 분류 기준이 뒤섞여 있음
  - 도구별: 클로드, 코파일럿, 제미나이
  - 분야별: 콘텐츠/마케팅
  - 시간별: OO기 스터디 모집
- 자주 쓰는 카테고리는 3~4개뿐, 나머지는 사실상 미사용
- 운영자도 "필요한 카테고리를 검색으로 찾는다" (연권)
- 마케터도 "글을 올릴 때 어떤 스페이스에 올려야 할지 고민된다" (소연)

**결과**: 신규 방문자가 홈 화면을 보고 "뭐가 이렇게 많지?"하고 이탈

---

### 문제 2: 유입은 되는데 전환이 안 된다

> "게시글 확인 후 회원가입으로 이어지는 것이 더딤" - 소연

**현재 전환 퍼널의 두 가지 병목**

```
[이탈 포인트 1] 게시글 읽기 → 회원가입
  - 게시글 가독성이 타 사이트 대비 부족 (소연)
  - 모바일 친화적이지 않음 (소연)
  - 회원가입 유도 장치가 게시글 내에 없음

[이탈 포인트 2] 상품 상세 → 스터디 구매
  - 상세페이지까지는 가지만 구매로 이어지지 않음 (소연)
  - 스터디 모집이 진행 중인지 홈에서 잘 안 보임 (연권)
```

**결과**: SEO로 유입은 잘 되지만 돈이 되는 전환(가입, 구매)에서 빠짐

---

### 문제 3: 운영이 너무 비효율적이다

> "관리 탭의 UI가 복잡해서 원하는 부분을 찾기 위해 여러 번 헤맴" - 연권

**홈 배너 하나 수정하는 데 7단계**
```
관리 → 디자인 스튜디오 → 컬렉션 및 공간 → 컬렉션 없음
  → 홈 → Body → 판매주간 홈배너 → Column 2 → 게시물 6
```

**스터디 관리도 다단계 프로세스**
```
스터디장 모집(외부) → 상세페이지 작성 요청 → 수정 확인
  → 최종제출 토글(스크롤해서 찾기) → 에어테이블 확인
  → 게시물 옮기기(점 세 개 → 해당 게시판)
```

**운영자가 사용하는 외부 도구: 11개**
슬랙, 에어테이블, n8n, 노션, 구글시트, 리니어, Zoom, 지메일 등

> "사이트 자체가 불편하게 느껴져서 잘 안 쓰려고 한다" - 연권

**결과**: 사이트가 핵심 운영 도구가 아닌 "LMS로 가는 중간 다리"로 전락

## 2. 어떻게 해결하는가

### 해결 1: 30개 카테고리 → 6개 + 태그

**Before**: 30개 카테고리 (도구별/분야별/시간별 혼재)
```
클로드 | 코파일럿 | 제미나이 | 바이브코딩 | 콘텐츠/마케팅 |
자동화 | 최신 AI 소식 | 프롬프트 | OO기 스터디 모집 |
커뮤니티 이벤트 | AI 캠프 | ... (약 30개)
```
- 사용자: "이 글 어디에 올리지?"
- 운영자: "찾으려면 검색해야 함"

**After**: 6개 카테고리 (주제별, 상호 배타적) + 태그 시스템
```
카테고리 (1개 필수 선택):
  AI 활용법 | 프롬프트 | 자동화/노코드 | 개발/코딩 | 비즈니스/마케팅 | AI 뉴스

태그 (다중 선택, 선택적):
  도구: #Claude #ChatGPT #Gemini #Cursor ...
  난이도: #입문 #중급 #고급
  형식: #튜토리얼 #사례 #리뷰 #질문
```

**예시**: "Claude로 마케팅 자동화 구축기"
→ 카테고리: **비즈니스/마케팅** / 태그: #Claude #자동화 #사례 #중급

**효과**:
- 글 올릴 때 고민 없음 (6개 중 하나만 선택)
- 탐색할 때 태그로 교차 필터링 가능
- 한 화면에 카테고리 전부 노출 가능

---

### 해결 2: 전환 퍼널 최적화

**게시글 → 회원가입 전환 개선**
```
현재: 글 읽기 → (유도 장치 없음) → 이탈
개선: 글 읽기 → 본문 중간/하단 CTA 노출 → 간편 가입
                              ↓
              "이 글이 도움되었나요? 매주 AI 인사이트를 받아보세요"
              [구글로 가입] [뉴스레터만 구독]
```

**게시글 가독성 대폭 개선**
- 본문 영역 max-width 720px, 충분한 행간(1.8)
- 모바일 최적화 레이아웃
- Sticky 사이드 액션바 (투표/댓글/북마크)

**스터디 상세 → 구매 전환 개선** (패스트캠퍼스 참고)
```
현재: 상세페이지 도달 → (설득력 부족) → 이탈
개선: 히어로(핵심 정보 + 모집률)
      → 소구 포인트("이런 분에게 추천")
      → 주차별 커리큘럼 (아코디언)
      → 스터디장 프로필
      → 수료생 후기 (사회적 증거)
      → Sticky CTA 버튼 ("잔여 8석 | 스터디 신청하기")
```

**홈에서 스터디 모집 가시성 확보**
- 히어로 배너에 현재 모집 중인 스터디 노출
- 별도 "모집 중인 스터디" 섹션 배치
- 모집 진행률 시각화 (████████░░ 60%)

---

### 해결 3: 운영 효율화

**관리자 UI 완전 재설계**
```
현재 (Bettermode):               개선:
배너 수정: 7단계              →  배너 수정: 2단계 (관리자 → 배너)
                                  드래그앤드롭으로 순서 변경

스터디 상태 변경: 5단계       →  스터디 관리: 2단계 (관리자 → 스터디)
  (게시글 진입→수정→스크롤       목록에서 바로 상태 드롭다운 변경
   →토글 찾기→ON)                최종제출 토글도 목록에서 원클릭

게시글 이동: 4단계            →  게시글 관리: 2단계 (관리자 → 게시글)
  (점 세 개→옵션→이동→선택)      목록에서 [이동] 버튼 원클릭
```

**임시저장 도입**
```
현재: 외부에서 작성 → 복사 붙여넣기 (연권: "내용이 날아질 수 있어서")
개선: 자동 저장 (30초 간격) + 수동 저장 + 임시저장 목록
```

## 3. 새로운 사이트 전체 구성

### 네비게이션 (GNB)

```
┌──────────────────────────────────────────────────────────┐
│  [GPTers]   탐색   스터디   커뮤니티   AI이력서   검색  로그인  │
└──────────────────────────────────────────────────────────┘
```

| 메뉴 | 역할 | 왜 필요한가 |
|------|------|------------|
| 탐색 | 콘텐츠 발견 허브 (피드 + 필터) | 핵심 문제 "콘텐츠 발견 어려움" 해결 |
| 스터디 | 모집/진행 중 스터디 목록 | "스터디 모집이 잘 안 보인다" 해결 |
| 커뮤니티 | 자유 게시판, Q&A, 이벤트 | 커뮤니티 성격 강화 |
| AI 이력서 | 프로필 + 스터디 이력 + 포트폴리오 | "활동을 경력으로 보여주기 어렵다" 해결 |

### 사이트맵

```
홈 (/)
├── 탐색 (/explore)
│   ├── 피드 ─ 6개 카테고리 탭 + 태그 필터 + 투표 정렬
│   └── 게시글 상세 (/posts/{slug}) ─ SEO 랜딩, 전환 CTA
│
├── 스터디 (/study)
│   ├── 목록 ─ 모집 중 우선 노출
│   ├── 상세 (/study/{slug}) ─ 전환 최적화 (패스트캠퍼스 참고)
│   └── 내 스터디 (/study/my) ─ LMS 바로가기
│
├── 커뮤니티 (/community)
│   ├── 자유 게시판
│   ├── Q&A
│   └── 이벤트/모임
│
├── AI 이력서 (/profile/{username})
│   ├── 스킬맵 (카테고리별 활동량 시각화)
│   ├── 스터디 이력 (기수, 수료, 프로젝트)
│   └── 포트폴리오
│
├── 글쓰기 (/write) ─ 임시저장 포함
│
└── 관리자 (/admin) ─ 최대 2단계 깊이
    ├── 대시보드 (오늘 요약 + 빠른 작업)
    ├── 게시글 관리 (검색 + 일괄 작업)
    ├── 스터디 관리 (상태/토글 원클릭)
    ├── 배너 관리 (드래그앤드롭)
    └── 회원/분류 관리
```

**핵심 설계 원칙**
- 모든 콘텐츠 3클릭 이내 도달
- 게시글 URL은 `/posts/{slug}` (카테고리 변경해도 URL 불변, SEO 안정)
- 한글 slug로 검색 엔진 노출 최적화

## 4. 유저별 액션 플로우

### Flow 1: 신규 방문자 → 회원 전환

```
구글에서 "Claude 마케팅 자동화" 검색
         │
         ▼
    게시글 페이지 유입 (SEO)
         │
         ├─ 가독성 높은 본문 읽기
         │  (720px, 행간 1.8, 모바일 최적화)
         │
         ├─ 스크롤 중 인라인 CTA 노출
         │  "이 글이 도움되었나요? GPTers에서 더 많은 인사이트를 받아보세요"
         │  [구글로 가입] [뉴스레터만 구독]
         │
         ├─ 관련 스터디 배너 노출
         │  "이 주제를 깊이 배우고 싶다면? → AI 자동화 스터디"
         │
         └─ 하단 관련 글 추천 → 추가 콘텐츠 소비 → 체류시간 증가
                                                         │
                                                         ▼
                                                   회원가입 전환
```

**기존 대비 개선점**:
- 게시글 내 회원가입 유도 장치 배치 (기존에는 없음)
- 가독성 개선으로 콘텐츠 소비 경험 향상
- 콘텐츠 → 스터디 연결 경로 추가

---

### Flow 2: 회원 → 콘텐츠 탐색

```
홈 방문
  │
  ├─ 히어로 배너: 현재 모집 중인 스터디 확인
  │
  ├─ 인기 콘텐츠: 투표수 기반 큐레이션 (Reddit식)
  │
  └─ 카테고리 탐색 → 탐색 페이지 이동
                          │
                ┌─────────┴─────────┐
                │                   │
          카테고리 탭 선택     태그 필터 조합
          (6개 중 택 1)       (#Claude #입문)
                │                   │
                └─────────┬─────────┘
                          │
                    정렬 선택
                    (인기순/최신순/추천순)
                          │
                          ▼
                    게시글 목록
                          │
                    ├─ 투표 (upvote/downvote)
                    ├─ 북마크
                    └─ 게시글 클릭 → 상세 읽기
```

**기존 대비 개선점**:
- 30개 카테고리 탐색 → 6개 탭 + 태그 필터로 단순화
- 투표 기반 큐레이션으로 양질의 콘텐츠 자연 상승
- 소연: "커뮤니티 특징을 살린 투표 기능" 요청 반영

---

### Flow 3: 회원 → 스터디 구매

```
홈 배너 or GNB "스터디" 클릭
         │
         ▼
    스터디 목록 (모집 중 우선 노출)
         │
         ├─ 카테고리/난이도/기간 필터
         │
         └─ 스터디 카드 클릭
                  │
                  ▼
         스터디 상세 페이지
                  │
    ┌─────────────┼─────────────┐
    │             │             │
  히어로        커리큘럼     수료생 후기
  (핵심 정보    (주차별      (사회적 증거)
   + 모집률)    아코디언)         │
    │             │             │
    └─────────────┼─────────────┘
                  │
                  ▼
         Sticky CTA: "잔여 8석 | 스터디 신청하기"
                  │
                  ▼
             결제 완료
                  │
                  ▼
         내 스터디 → LMS 바로가기
```

**기존 대비 개선점**:
- 스터디 진입: GNB 1클릭 (기존: 배너 발견 의존)
- 모집 진행률 시각화로 긴급성 유발
- 수료생 후기로 사회적 증거 제공
- Sticky CTA로 어느 스크롤 위치에서든 신청 가능

---

### Flow 4: 스터디 참여자 → 학습

```
로그인 후
  │
  ├─ 방법 1: GNB "스터디" → 내 스터디 (2클릭)
  │
  └─ 방법 2: 프로필 드롭다운 → "내 스터디" (1클릭, 최상단 배치)
                  │
                  ▼
         내 스터디 목록
         (현재 참여 중인 스터디)
                  │
                  ▼
         LMS 바로가기 → 학습 진행
                  │
                  ▼
         사례글 작성 (에디터, 임시저장 지원)
                  │
                  ▼
         AI 이력서에 자동 반영
         (스터디 이력, 사례글, 스킬맵 업데이트)
```

**기존 대비 개선점**:
- 스터디 진입: 최대 2클릭 (기존: 프로필→내 AI스터디→로그인→LMS = 4단계)
- "스터디 어디서 들어요?" 문의 해소
- 활동이 AI 이력서에 자동 축적

---

### Flow 5: 운영자 → 스터디 관리

```
프로필 드롭다운 → "관리자 대시보드"
         │
         ▼
    관리자 대시보드
    (오늘 신규가입 +12, 게시글 +34, 스터디 신청 +5)
         │
         ├─ 빠른 작업: [배너 수정] [스터디 상태변경] [게시글 이동]
         │
         └─ 사이드바 "스터디" 클릭
                  │
                  ▼
         스터디 관리 목록
                  │
    ┌─────────────┼──────────────┐
    │             │              │
  상태 변경     최종제출 토글   랜딩페이지 이동
  [작성중 ▼]    [OFF → ON]     [원클릭 이동]
  드롭다운으로   목록에서 바로   점 세 개 찾을
  즉시 변경      원클릭 조작     필요 없음
```

**기존 대비 개선점**:
- 배너 수정: 7단계 → 2단계
- 최종제출 토글: 게시글 진입 + 스크롤 → 목록에서 원클릭
- 게시물 이동: 점 세 개 메뉴 탐색 → 원클릭 버튼

---

### Flow 6: 마케터 → 콘텐츠 발행

```
글쓰기 버튼 클릭 (GNB)
         │
         ▼
    에디터 페이지
         │
         ├─ 카테고리 선택 (6개 중 1개, 명확한 기준)
         ├─ 태그 입력 (자동완성)
         ├─ 제목 + 본문 작성
         │
         ├─ 자동 저장 (30초마다) ← 기존 임시저장 부재 해소
         │
         └─ [발행] 클릭
                  │
                  ▼
         게시글 공개 → SEO 자동 최적화
         (한글 slug, 구조화 데이터, sitemap 자동)
```

**기존 대비 개선점**:
- 임시저장으로 "외부에서 작성 후 복붙" 우회 프로세스 제거
- 카테고리 6개로 "어디에 올려야 하지?" 고민 해소
- SEO 자동 최적화 (한글 slug, JSON-LD, sitemap)
