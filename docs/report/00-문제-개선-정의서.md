# 문제-개선 정의서 (초안)

> 목적: 프로토타입 기획의 근간이 되는 문제 정의와 개선 방향 확정
> 데이터 소스: 인터뷰 4건 (신연권, 소연, 재호, 다혜) + 유저 불만 24건 + 리포트 6건
> 상태: **검토 완료** (2026-02-25)

---

## 1. 방문자 (비회원)

### P1. 게시글 → 가입 전환이 약하다 ✅
- **현재**: 리드인 페이월이 있지만 전환이 더딤. 페이월 시점에 "왜 가입해야 하는지" 설득이 부족
- **근거**: 소연 — "게시글 확인 후 회원가입으로 이어지는 것이 더딤"
- **심각도**: High
- **있어야 할 상태**: 문맥 기반 CTA + 독자층 정보 같은 소셜 프루프 + 댓글/스크랩 등 상호작용 게이팅으로 자연스러운 전환 유도 (요즘IT 참조)

### P2. 게시글 가독성이 나쁘다 ✅
- **현재**: 타이포그래피, 여백, 이미지 처리가 타 사이트 대비 부족. 모바일은 더 심각
- **근거**: 소연 — "게시글 가독성이 좋지 않다", "모바일도 친화적이지 않음"
- **심각도**: Medium (새 사이트에서 디자인 토큰 기반으로 구조적 해결 가능)
- **있어야 할 상태**: 디자인 시스템 텍스트 토큰(font-size, line-height, spacing)을 잘 설정하여 일관된 가독성 확보. 모바일 반응형 기본 적용

### P3. 홈에 들어오면 뭐가 뭔지 모르겠다 ✅
- **현재**: 띠배너 + 웨비나 + 최신글 + 인기글 + 베스트사례 + 사이드바 30개 카테고리가 한꺼번에 노출
- **근거**: 신연권 — 신규 회원 반응: "뭐가 이렇게 많지?"
- **심각도**: Critical
- **있어야 할 상태**: 정보 계층이 명확한 홈 UI 구조. 핵심 가치(콘텐츠, 스터디)가 즉시 파악되고, 섹션별 역할이 분명한 레이아웃

## 2. 일반 회원

### P4. 카테고리가 30개인데 기준도 제각각이라 원하는 콘텐츠를 못 찾는다 ✅
- **현재**: 도구별(클로드, 코파일럿), 분야별(마케팅), 시간별(기수) 분류가 혼재. 자주 쓰는 건 3~4개뿐
- **근거**: 신연권 — "중구난방인 카테고리", 소연 — "어떤 스페이스에 올려야 할지 고민", 다혜 — "통합 필요", 재호 — "카테고리가 너무 많다"
- **심각도**: Critical (인터뷰 4명 전원 언급)
- **있어야 할 상태**: 명확한 기준의 소수 카테고리 + 태그 조합으로 유연한 탐색

### P5. 검색이 안 되고, 정렬 기준도 모르겠고, 북마크도 사라졌다 ✅
- **현재**: 검색 결과 부정확, 정렬 기준 불명확, 북마크(찜) 기능 실종, 수백 개 영상 콘텐츠에 색인 없음
- **근거**: 유저 불만 24건 (다혜 취합) — 검색/정렬/북마크 관련 다수 불만
- **심각도**: High
- **있어야 할 상태**: 정확한 검색 + 명확한 정렬(최신/인기/추천) + 북마크 기능 + 콘텐츠 색인

### P6. 가볍게 글을 쓸 수가 없다 — 게시판이 "과제 제출용" ✅
- **현재**: 게시글 형식이 무겁고(긴 사례글만 가능), 스터디 참여자만 작성 가능. 가벼운 공유/자랑은 할 수 없음
- **근거**: 다혜 — "현재 게시글이 과제 제출용", "가벼운 포스트 필요", "누구나 게시글 올리기"
- **심각도**: High
- **있어야 할 상태**: 레딧처럼 텍스트 몇 줄 + 이미지/링크로 바로 올릴 수 있는 가벼운 피드 포스트 + 기존 사례 게시글 2트랙. 누구나 작성 가능

### P7. 글 쓰다가 날아간다 — 임시저장이 없다 ✅
- **현재**: 작성 중 페이지 이탈 시 경고 없이 전체 소실. 외부 도구에서 먼저 작성 후 복붙하는 우회 프로세스 발생
- **근거**: 신연권 — "임시저장 기능이 없어서 작성 중인 내용이 날아질 수 있다. 외부 페이지에서 작성 완료 후 복사 붙여넣기"
- **심각도**: Medium (기술 구현 문제, UX 설계 범위 밖. 프로토타입 단계 범위 밖)
- **있어야 할 상태**: 자동 저장 + 임시저장 목록 + 이탈 시 경고 (개발 단계에서 처리)

### P8. 게시/수정 버튼 위치가 반대고, 댓글도 하나씩 펼쳐야 한다 ✅
- **현재**: 게시 버튼은 최하단, 수정 버튼은 최상단. 긴 글에서 스크롤 압박. 댓글은 "1개 더 보기"를 일일이 클릭
- **근거**: 유저 불만 게시글 — 반복 언급
- **심각도**: Low (새 사이트에서 기본 설계로 해결)
- **있어야 할 상태**: 버튼 위치 일관성 + 댓글 전체 펼치기 옵션

### P9. 이모지가 오히려 혼란스럽다 ✅
- **현재**: 모든 메뉴/카테고리 앞에 이모지가 붙어있어 시각적 노이즈
- **근거**: 재호 — "이모지가 오히려 혼란을 준다"
- **심각도**: Low (디자인 시스템에서 SVG 아이콘만 사용으로 자동 해결)
- **있어야 할 상태**: 시스템 아이콘 최소 사용, 이모지 배제

## 3. 스터디 참여자

### P10. 스터디를 어디서 듣는지 모르겠다 ✅
- **현재**: 프로필 → 내 AI스터디 → 로그인 → LMS 이동이라는 4단계. 신규 참여자가 매번 문의
- **근거**: 신연권 — "스터디를 들으려면 어디로 들어가야 하나요?" 빈번한 질문
- **심각도**: Critical (최우선 — 유료 고객이 서비스 진입을 못 하는 문제)
- **있어야 할 상태**: 로그인 후 "내 스터디" 바로 접근 가능. GNB + 프로필 드롭다운 모두에서 1클릭 도달

### P11. 스터디 상세페이지까지 가도 결제를 안 한다 ✅
- **현재**: 상세 도달 후 전환 저조. 랜딩 설득력 부족 + 사이드바/GNB 그대로 유지되어 구매 몰입감 없음 + 사이트 내 스터디 CTA 노출 부족
- **근거**: 소연 — "상품 상세 페이지까지는 가지만 전환이 잘 안 됨"
- **심각도**: High
- **있어야 할 상태**: (1) 스터디 상세는 전용 랜딩 레이아웃 (사이드바 제거, Sticky CTA) (2) 홈/피드/게시글 등 사이트 곳곳에 모집 중 스터디 CTA 자연스럽게 노출

### P12. 내 활동이 이력/포트폴리오로 안 보인다 ✅
- **현재**: 프로필이 단순. 스터디 이력, 작성글, 기여도가 외부에 보여지지 않음
- **근거**: 다혜 — "마이페이지의 브런치화", "셀프브랜딩 가능한 프로필", "AI계의 LinkedIn" 비전
- **심각도**: Critical (GPTers 핵심 비전 "AI계의 LinkedIn" 실현의 근간)
- **있어야 할 상태**: 링크드인처럼 스터디 수료 이력, 작성글, 뱃지, AI 활용 요약이 프로필에 자동 반영. 외부 공유 가능한 셀프브랜딩 페이지

## 4. 운영자/관리자

### P13. 관리 탭이 7단계 이상 — 원하는 걸 수정하려면 헤맨다 ✅
- **현재**: 홈 배너 수정에 관리→디자인스튜디오→컬렉션및공간→컬렉션없음→홈→Body→판매주간홈배너→Column2→게시물6
- **근거**: 신연권 — "관리 탭의 UI가 복잡해서 원하는 부분을 찾기 위해 여러 번 헤매곤 한다"
- **심각도**: Critical
- **있어야 할 상태**: 관리자 대시보드에서 자주 쓰는 기능(배너, 게시글, 스터디) 1~2클릭 접근
- **참고**: 어드민 기획문서 별도 작성 필요

### P14. 어드민에서 뭘 수정하면 프론트 어디가 바뀌는지 모른다 ✅
- **현재**: 스페이스와 실제 UI의 매핑이 불명확. 수정 시행착오 반복
- **근거**: 재호 — "어떤 스페이스를 수정해야 원하는 UI가 변경되는지 찾기가 어려움"
- **심각도**: Critical
- **있어야 할 상태**: 관리자 UI에서 프론트 페이지와의 연결이 명확하게 표시. 미리보기 가능
- **참고**: P13과 함께 어드민 기획문서에서 다룰 것

### P15. 멤버 게시글을 다른 게시판으로 옮길 수 없다 → 웹훅이 2번 터진다 ✅
- **현재**: 멤버가 잘못된 게시판에 올리면 삭제→재게시해야 함. 이 과정에서 n8n 웹훅이 2회 발동되어 자동 요약이 중복 생성
- **근거**: 다혜 — "자주 발생하는 문제", "웹훅 중복 발동"
- **심각도**: Critical
- **있어야 할 상태**: 게시글 이동 기능 제공 + 웹훅 중복 방지
- **참고**: P13, P14와 함께 어드민 기획문서에서 다룰 것

### P16. 상품/쿠폰 관리를 Retool에서 해야 한다 ✅
- **현재**: 상품 생성, 쿠폰 연결이 Bettermode 밖의 Retool이라는 별도 도구에 의존
- **근거**: 재호 — "상품을 생성하고 쿠폰을 연결하는 과정이 리툴이라는 별도 도구를 사용해야 해서 불편"
- **심각도**: Medium
- **있어야 할 상태**: 어드민 내에서 상품/쿠폰 CRUD 가능. 외부 도구 의존 제거
- **참고**: P13~P15와 함께 어드민 기획문서에서 다룰 것

### P17. 콘텐츠(타이틀, 문구)를 자유롭게 수정할 수 없다 ✅
- **현재**: 각 섹션의 타이틀/문구 수정에 제약이 많음
- **근거**: 재호 — "각 섹션에서 타이틀이나 문구 등을 자유롭게 수정하는 데 제약사항이 많다"
- **심각도**: Medium
- **있어야 할 상태**: 마케터가 주요 영역의 텍스트를 직접 수정 가능
- **참고**: P13~P16과 함께 어드민 기획문서에서 다룰 것

### P18. DM이 자동 메시지에 묻혀서 중요한 메시지를 놓친다 ✅
- **현재**: 신규가입 자동 DM이 수신함을 점령. 이메일 알림 클릭해도 해당 메시지로 이동 안 됨. 중요 메시지 완전 유실
- **근거**: 다혜 — "보낸 DM 내역이 다 보여져서 실제 받은 DM이 묻힘", "이메일에서 클릭해도 리디렉션이 안 됨"
- **심각도**: Medium (탭 분리로 구조적 해결 가능 — 와이어프레임에서 이미 3탭 분리 설계됨)
- **있어야 할 상태**: 자동 알림과 개인 메시지 탭 분리. 알림 클릭 시 정확한 메시지로 딥링크

## 5. 브랜드/디자인

### P19. 디자인 일관성이 없어서 브랜드가 약하다 ✅
- **현재**: 페이지마다 레이아웃, 스타일, 컴포넌트가 다름
- **근거**: 소연 — "디자인 일관성이 부족하게 느껴짐", "브랜딩이 더 필요하다"
- **심각도**: Low (디자인 시스템 토큰 + 공통 컴포넌트로 자동 해결)
- **있어야 할 상태**: 디자인 시스템 기반의 일관된 UI

### P20. 광고/프로모션 지면이 잘 안 보인다 ✅
- **현재**: 배너 등 프로모션 영역이 효과적으로 활용되지 않음
- **근거**: 소연 — "광고 배너 같은 게 더 잘 보였으면 좋겠음"
- **심각도**: Medium (프로모션 영역을 어디에 어떻게 배치할지는 기획 단계에서 설계 필요)
- **있어야 할 상태**: 홈/피드/게시글 등 주요 동선에 프로모션 영역이 전략적으로 배치되고, 어드민에서 관리 가능

## 6. 플랫폼 구조

### P21. 사이트가 "LMS 가는 중간 다리"로만 쓰인다 ✅
- **현재**: 스터디 진행의 실제 학습(다시보기, 세션 참여)은 외부 LMS. 사이트는 결제/과제 제출 진입점에 그침
- **근거**: 신연권 — 사이트는 "중간 다리" 역할
- **심각도**: Critical (외부 LMS를 사이트에 내재화해야 해결되는 구조적 문제)
- **있어야 할 상태**: 다시보기(VOD), 과제 연동 권한, 청강, 진도 추적 등 LMS 기능이 사이트 안에 내재화. 수강생 대시보드 필요
- **참고**: 수강생 대시보드 기획문서 별도 작성 필요 (현재 미설계 영역)

### P22. 운영자가 사이트보다 외부 도구 11개를 더 많이 쓴다 ✅
- **현재**: 슬랙, 에어테이블, n8n, 노션, 구글시트, 리니어, Retool 등 11개 도구 의존
- **근거**: 신연권 — "사이트 자체가 불편하게 느껴져서 잘 안 쓰려고 한다"
- **심각도**: High
- **있어야 할 상태**: 핵심 운영 기능이 사이트 안에 내재화. 외부 도구 연동은 선택적
- **참고**: 어드민 재설계(P13~P17) + LMS 내재화(P21)로 자연스럽게 해결

### P23. 스터디장(리더)의 관리 경험이 나쁘다 ✅
- **현재**: 신규 스터디장이 상세페이지 위치를 못 찾고, 최종제출 토글은 긴 스크롤 속에 숨어있음. 스터디장은 운영자가 아닌 사용자인데 어드민 수준의 네비게이션을 강요받음
- **근거**: 신연권 — "신규 스터디장들은 사이트 자체가 생소하고 상세페이지 위치를 찾기 어려워한다", "스크롤을 내릴 때 세로로 나열된 많은 내용이 있어서 한눈에 찾기 쉬운 구조가 아님"
- **심각도**: High
- **있어야 할 상태**: 스터디장 전용 대시보드 또는 중간관리자 권한으로 자기 스터디를 쉽게 관리. LMS 내재화(P21)와 함께 설계 필요
- **참고**: 수강생 대시보드 + 스터디장 권한 기획에서 다룰 것

### P24. 불필요한 페이지/기능이 정리 안 되어 있다 ✅
- **현재**: 가입인사 게시판 등 의미 불명확한 페이지가 존재. 안 쓰는 페이지가 쓰는 페이지보다 훨씬 많음
- **근거**: 재호 — "가입인사 게시판이 의미 있는지 의문", 신연권 — "안 쓰는 페이지가 쓰는 페이지보다 훨씬 많다"
- **심각도**: Medium
- **있어야 할 상태**: 모든 페이지/기능의 역할이 명확하고, 불필요한 것은 제거 또는 통합. 새 사이트 설계 시 기능별 존재 이유 검증 필요

### P25. 신규 회원 온보딩이 없다 ✅
- **현재**: 가입 후 자동 DM 하나만 발송. "다음에 뭘 하라"는 안내가 없음. 스터디 참여자는 더 심각 — 결제 후 어디로 가야 하는지 모름
- **근거**: 신연권 — 신규 참여자 빈번한 질문 "어디로 들어가야 하나요?", 다혜 — 홈 화면 혼란
- **심각도**: High (스터디 운영 커뮤니티 특성상 온보딩이 중요)
- **있어야 할 상태**: 가입 후 관심분야 선택 → 맞춤 콘텐츠 추천, 스터디 결제 후 "내 스터디" 안내 등 단계별 온보딩 플로우
