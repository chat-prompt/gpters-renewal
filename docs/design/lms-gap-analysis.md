## 4. 스터디 페이지 구조 불일치 (Critical)

실제 운영 중인 스터디 판매 페이지(`/ai-study-list`, `publish-21.html`)와 프로토타입의 `/study` 페이지 구조가 근본적으로 다름.

### 실제 운영 구조

**기수 단위 원페이지 랜딩** — 하나의 기수(예: 21기)가 하나의 페이지에서 소개·판매됨.

| # | 섹션 | 내용 |
|---|------|------|
| 1 | Hero | 기수명 + 핵심 카피 + CTA ("21기 AI 스터디 신청하기") |
| 2 | Why | 문제/해결 비교 (질문하는 방식 vs 시키는 방식) |
| 3 | Previous Results | 이전 기수 대표 사례 4건 (before→after) |
| 4 | Curriculum | 5개 이벤트 (핵심강의/OT/오프라인/사례발표/베스트발표회) + 주간 캘린더 |
| 5 | Programs | **3개 카테고리, 18개 스터디** (썸네일+레벨+제목+스터디장+요일) |
| 6 | Target Checklist | "이런 고민이 있었다면" 5개 체크 항목 |
| 7 | Pricing | 슈퍼얼리버드 269,000 / 얼리버드 284,000 / 정가 299,000원 + 추가 할인 |
| 8 | Reviews | 수료생 후기 9건 (캐러셀) |
| 9 | Study OT | 설명회 안내 + YouTube 영상 |
| 10 | FAQ | 10개 아코디언 |
| 11 | Footer CTA | 마감일 + 최종 CTA |

**상태별 CTA 전환:**
- **모집 중**: 가격 표시 + "신청하기" CTA
- **휴지기**: 같은 페이지 구조, CTA가 **"대기자 신청"** 형태로 전환

**핵심 사실:**
- 스터디 프로그램 수: 6개가 아니라 **18개** (3개 카테고리)
- 가격: 150,000원이 아니라 **299,000원** (기수 단위 단일 가격)
- 각 프로그램은 레벨 구분 있음: 입문🐥 / 중급
- 추가 할인: 연속참여 10만원, 설명회 참여 1만원

### 프로토타입 현재 구조 (불일치)

- `/study` — 6개 스터디를 **개별 카드 그리드**로 나열 (마치 별도 상품처럼)
- `/study/[slug]` — 개별 스터디 상세 랜딩 (5탭 + Sticky CTA)
- 가격 150,000원으로 하드코딩

### 수정 방향

1. **`/study` → 기수 원페이지 랜딩**으로 전면 교체 (위 12개 섹션 구조)
2. **`/study/[slug]` → 개별 프로그램 상세**로 역할 변경 (기수 랜딩의 프로그램 클릭 시 진입)
3. **가격 구조** — 기수 단위 단일 가격 (299,000원) 반영
4. **상태 전환** — 모집 중 ↔ 휴지기(대기자 모집) CTA 분기

## 5. 요약

**스터디 페이지 구조 불일치 (Critical)**:
- `/study` 페이지가 개별 카드 그리드인데 실제는 **기수 원페이지 랜딩** 구조
- 프로그램 수 6개 → 실제 **18개** (3카테고리), 가격 150,000원 → 실제 **299,000원**
- 휴지기에는 같은 페이지에서 CTA가 **대기자 모집**으로 전환

**문서 → 미구현 (6건)**:
- 리더보드, 찐친챌린지, 베스트발표자, 세션 카드 → 컴포넌트 만들었다가 삭제됨
- 결제 후 온보딩 → 페이지 자체 없음
- GNB "내 스터디" 1클릭 → 드롭다운에만 있고 GNB 메인에 없음

**구현 → 문서 외 (2건)**: 구현 과정에서 디테일 보강한 수준 (문제 아님)

**일치 (14개 페이지)**: PRD에 정의된 핵심 기능은 거의 모두 구현됨
