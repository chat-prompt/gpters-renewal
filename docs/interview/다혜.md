# 다혜 인터뷰

> 역할: 커뮤니티/스터디 운영 (운영진)
> 인터뷰 일시: 2026-02-25

## 역할별 질문

### 1. 하루 루틴

오늘 사이트에서 한 작업들:

- **21기 랜딩페이지로 스터디 일괄 공간옮기기**: 기존 "작성중 스터디" 공간에 있던 스터디들을 어드민에서 일괄로 공간 이동. 어드민에서 한 번에 처리 가능.
- **커뮤니티 이벤트 게시판에 스터디 설명회 게시 및 내용 수정**
- **스터디 CMS 커스텀필드 수정**
- **뽀짝이(봇)로 베터모드 API 활용**: 스터디 상세페이지 자동 생성. 에어테이블 → 뽀짝이 → 베터모드 흐름으로 자동화. 수량이 많고 데이터 연동이 필요해서 자동화가 필수.
- **멤버 프로필페이지 스터디 결제내역 DB 직접 수정**: 데이터 오류 수정 목적. 어드민에서 할 수 없는 작업이라 DB를 직접 수정해야 함. 가끔 발생하는 이슈.

### 2. 스터디 운영 전체 흐름

(상세 단계별 설명은 넘어감 — 핵심 흐름은 하루 루틴에서 파악됨)

- 에어테이블에 스터디 데이터 입력 → 뽀짝이가 베터모드에 상세페이지 자동 생성 → 랜딩페이지 공간으로 이동

### 3. 사이트 밖에서 하는 작업

(구체적인 외부 도구 사용은 별도 논의 예정)

사이트의 핵심 역할로 언급된 것:
- 스터디 랜딩페이지
- 설명회 신청
- 무료 AI 웨비나 신청
- 마이페이지에서 스터디 결제내역 확인
- 마이페이지에서 별도 LMS로 이동 (AI스터디 참여자용)
- 로그인 세션 유지

### 4. 관리자 기능 (어드민)

**주로 쓰는 기능:**
- 게시판 생성/수정
- CMS 설정 (커스텀필드 등)
- 게시글 숨기기/삭제하기

**심각한 불편 사항:**
- **멤버의 게시글 공간옮기기 불가**: 멤버가 자기 게시글의 공간을 변경할 수 없음. 잘못 게시하면 삭제 후 재게시해야 함.
- **웹훅 중복 발동 문제**: 게시글이 올라오면 웹훅 → n8n이 작동하여 요약 자동 생성하는 구조인데, 삭제 후 재게시하면 웹훅이 2번 날아가서 n8n도 2번 작동. **자주 발생하는 문제.**

### 5. 참여자들의 불만/질문

**게시판 불만사항 모음 게시글** (https://www.gpters.org/notice/post/please-leave-your-bulletin-mJkuDaUnd53taRY) 에 24개 댓글로 수집됨.

주요 불만사항:

1. **게시/수정 버튼 위치 문제**: 게시 버튼이 제일 하단, 수정 버튼이 제일 상단에 있어서 긴 글의 경우 스크롤 압박. 우측에 고정 배치 제안.
2. **스터디 종류 오기입**: 스터디 종류가 스크롤하면 안 보여서 다른 스터디로 잘못 기입하는 경우 발생. 우측에 고정 표시 필요.
3. **게시글 프레임 보기 기능 부재**: 상단 바, 좌측 메뉴 없이 글 자체만 볼 수 있는 "새창에서 열기" 기능 요청.
4. **검색 기능 불편**: 원하는 글 찾기 어려움, 정렬 기준(최신순 등) 불명확, 게시판 상단 노출이 추천순이 아님.
5. **북마크(찜) 기능 실종**: mark 기능이 사라져서 찜해놓은 게시물을 찾을 수 없는 문제.
6. **댓글 접힘 불편**: "1개 더 많은 답글 보기" 버튼을 일일이 클릭해야 함. 한 번에 전체 댓글 보기 옵션 필요.
7. **영상 콘텐츠 검색 어려움**: 수백 개 AI캠프 영상에서 원하는 걸 찾기 어려움. 기수별-분야별 콘텐츠 맵/색인 기능 요청.

### 6. 콘텐츠 관리

- 스터디 베스트 사례가 선정되면 CMS의 `isBest` 필드가 `true`로 변경
- `isBest: true`인 게시글만 **'베스트 사례 큐레이션'** 공간에 노출

### 7. DM 문제 (추가 발견)

**심각한 문제:**
- GPTers 계정에서 신규가입 시 자동 DM을 보내는데, 보낸 DM 내역이 다 보여져서 실제 받은 DM이 묻힘
- 누군가 DM을 보내면 완전 누락하는 경우가 많음
- 이메일로 "DM 왔다"는 알림이 와서 클릭해도 해당 메시지로 리디렉션이 안 됨 → 원본을 아예 찾지 못함

(앞에서 언급한 내용과 동일하다고 답변)

3. **검색/정렬/북마크 기능 부재** — 유용한 콘텐츠를 찾고 저장하기 어려움

## 핵심 인사이트 요약

1. **에어테이블 → 뽀짝이 → 베터모드 자동화 파이프라인이 핵심 운영 인프라**: 스터디 상세페이지를 수동으로 만들기엔 수량이 많고, 데이터가 에어테이블에 있어서 자동화가 필수. 사이트 어드민만으로는 운영이 불가능한 구조.

2. **멤버의 게시글 공간옮기기 불가가 웹훅 중복까지 연쇄적으로 문제를 일으킴**: 단순 UI 불편이 아니라, n8n 자동 요약 중복 생성이라는 시스템 문제로 이어짐. 자주 발생.

3. **DM 시스템이 사실상 작동하지 않음**: 자동 DM과 수동 DM이 섞여 누락되고, 이메일 알림 → 사이트 리디렉션도 안 되어 중요한 소통을 놓침.

4. **"AI계의 LinkedIn"이라는 명확한 비전**: 마이페이지를 셀프브랜딩 포트폴리오로 만들고, AI활용사례가 자동 요약되어 프로필에 반영되길 원함. 단순 게시판이 아닌 커리어 플랫폼.

5. **현재 게시판이 "과제 제출용"으로만 기능하고 있음**: 누구나 가볍게 참여할 수 있는 피드형 포스트가 없고, 사례게시글 포맷이 너무 무거움. 진입장벽이 높음.

6. **검색/정렬/북마크 등 콘텐츠 발견 기능이 전반적으로 부족**: 수백 개 영상 콘텐츠 색인 부재, 게시글 검색 불편, 북마크 기능 실종 등. 양질의 콘텐츠가 있지만 찾기 어려움.

7. **iframe 기반 마이페이지 확장이라는 기술적 현실**: 베터모드 한계를 iframe + 서브도메인으로 우회하되, 속도와 SEO 문제를 감수해야 하는 상황. 바이브코딩으로 빠르게 구현하되 구조적 한계 인식.
