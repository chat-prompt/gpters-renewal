# 신연권 인터뷰

> 역할: 커뮤니티/스터디 운영
> 인터뷰 일시: 2026-02-25

## 역할별 질문

### 1. 하루 루틴

서포트 계정으로 로그인해서 작성 중인 AI스터디에 들어가 AI토크가 잘 생성되었는지 확인한다. 그 후 확인된 AI토크를 커뮤니티 이벤트로 옮긴다.

가끔 홈 화면 수정이 필요할 때가 있는데, 이때는 '관리' 탭에 들어가서 수정 작업을 진행한다. 다만 관리 탭의 UI가 복잡해서 원하는 부분을 찾기 위해 여러 번 헤매곤 한다. 구체적인 경로는 관리 → 디자인 스튜디오 → 컬렉션 및 공간 → 컬렉션 없음 → 홈 → Body → 판매주간 홈배너 → Column 2 → 게시물 6 정도 들어가야 한다.

### 2. 스터디 운영 전체 흐름

**스터디장 모집:** 외부 페이지에서 진행한다.

**상세페이지 작성/수정:** 모집이 완료되면 각 스터디장별로 상세페이지 수정 및 확인을 요청한다. 스터디장들은 "작성 중 AI스터디"에서 목록으로 자신의 스터디를 찾을 수 있다. 다만 신규 스터디장들은 사이트 자체가 생소하고 메인 화면을 봤을 때 한눈에 보이는 구조가 아니라서 상세페이지 위치를 찾기 어려워한다.

**최종제출 토글:** 상세페이지 수정이 확인되면 각 게시글마다 최종제출 토글을 on 해달라고 요청한다. 이는 해당 게시글에 들어가 "수정하기" 버튼을 누른 후 여러 내용이 있는 페이지를 스크롤 내려서 "최종제출" 탭을 찾아 on 하는 방식이다. 다만 스크롤을 내릴 때 세로로 나열된 많은 내용이 있어서 한눈에 찾기 쉬운 구조가 아니며, 스터디장이 내용을 확인하는 데 시간이 꽤 걸린다.

**에어테이블 확인:** 최종제출 토글이 on 되면 에어테이블에 자동으로 연동되어 표 형태로 확인할 수 있다.

**랜딩페이지 이동:** 상세페이지까지 최종제출되면 게시물 상단의 점 세 개 아이콘을 클릭해서 "해당 게시판으로 옮겨주기" 옵션으로 스터디 랜딩페이지('OO기 AI스터디 모집' 게시판)로 옮긴다.

**진행 중:** 이후 스터디 멤버들이 주차별로 LMS 페이지에서 스터디를 듣고 사례글을 작성하며 수료 확인을 한다. 사이트는 LMS로 가는 "중간 다리" 역할을 한다.

### 3. 사이트 밖에서 하는 작업

다양한 외부 도구를 활용한다:
- **슬랙**: 팀원들과의 소통
- **에어테이블**: 운영자용 DB 확인
- **n8n**: 자동화 작업
- **클로드코드**: 개발/스크립팅
- **지메일**: 이메일
- **리니어**: 프로젝트 관리/이슈
- **노션**: 정리된 문서
- **구글시트**: 표 형태의 정보
- **구글 캘린더**: 일정 관리
- **Zoom**: 화상회의
- **카카오톡**: 팀 메시지

**이유:** 사이트에서 처리하는 것보다 목적에 맞는 각 도구를 사용하는 것이 효율적이다. 또한 사이트 자체가 불편하게 느껴져서 잘 안 쓰려고 한다.

### 4. 관리자 기능

주로 **게시글 관리(작성/수정)** 작업을 진행한다.

**문제점:**
- 왼쪽 사이드바의 카테고리가 약 30개 정도로 매우 많다
- 자주 쓰는 카테고리는 3~4개 정도 (21기 스터디 모집, 커뮤니티 이벤트, AX LAB)
- 카테고리의 분류 기준이 모호하다. 어떤 것은 분야별(예: 콘텐츠/마케팅)이고, 어떤 것은 도구별(예: 클로드, 코파일럿, 제미나이)이어서 찾기 어렵다
- 필요한 카테고리를 찾을 때 검색으로 찾는 경향이 있다

**임시저장 기능 부재:** 게시글 작성 중 페이지에서 나가면 내용이 날아간다. 이 때문에 외부 페이지에서 작성을 완료한 후 복사 붙여넣기 방식으로 사이트에 올린다.

### 5. 참여자들의 불만/질문

**기술적 문제:**
- 임시저장 기능이 없어서 작성 중인 내용이 날아질 수 있다
- 가끔씩 서버가 터지는 일이 있다 (어쩔 수 없는 일이라고 생각함)

**네비게이션 관련:**
신규 참여자들이 자주 묻는 질문: "스터디를 들으려면 어디로 들어가야 하나요?"
- 프로필 클릭 → 내 AI스터디 클릭 → 로그인 → LMS 페이지에서 확인
- 이 과정이 복잡하고 직관적이지 않아 신규 참여자들이 자주 문의한다

### 6. 신규 회원 적응 과정

신규 회원이 홈 화면을 봤을 때 "뭐가 이렇게 많지?"라는 반응을 한다. 이후 새로 올라온 포스트를 쭉 둘러보다가 마음에 드는 것이 있으면 가입할 정도로 적응된다. 다만 **스터디 모집이 진행되고 있는지는 잘 안 보일 수 있을 것 같다**는 우려가 있다.

## 공통 질문

### 카테고리/메뉴 구조

앞의 역할별 질문에서 충분히 설명했음. 약 30개의 카테고리가 있고, 분류 기준이 일관성 없어서 찾기 어렵다.

### 잘 안 쓰는 기능/페이지

쓰는 페이지보다 안 쓰는 페이지가 훨씬 더 많아서 설명하기 어렵다. 또한 페이지의 히스토리를 잘 모르기 때문에 정확한 답변이 어렵다.

### 모바일

모바일에서 크게 불편한 점은 못 느꼈다.

### 가장 불편한 점 3가지

1. **사용자 친화적이지 못한 UI**: 게시글 수정 시 임시저장이 없고, 스크롤이 길어서 한눈에 찾기 어렵다. 관리 탭도 마찬가지로 깊이가 깊고 구조가 복잡하다.
2. **가끔씩 터지는 서버**: 예측 불가능한 기술적 문제
3. **중구난방인 카테고리 분류**: 분야별, 도구별 기준이 섞여 있어서 찾기 어렵다.

## 핵심 인사이트 요약

1. **깊고 복잡한 네비게이션 구조**: 관리자 기능도, 사용자 기능도 필요한 페이지까지 도달하는 경로가 너무 길고 복잡해서 여러 번 헤메는 경험이 발생한다. 특히 신규 스터디장이나 신규 참여자들이 큰 어려움을 겪는다.

2. **카테고리 분류의 일관성 부재**: 약 30개의 카테고리가 분야별, 도구별 기준이 혼합되어 있어서 사용자가 검색에 의존하게 된다. 3~4개의 자주 쓰는 카테고리만 쉽게 접근 가능하고, 나머지는 찾기 어렵다.

3. **임시저장 기능의 부재로 인한 프로세스 변화**: 사용자가 외부 도구(노션, 구글시트 등)에서 먼저 작성한 후 복사 붙여넣기하는 우회 프로세스를 사용하게 된다. 이는 사이트 자체의 UX 문제가 외부 도구 의존도를 증가시키는 증거다.

4. **신규 사용자의 높은 진입장벽**: 신규 참여자가 스터디를 시작하려면 프로필 → 내 AI스터디 → 로그인 → LMS 이동이라는 4단계를 거쳐야 하고, 신규 스터디장은 상세페이지 위치를 찾기 어려워한다. 이는 사이트 구조가 신규 사용자를 위해 최적화되지 않았음을 의미한다.

5. **운영자의 외부 도구 높은 의존도**: 슬랙, 에어테이블, 노션, 구글시트, 리니어 등 11개의 외부 도구를 활용한다. 사이트보다 각 도구가 더 효율적이고 직관적이라는 판단이 반영된 결과다.

6. **임시 기능 vs 완성 기능의 격차**: 스터디 관리 페이지(상세페이지, AI토크 확인 등)는 비교적 작동 잘되지만, 홈 화면 수정 같은 관리 기능은 매우 복잡하다. 기능별 UX 수준의 편차가 크다.

7. **사이트의 "중간 다리" 역할화**: 스터디 진행의 실제 학습은 LMS에서 이루어지고, 사이트는 LMS로 가는 진입점 역할로 평가되고 있다. 이는 사이트가 핵심 가치를 전달하지 못하고 있음을 시사한다.
