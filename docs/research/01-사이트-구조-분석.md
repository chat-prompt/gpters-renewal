# GPTers 현재 사이트 구조 분석

> 분석 일시: 2026-02-25
> 대상: gpters.org (Bettermode 기반)
> 분석 방법: WebFetch 크롤링 + 인터뷰 데이터 교차 검증
> 데이터 소스: 신연권(대표), 소연(디자이너), 재호(마케터), 다혜(커뮤니티/스터디 운영) 인터뷰 4건 + 유저 불만 수집 게시글(24건)

## 2. 네비게이션 구조 분석

### 2.1 글로벌 헤더 (4개 항목)
| 메뉴 | 경로 | 역할 | 비고 |
|------|------|------|------|
| 커뮤니티 | /home | 메인 홈 | 기본 랜딩 |
| AI스터디 | /ai-study-list | 스터디 목록 | 핵심 전환 페이지 |
| 스터디장 지원 | 외부 링크 | 스터디장 모집 | 새 탭, 외부 폼 |
| 기업교육 | 외부 링크 | B2B 문의 | 새 탭, 외부 페이지 |

### 2.2 사이드바 빠른 링크 (4개 항목)
| 메뉴 | 역할 |
|------|------|
| 홈 | 메인 홈 복귀 |
| 커뮤니티 이벤트 | AI토크, 웨비나 등 |
| GPTers 사용가이드 | 신규 사용자 온보딩 |
| 공지사항 | 운영 공지 |

### 2.3 사이드바 컬렉션 (8개+ 그룹)
인터뷰에서 언급된 "약 30개 카테고리"는 이 컬렉션의 하위 스페이스를 모두 합산한 수치.

| 컬렉션 | 추정 하위 스페이스 | 분류 기준 |
|--------|-------------------|-----------|
| 지피터스 꿀팁 | 도구별 팁 (클로드, 코파일럿, 제미나이 등) | **도구별** |
| 서로 도와요 | 질문/답변 | **행동별** |
| 스터디 | 기수별 모집, 진행 중 | **시간별** |
| 활용 사례 | 바이브코딩, 콘텐츠/마케팅 등 | **분야별/도구별 혼재** |
| AI 캠프 | 캠프 프로그램 | **프로그램별** |
| 아카이브 | 과거 콘텐츠 | **시간별** |
| 테스트 | 내부용 | N/A |
| 주제별 스터디 | 주제별 분류 | **주제별** |
| 가입인사 | 신규 회원 자기소개 | **의례적** (실질 가치 의문) |

**핵심 문제**:
- 분류 기준이 도구별, 분야별, 시간별, 행동별로 혼재되어 있어 사용자가 원하는 콘텐츠를 찾기 어려움
- **모든 메뉴 앞 이모지가 오히려 혼란을 가중** (재호: 이모지가 정보 전달을 방해)
- 가입인사 게시판은 형식적 절차일 뿐 실질적 활용 가치 의문 (재호)
- 카테고리/스페이스가 너무 많아 통합 필요 (신연권, 다혜 공통 지적)

## 3. 주요 페이지별 역할과 콘텐츠 유형

### 3.1 홈 페이지 (/)
- **역할**: 랜딩, 콘텐츠 발견, 프로모션
- **콘텐츠**: 띠배너(프로모션), 무료 웨비나, 최신 포스트, 인기 게시글, 베스트 사례
- **문제점**: 정보 과밀 → 신규 회원 "뭐가 이렇게 많지?" 반응 (신연권 인터뷰)

### 3.2 AI스터디 목록 (/ai-study-list)
- **역할**: 스터디 탐색 및 구매 전환
- **콘텐츠**: 기수별 스터디 카드, 가격, 모집 일정
- **문제점**: 상품 상세 → 구매 전환 병목 (소연 인터뷰)

### 3.3 커뮤니티 이벤트 (/community-event)
- **역할**: AI토크, 웨비나 등 이벤트 안내
- **콘텐츠**: 이벤트 목록, 참여 안내

### 3.4 개별 게시글 (/post/[slug])
- **역할**: SEO 유입의 핵심 페이지, 콘텐츠 소비
- **콘텐츠**: 글 본문, 댓글, 반응
- **문제점**:
  - 가독성 부족 (소연 인터뷰)
  - 회원가입 유도 부족
  - **게시판이 "과제 제출용"으로만 기능** — 스터디 참여자가 과제를 제출하는 용도로만 쓰이고, 누구나 가볍게 참여할 수 있는 피드형 포스트가 없음. 진입장벽이 높아 커뮤니티 활성화 저해 (다혜)
  - 게시/수정 버튼 위치 불일치 (게시 하단, 수정 상단 — 긴 글 작성 시 스크롤 압박)
  - 댓글 접힘 불편 ("1개 더 많은 답글 보기" 일일이 클릭 필요)
  - 게시글 프레임 보기 기능 부재 (상단바/좌측메뉴 없이 글만 보기 불가)

### 3.5 DM/메시징 시스템
- **역할**: 회원 간 개인 메시지, 운영자 공지
- **현황**: **완전 미작동 상태**
  - 자동 DM (신규 회원 환영 메시지)이 수신함을 점령 → 실제 중요한 DM이 묻힘
  - 이메일 알림("DM이 도착했습니다") 클릭 시 해당 메시지로 리다이렉트되지 않음
  - 운영 목적의 중요 메시지가 완전히 유실되는 상황 발생 (다혜)

### 3.6 사용자 프로필
- **역할**: 개인 활동 내역, LMS 접근
- **문제점**: 활동을 포트폴리오로 보여주기 어려움 (방향성.md)

### 3.7 관리 탭 (관리자 전용)
- **역할**: 사이트 설정, 콘텐츠 관리
- **문제점**:
  - 홈 수정에 7단계 이상 필요 (신연권 인터뷰)
  - **Admin 스페이스-UI 매핑 불명확** — 어떤 스페이스를 수정해야 원하는 프론트엔드 UI가 바뀌는지 찾기 어려움 (재호)
  - 각 섹션 타이틀/문구 수정에 제약이 많음 (재호)
- **CMS 커스텀 필드**: `isBest` 필드로 베스트 사례 큐레이션 관리 (다혜)

### 3.8 검색/탐색 기능
- **현황**: 기본 검색만 제공
- **문제점**:
  - 원하는 글 찾기 어려움, 정렬 기준 불명확
  - 북마크(찜) 기능 실종 (mark 기능이 사라짐)
  - 수백 개 AI캠프 영상 콘텐츠 검색/색인 부재
  - 스터디 종류 오기입 (스크롤하면 선택한 스터디가 안 보여서 다른 스터디로 잘못 기입)
  - (이상 유저 불만 24건 수집 게시글 기반, 다혜 취합)

## 4. 유저 플로우 분석

### 4.1 신규 방문자 → 회원가입 플로우
```
구글 검색 (AI 관련 키워드)
  → 게시글 페이지 유입 (SEO)
    → 글 읽기 (비회원 열람 가능)
      → [이탈 포인트 1] 회원가입 유도 부족
        → 가입 (이메일/소셜 로그인)
          → 홈 화면 도착
            → "뭐가 이렇게 많지?" (정보 과부하)
              → [이탈 포인트] 탐색 포기 또는 무작위 탐색
```

### 4.2 회원 → 스터디 구매 플로우
```
뉴스레터 / 홈 화면 배너
  → AI스터디 목록 페이지
    → 스터디 상세 페이지
      → [이탈 포인트 2] 상세 → 구매 전환 병목
        → 결제
```

### 4.3 스터디 참여자 → 학습 플로우
```
로그인
  → 프로필 클릭
    → 내 AI스터디 클릭
      → LMS 페이지 이동 (외부)
        → 주차별 학습, 사례글 작성
```
**문제점**: 4단계 경로, 신규 참여자가 자주 "스터디 어디서 듣나요?" 질문 (신연권 인터뷰)

### 4.4 운영자 → 홈 화면 수정 플로우
```
관리 탭
  → 디자인 스튜디오
    → 컬렉션 및 공간
      → 컬렉션 없음
        → 홈
          → Body
            → 판매주간 홈배너
              → Column 2
                → 게시물 6
```
**문제점**: 7단계 이상 딥 네비게이션, UI 복잡 (신연권 인터뷰)

### 4.5 운영자 → 스터디 관리 플로우
```
스터디장 모집 (외부)
  → 상세페이지 작성 요청
    → 작성 중 AI스터디에서 목록 확인
      → 상세페이지 수정
        → 최종제출 토글 ON (스크롤 내려 찾기)
          → 에어테이블 자동 연동
            → 게시물 옮기기 (점 세 개 → 해당 게시판으로)
              → 랜딩페이지 이동 완료
```
**문제점**: 다단계 프로세스, 신규 스터디장 혼란

### 4.6 마케터 → 콘텐츠 발행 플로우
```
외부에서 콘텐츠 작성 (임시저장 없으므로)
  → 로그인
    → 적절한 스페이스 선택 (분류 고민)
      → 복사 붙여넣기
        → 게시
          → SEO 노출 대기
```
**문제점**: 임시저장 부재, 스페이스 분류 혼란, 가독성 한계

### 4.7 운영자 → 스터디 상세페이지 자동화 플로우 (핵심 운영 인프라)
```
운영자가 에어테이블에 스터디 정보 입력
  → 뽀짝이(자동화 봇)가 데이터 감지
    → Bettermode API 호출
      → 스터디 상세페이지 자동 생성/업데이트
        → 사이트에 반영
```
**핵심**: Airtable → 뽀짝이(봇) → Bettermode 자동화 파이프라인이 전체 운영의 근간. 이 파이프라인 없이는 스터디 운영 자체가 불가능 (다혜). 리뉴얼 시 이 자동화 흐름을 반드시 대체/보전해야 함.

### 4.8 DM/메시징 플로우 (미작동)
```
[자동 DM 경로]
신규 회원 가입
  → 시스템이 자동 환영 DM 발송
    → 수신함에 자동 DM 대량 적재
      → 실제 중요 DM이 묻힘

[개인 DM 경로]
운영자/멤버가 DM 발송
  → 수신자에게 이메일 알림 도착 ("DM이 도착했습니다")
    → 이메일 내 링크 클릭
      → [장애] 해당 메시지로 리다이렉트 안 됨
        → 메시지 찾기 불가 → 유실
```
**문제점**: 자동 DM 범람 + 리다이렉트 미작동 → DM 채널 자체가 신뢰 불가능. 운영상 중요한 커뮤니케이션 수단이 완전 상실 (다혜)

### 4.9 마케터 → 상품/쿠폰 관리 플로우
```
상품 생성 필요
  → Retool (외부 도구) 접속
    → 상품 정보 입력
      → 쿠폰 생성/연결
        → DB 반영
          → 사이트에서 상품/쿠폰 활성화
```
**문제점**: 상품-쿠폰 관리가 Bettermode 내부가 아닌 Retool이라는 별도 외부 도구에 의존. 관리 포인트 분산, 학습 비용 증가 (재호)

## 5. 기술 플랫폼 특성 (Bettermode + 외부 도구 생태계)

### 5.1 Bettermode 기본 정보

| 항목 | 현황 |
|------|------|
| 플랫폼 | Bettermode (구 Tribe) |
| 인증 | SSO + Firebase 연동 |
| 테마 | 다크/라이트 모드 지원 |
| 브랜드 컬러 | 오렌지 (#EB5A10) |
| 반응형 | 모바일 지원 (다만 친화적이지 않음) |
| 멤버십 | Open (공개 가입) |
| 커스텀 도메인 | www.gpters.org |
| CMS 커스텀 필드 | isBest (베스트 사례 큐레이션) 등 |

### 5.2 연동 외부 도구

| 도구 | 용도 | 의존도 | 비고 |
|------|------|--------|------|
| **Airtable** | 스터디 정보 관리, 자동화 트리거 | 핵심 | 뽀짝이 봇의 데이터 소스 |
| **뽀짝이 (자동화 봇)** | Airtable → Bettermode 자동화 | 핵심 | 스터디 상세페이지 자동 생성 |
| **n8n** | 웹훅 기반 자동화 워크플로우 | 높음 | 게시글 이동 시 웹훅 이중 발화 문제 있음 |
| **Retool** | 상품 생성, 쿠폰 연결 | 중간 | 마케터 필수 도구, 별도 학습 필요 |
| **스티비** | 뉴스레터 발송 | 중간 | 마케팅 채널 |
| **구글시트** | UTM 파라미터 관리 | 낮음 | 마케팅 분석 |
| **메타 광고관리자** | 페이스북/인스타 광고 | 낮음 | 외부 마케팅 |
| **카카오 모먼트** | 카카오 광고 | 낮음 | 외부 마케팅 |

### 5.3 알려진 기술적 문제

| 문제 | 설명 | 심각도 |
|------|------|--------|
| 웹훅 이중 발화 | 멤버 게시글 공간옮기기 불가 → 삭제 후 재게시 → 웹훅 2회 발화 → n8n 이중 실행 | 높음 |
| DM 리다이렉트 미작동 | 이메일 알림의 DM 링크가 실제 메시지로 연결되지 않음 | 높음 |
| 자동 DM 범람 | 신규 회원 환영 DM이 수신함을 점령 | 중간 |
| 스페이스-UI 매핑 불투명 | Admin에서 스페이스 수정 시 프론트 어디가 바뀌는지 파악 어려움 | 중간 |

## 6. 핵심 구조적 문제 요약

| # | 문제 | 심각도 | 영향 | 근거 |
|---|------|--------|------|------|
| 1 | **DM 시스템 완전 미작동** — 자동 DM 범람 + 리다이렉트 미작동으로 메시징 채널 전체 무력화 | Critical | 운영 커뮤니케이션 수단 상실, 중요 메시지 유실 | 다혜: 실제 DM이 묻히고, 이메일 링크도 작동 안 함 |
| 2 | **멤버 게시글 공간옮기기 불가 → 웹훅 이중 연쇄 문제** — 삭제+재게시해야 하고, 이로 인해 n8n 자동화가 이중 실행됨 | Critical | 데이터 무결성 훼손, 운영 리스크 | 다혜: 빈번하게 발생하는 연쇄 장애 |
| 3 | 카테고리 분류 기준 혼재 (도구/분야/시간) + 스페이스 과다 | High | 콘텐츠 발견성 저하, 사용자 혼란 | 신연권: "중구난방인 분류", 다혜: 통합 필요 |
| 4 | 홈 화면 정보 과밀 | High | 신규 사용자 이탈 | 신연권: "뭐가 이렇게 많지?" |
| 5 | **검색/정렬/북마크 기능 부족** — 검색 결과 불명확, 정렬 기준 모호, 북마크 기능 실종, 영상 콘텐츠 색인 부재 | High | 원하는 콘텐츠를 찾을 수 없음 | 유저 불만 24건 수집 게시글 (다혜 취합) |
| 6 | **게시글이 "과제 제출용"으로만 작동** — 가벼운 참여(피드형 포스트) 불가, 진입장벽 높음 | High | 커뮤니티 활성화 저해, 비스터디 회원 참여 낮음 | 다혜: 과제 외 게시 동기 없음 |
| 7 | 깊은 네비게이션 (관리 7단계+) | High | 운영 효율 저하 | 신연권: 관리 탭 복잡 |
| 8 | **Admin 스페이스-UI 매핑 불명확** — 어떤 스페이스가 어떤 UI에 연결되는지 파악 불가 | High | 운영자 혼란, 수정 시행착오 반복 | 재호: 원하는 UI 찾기 어려움 |
| 9 | **상품/쿠폰 관리 외부 도구(Retool) 의존** | Medium | 관리 포인트 분산, 학습 비용 | 재호: Retool 별도 접속 필요 |
| 10 | 스터디 접근 경로 불명확 | Medium | 참여자 혼란, 문의 증가 | 신연권: "어디서 듣나요?" 빈번 |
| 11 | 게시글 가독성 부족 | Medium | 콘텐츠 효과 저하, SEO 전환 병목 | 소연: 타 사이트 대비 부족 |
| 12 | **이모지 과다로 메뉴 혼란** | Medium | 정보 전달 방해, 전문성 저하 | 재호: 모든 메뉴 앞 이모지가 오히려 혼란 |
| 13 | 회원가입 유도 장치 부족 | Medium | 유입→가입 전환 병목 | 소연: 전환이 더딤 |
| 14 | 임시저장 기능 부재 | Medium | 외부 도구 의존 강화 | 신연권: 외부에서 작성 후 복붙 |
| 15 | 포트폴리오/이력 기능 부재 | Medium | 활동의 가시적 가치 부족 | 방향성.md |
| 16 | 디자인 일관성 부족 | Low | 브랜딩 약화 | 소연: 브랜딩 필요 |
| 17 | 광고/프로모션 지면 비효율 | Low | 마케팅 효과 감소 | 소연: 배너 활용 부족 |
