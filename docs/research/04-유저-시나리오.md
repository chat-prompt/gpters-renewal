## 페르소나 1: AI 입문자 (신규 방문자)

### 프로필

| 항목 | 내용 |
|------|------|
| 이름 | 김지민 |
| 나이 | 32세 |
| 직업 | 중소기업 마케팅 담당자 |
| AI 숙련도 | 초급 (ChatGPT 가끔 사용) |
| 동기 | 업무에 AI를 활용하고 싶지만 어디서부터 시작할지 모름. "마케팅 AI 활용"을 검색하다가 GPTers 게시글 발견 |

**핵심 목표:** 나에게 맞는 AI 활용법을 배우고, 체계적으로 학습할 수 있는 커뮤니티를 찾고 싶다.

### As-Is 시나리오: 현재 사이트에서의 경험

1. **유입**: 구글에서 "마케팅 AI 활용 사례"를 검색하고, GPTers 게시글 링크를 클릭한다.
2. **첫인상**: 게시글 페이지에 도착하지만, 가독성이 좋지 않아 글을 끝까지 읽기 어렵다. 텍스트 레이아웃과 타이포그래피가 다른 블로그에 비해 부족하게 느껴진다.
3. **탐색 시도**: 글이 유용해서 관련 콘텐츠를 더 보고 싶지만, 홈 화면으로 이동하면 "뭐가 이렇게 많지?"라는 느낌을 받는다. 약 30개의 카테고리가 분야별, 도구별로 섞여 있어 어디를 봐야 할지 모른다.
4. **콘텐츠 발견 실패**: 자신의 수준과 관심사에 맞는 콘텐츠나 행사, 이벤트를 발견하기 어렵다. 최신 포스트를 스크롤하며 둘러보지만, 체계적인 큐레이션이 없어 원하는 정보를 찾지 못한다.
5. **스터디 인지 실패**: 스터디 모집이 진행 중인지 잘 보이지 않는다. 배너나 광고 지면이 눈에 띄지 않아 핵심 상품을 인지하지 못한 채 이탈한다.
6. **회원가입 망설임**: 게시글에서 회원가입으로 이어지는 유도 장치가 부족하다. 가입할 명확한 이유를 느끼지 못하고 브라우저 탭을 닫는다.

**Pain Points:**
- 게시글 가독성이 낮아 콘텐츠의 가치를 충분히 느끼지 못함
- 홈 화면의 정보 과잉으로 무엇을 봐야 할지 혼란
- 자신에게 맞는 콘텐츠/스터디를 발견할 수 있는 경로가 없음
- 회원가입 후 얻을 수 있는 가치가 명확하지 않음

### To-Be 시나리오: 리뉴얼 사이트에서의 이상적 경험

1. **유입**: 구글에서 "마케팅 AI 활용 사례"를 검색하고, SEO 최적화된 GPTers 게시글을 클릭한다.
2. **첫인상**: 깔끔한 타이포그래피와 여백으로 가독성이 높은 게시글 페이지에 도착한다. 목차, 이미지, 코드 블록 등이 잘 정리되어 있어 끝까지 읽게 된다.
3. **관련 콘텐츠 발견**: 게시글 하단에 "이 글을 읽은 분들이 함께 본 글", "마케팅 AI 활용 시리즈" 같은 관련 콘텐츠가 추천된다. 자연스럽게 2-3개의 글을 더 읽는다.
4. **스터디 인지**: 게시글 사이드바 또는 인라인 배너에서 "마케팅 AI 활용 스터디 21기 모집 중"이라는 CTA를 발견한다. 스터디 상세 페이지로 이동해 커리큘럼과 후기를 확인한다.
5. **회원가입**: 게시글 읽기 중 자연스러운 위치에 "무료 가입하고 더 많은 AI 활용 사례 받아보기" CTA가 있다. 가입하면 뉴스레터 구독과 커뮤니티 채팅방 참여가 자동으로 안내된다.
6. **온보딩**: 가입 직후 간단한 관심사 선택(마케팅, 개발, 디자인 등)을 거치면, 맞춤형 콘텐츠 피드와 추천 스터디가 표시된다.

**핵심 개선:**
- 게시글 가독성 대폭 향상 (SEO + 사용자 경험)
- 관련 콘텐츠 자동 추천으로 탐색 지속
- 스터디 모집을 자연스럽게 인지시키는 CTA 배치
- 게시글 → 회원가입 전환 경로 명확화

### 핵심 니즈와 기대

| 니즈 | 기대 |
|------|------|
| 내 수준에 맞는 콘텐츠 발견 | 관심사 기반 콘텐츠 큐레이션 |
| 읽기 편한 게시글 | 깔끔한 레이아웃, 좋은 가독성 |
| 학습 경로 안내 | "다음에 이것을 읽어보세요" 같은 가이드 |
| 커뮤니티 소속감 | 가입 후 즉시 참여할 수 있는 채널 |
| 스터디 정보 접근 | 모집 중인 스터디를 쉽게 찾고 비교 |

## 페르소나 2: 스터디 참여자

### 프로필

| 항목 | 내용 |
|------|------|
| 이름 | 이수현 |
| 나이 | 28세 |
| 직업 | 스타트업 프로덕트 디자이너 |
| AI 숙련도 | 중급 (ChatGPT, Midjourney 업무 활용 중) |
| 동기 | GPTers 20기 스터디에 참여했고, 21기도 신청하려 함. AI 스터디를 통해 포트폴리오를 쌓고 싶음 |

**핵심 목표:** 스터디를 원활히 진행하고, 사례글을 작성하며, 수료 이력을 포트폴리오로 보여주고 싶다.

### As-Is 시나리오: 현재 사이트에서의 경험

1. **스터디 접근**: 스터디를 들으려면 프로필 클릭 → 내 AI스터디 클릭 → 로그인 → LMS 페이지 이동이라는 4단계를 거쳐야 한다. 이 경로가 직관적이지 않아 "스터디를 들으려면 어디로 들어가야 하나요?"라는 질문을 자주 한다.
2. **사례글 작성**: 주차별 학습 후 사례글을 작성하는데, 임시저장 기능이 없어서 작성 중 페이지에서 나가면 내용이 날아간다. 결국 노션이나 외부 에디터에서 먼저 작성한 후 복사 붙여넣기로 올린다.
3. **게시판 찾기**: 사례글을 올릴 게시판을 찾기 어렵다. 약 30개의 카테고리가 분야별, 도구별로 혼재되어 있어 매번 고민한다.
4. **수료 확인**: LMS에서 수료 확인을 하는데, 사이트는 LMS로 가는 "중간 다리" 역할만 한다. 사이트 자체에서 학습 진행 상황을 확인할 수 없다.
5. **포트폴리오**: 스터디 참여 이력과 작성한 사례글을 한눈에 보여줄 수 있는 페이지가 없다. 지피터스에서의 활동을 경력이나 포트폴리오로 보여주기 어렵다.
6. **커뮤니티 참여**: 다른 참여자들의 사례글을 발견하기 어렵고, 같은 스터디 멤버와의 교류가 사이트 내에서 잘 이루어지지 않는다.

**Pain Points:**
- 스터디 진입까지 4단계의 복잡한 경로
- 임시저장 기능 부재로 외부 도구 의존
- 카테고리 혼란으로 글 올릴 곳 찾기 어려움
- 활동 이력이 포트폴리오화되지 않음
- 사이트가 LMS의 중간 다리에 머무름

### To-Be 시나리오: 리뉴얼 사이트에서의 이상적 경험

1. **스터디 접근**: 로그인 후 대시보드에 "나의 진행 중 스터디" 카드가 바로 보인다. 클릭 한 번으로 LMS 또는 이번 주차 과제 페이지로 이동한다.
2. **사례글 작성**: 사이트 내 에디터에서 바로 사례글을 작성한다. 자동 임시저장이 되어 중간에 나가도 안전하다. 스터디별로 작성 양식이 미리 세팅되어 있어, 어디에 올려야 할지 고민할 필요가 없다.
3. **학습 진행 현황**: 사이트 내에서 주차별 학습 진행률, 남은 과제, 수료 조건 충족 여부를 한눈에 확인할 수 있다.
4. **AI 이력서/포트폴리오**: 프로필 페이지에 "나의 AI 이력서"가 있다. 참여한 스터디, 작성한 사례글, 수료 뱃지, 발표 이력 등이 자동으로 정리되어 있다. 이 페이지의 공유 링크를 이력서나 LinkedIn에 첨부할 수 있다.
5. **커뮤니티 교류**: 같은 스터디 멤버의 사례글을 모아볼 수 있고, 댓글과 좋아요로 피드백을 주고받는다. 스터디별 채팅방에서 실시간 교류가 이루어진다.
6. **수료 후**: 수료 시 디지털 수료증이 발급되고, AI 이력서에 자동 반영된다. 다음 기수 스터디 추천도 함께 안내된다.

**핵심 개선:**
- 원클릭 스터디 접근 (대시보드)
- 자동 임시저장 에디터
- 스터디별 사례글 자동 분류
- AI 이력서 페이지 (포트폴리오화)
- 학습 진행 현황 대시보드

### 핵심 니즈와 기대

| 니즈 | 기대 |
|------|------|
| 스터디 빠른 접근 | 로그인 후 1클릭으로 스터디 진입 |
| 안전한 글쓰기 | 자동 임시저장, 스터디별 작성 양식 |
| 활동 포트폴리오화 | AI 이력서 페이지, 공유 가능한 링크 |
| 학습 진행 추적 | 주차별 진행률, 수료 조건 한눈에 |
| 동료와의 교류 | 스터디 멤버 사례글 모아보기, 피드백 |

## 페르소나 3: 커뮤니티 운영진

### 프로필

| 항목 | 내용 |
|------|------|
| 이름 | 신연권 / 다혜 (실제 운영진) |
| 나이 | 30대 |
| 직업 | GPTers 커뮤니티/스터디 운영 |
| AI 숙련도 | 고급 (n8n 자동화, 클로드코드 등 적극 활용) |
| 동기 | 스터디 모집-진행-수료 전 과정을 관리하고, 커뮤니티 콘텐츠와 홈 화면을 효율적으로 운영하고 싶음 |

**핵심 목표:** 복잡한 관리 프로세스를 줄이고, 효율적으로 커뮤니티를 운영하고 싶다.

### As-Is 시나리오: 현재 사이트에서의 경험

**일일 운영 (신연권):**

1. **AI토크 확인**: 서포트 계정으로 로그인 → "작성 중인 AI스터디"에서 AI토크 생성 확인 → 확인된 AI토크를 "커뮤니티 이벤트"로 수동 이동. 매일 반복되는 작업이다.
2. **홈 화면 수정**: 관리 탭 → 디자인 스튜디오 → 컬렉션 및 공간 → 컬렉션 없음 → 홈 → Body → 판매주간 홈배너 → Column 2 → 게시물 6까지 약 8단계 깊이의 네비게이션을 거쳐야 한다. UI가 복잡해서 원하는 부분을 찾기 위해 여러 번 헤맨다.

**스터디 운영 전체 흐름 (신연권):**

3. **스터디장 모집**: 외부 페이지에서 별도로 진행한다. 사이트에서 처리할 수 없다.
4. **상세페이지 수정 요청**: 스터디장에게 "작성 중 AI스터디"에서 상세페이지를 수정해달라고 요청하지만, 신규 스터디장은 사이트가 생소하고 메인 화면에서 한눈에 보이는 구조가 아니라 상세페이지 위치를 찾기 어려워한다.
5. **최종제출 토글**: 게시글 → 수정하기 → 긴 페이지를 스크롤 내려서 "최종제출" 탭을 찾아 토글을 on으로 변경해야 한다. 세로로 나열된 많은 내용 때문에 한눈에 찾기 어렵다. 스터디장 한 명 한 명에게 이 과정을 안내해야 한다.
6. **랜딩페이지 이동**: 게시물 상단 점 세 개 아이콘 → "해당 게시판으로 옮겨주기"로 스터디 랜딩페이지로 수동 이동한다.
7. **게시글 관리**: 왼쪽 사이드바에 약 30개의 카테고리가 있으나, 자주 쓰는 것은 3-4개(21기 스터디 모집, 커뮤니티 이벤트, AX LAB). 분류 기준이 분야별, 도구별로 혼재되어 검색에 의존한다.

**스터디 페이지 자동화 및 DB 관리 (다혜):**

8. **스터디 페이지 생성 파이프라인**: 에어테이블에서 스터디 정보를 정리하고, 뽀짝이(봇)를 통해 베터모드로 전달하는 자동화 파이프라인을 사용한다. 에어테이블 → 뽀짝이 → 베터모드라는 외부 도구 체인에 의존하고 있다.
9. **스터디 공간 일괄 이동**: "작성중 스터디"에 있는 스터디 페이지들을 21기 랜딩페이지로 일괄 이동하는 작업을 어드민에서 수행한다.
10. **CMS 커스텀 필드 편집**: CMS의 커스텀 필드를 직접 수정해야 하는 경우가 빈번하다.
11. **멤버 결제 DB 직접 수정**: 멤버 프로필 페이지의 스터디 결제 DB를 직접 수정해야 하는데, 어드민에서는 이 작업이 불가능하기 때문에 DB에 직접 접근하여 수정한다.
12. **스터디 설명회 등록**: 스터디 설명회 일정을 커뮤니티 이벤트 게시판에 수동으로 포스팅한다.

**DM/메시징 시스템 장애 (다혜):**

13. **DM 시스템 미작동**: 자동 DM이 대량으로 발송되면서 받은편지함이 자동 메시지로 넘쳐나고, 실제 중요한 DM이 묻혀서 확인할 수 없다. 이메일 리다이렉트 기능도 제대로 동작하지 않아, 중요한 소통이 누락되는 상황이 반복된다.

**웹훅 중복 연쇄 문제 (다혜):**

14. **게시글 공간이동 불가 → 웹훅 중복**: 멤버가 잘못된 스페이스에 글을 올렸을 때, 멤버 본인은 게시글을 다른 공간으로 이동할 수 없다. 결국 해당 글을 삭제하고 올바른 스페이스에 다시 작성하게 되는데, 이 과정에서 웹훅이 두 번 발동되고, n8n 자동 요약이 중복 실행되는 연쇄 문제가 발생한다.

**유저 민원 관리 (다혜):**

15. **24건의 유저 민원**: 다혜가 수집한 유저 민원이 24건에 달한다. 사이트의 다양한 불편 사항에 대한 유저 불만이 지속적으로 축적되고 있다.

**외부 도구 의존:**

16. 슬랙, 에어테이블, n8n, 클로드코드, 지메일, 리니어, 노션, 구글시트, 구글 캘린더, Zoom, 카카오톡, 뽀짝이(봇), 베터모드 어드민 등 13개 이상의 외부 도구를 사용한다. 사이트보다 각 도구가 더 효율적이라고 판단하기 때문이다.

**Pain Points:**
- 홈 화면 수정에 8단계 네비게이션 필요
- 스터디 운영 흐름이 수동 작업의 연속 (토글, 게시판 이동 등)
- 임시저장 기능 부재로 외부 에디터에서 작성 후 복사 붙여넣기
- 30개 카테고리의 비일관적 분류로 검색 의존
- 신규 스터디장에 대한 반복적인 사용법 안내 필요
- 사이트 자체가 불편해서 안 쓰려고 하게 됨
- DM 시스템 완전 미작동으로 중요 소통 누락 (자동 DM 범람 → 실제 DM 매몰 → 이메일 리다이렉트 미작동)
- 멤버 게시글 공간이동 불가 → 삭제 후 재작성 → 웹훅 중복 발동 → n8n 자동화 이중 실행의 연쇄 문제
- DB 직접 수정 필요 (어드민으로 불가한 작업 존재 — 멤버 결제 정보 등)
- 자동화 파이프라인의 외부 도구 의존 (에어테이블 → 뽀짝이 → 베터모드)
- 24건의 유저 민원 축적 — 운영 부담 가중

### To-Be 시나리오: 리뉴얼 사이트에서의 이상적 경험

**일일 운영:**

1. **운영 대시보드**: 로그인하면 운영자 전용 대시보드가 바로 열린다. 오늘의 할 일(AI토크 확인, 게시글 승인 대기, 스터디 상태, 유저 민원 등)이 카드 형태로 정리되어 있다.
2. **AI토크 관리**: 대시보드에서 "미배치 AI토크" 목록을 확인하고, 드래그 앤 드롭 또는 원클릭으로 "커뮤니티 이벤트"로 이동시킨다.
3. **홈 화면 수정**: CMS 관리자 페이지에서 홈 배너, 추천 콘텐츠, CTA 영역을 시각적으로 편집한다. 드래그 앤 드롭 방식의 블록 에디터로 2-3클릭 내에 수정을 완료한다.

**스터디 운영 전체 흐름:**

4. **스터디 관리 워크플로**: 스터디 관리 전용 페이지에서 모집 → 상세페이지 작성 → 검수 → 게시의 전 과정을 단계별로 관리한다. 각 스터디의 현재 상태(작성 중/검수 대기/게시 완료)가 한눈에 보인다.
5. **스터디장 온보딩**: 신규 스터디장에게 전용 URL을 보내면, 자신의 스터디 상세페이지 편집 화면으로 바로 이동한다. 단계별 가이드가 포함된 작성 양식이 제공되어 별도 안내가 필요 없다.
6. **최종제출 워크플로**: 스터디 관리 목록에서 "검수 완료" 버튼을 클릭하면 자동으로 랜딩페이지에 게시된다. 토글을 찾아 스크롤할 필요가 없다.
7. **카테고리 관리**: 핵심 카테고리 8-10개로 정리된 정보구조. 자주 사용하는 카테고리에 즐겨찾기/핀 기능이 있어 빠르게 접근한다.

**DM/메시징 시스템 재설계:**

8. **자동 메시지와 실제 DM 분리**: 자동 발송 메시지(시스템 알림, 봇 메시지)와 실제 사람 간 DM이 별도 채널로 분리된다. 중요한 DM이 자동 메시지에 묻히지 않는다.
9. **알림 이메일 정상 작동**: DM 수신 시 이메일 알림이 정상적으로 전달되어 사이트에 접속하지 않아도 중요한 메시지를 놓치지 않는다.

**멤버 셀프서비스 및 이벤트 처리:**

10. **멤버 게시글 공간이동**: 멤버가 직접 자신의 게시글을 다른 스페이스로 이동할 수 있다. 삭제 후 재작성이 불필요하므로 웹훅 중복 발동이 원천 차단된다.
11. **웹훅 중복 방지**: 이벤트 핸들링에 중복 방지(deduplication) 로직이 적용되어, 동일 게시글에 대한 웹훅이 중복 실행되지 않는다.

**DB 관리 및 자동화 내재화:**

12. **어드민 통합 DB 관리**: 멤버 결제 정보, 스터디 수강 이력 등을 어드민 페이지에서 직접 관리할 수 있다. DB에 직접 접근할 필요가 없다.
13. **스터디 페이지 생성 내재화**: 에어테이블 → 뽀짝이 → 베터모드로 이어지던 외부 자동화 파이프라인이 사이트 내 스터디 관리 워크플로로 대체된다.

**통합 도구:**

14. **임시저장 에디터**: 자동 임시저장이 되는 리치 텍스트 에디터. 외부에서 작성할 필요가 없다.
15. **알림 통합**: 슬랙 연동으로 주요 알림(게시글 승인 요청, 스터디장 제출 등)을 슬랙에서도 확인하고 사이트에서도 확인할 수 있다.

**핵심 개선:**
- 운영자 대시보드로 일일 업무 한눈에 관리
- 스터디 운영 워크플로 자동화 (수동 토글/이동 제거)
- 홈 화면 수정 8단계 → 2-3클릭으로 단축
- 카테고리 30개 → 핵심 8-10개로 재구성
- 신규 스터디장 셀프 온보딩 가능
- DM/메시징 시스템 정상화 (자동 메시지 vs 실제 DM 분리)
- 멤버 게시글 공간이동 → 웹훅 중복 원천 차단
- 어드민에서 DB 관리 가능 (직접 DB 접근 불필요)
- 외부 자동화 파이프라인 내재화

## 페르소나 4: 마케터/콘텐츠 크리에이터

### 프로필

| 항목 | 내용 |
|------|------|
| 이름 | 소연 / 재호 (실제 마케터) |
| 나이 | 20대 후반 ~ 30대 초반 |
| 직업 | GPTers 콘텐츠 마케터 |
| AI 숙련도 | 중급~고급 (AI 도구를 마케팅에 적극 활용) |
| 동기 | SEO 최적화된 콘텐츠로 유입을 늘리고, 유입된 방문자를 회원가입 → 스터디 구매로 전환시키고 싶음 |

**핵심 목표:** 유입 → 가입 → 스터디 구매 전환 퍼널을 최적화하고 싶다.

### As-Is 시나리오: 현재 사이트에서의 경험

**콘텐츠 제작 (소연):**

1. **게시글 작성**: 로그인 후 "포스트 - 최신 AI 소식" 또는 "사례 게시글 - 바이브 코딩" 등 맞는 게시판을 찾아 글을 올린다. 어떤 스페이스에 올려야 할지 고민될 때가 있다.
2. **가독성 한계**: 작성한 게시글의 가독성이 다른 사이트 대비 부족하다. 뉴스레터에서 게시글로 랜딩시키는데, 랜딩 후 게시글의 가독성이 콘텐츠의 효과를 떨어뜨린다.
3. **디자인 일관성 부재**: 사이트 전반의 디자인 일관성이 부족해 브랜딩이 약하게 느껴진다.

**캠페인/전환 (소연):**

4. **유입 경로**: 구글 검색이 가장 큰 유입 채널이다. AI 관련 검색어에서 GPTers 게시글 클릭 → 글 읽기 → (이탈 포인트) → 회원가입 → 뉴스레터/홍보 메시지 → 스터디 구매 순서다.
5. **전환 병목 1**: 게시글 확인 후 회원가입으로 이어지는 것이 더디다. 게시글 내에서 회원가입을 유도하는 장치가 부족하다.
6. **전환 병목 2**: 상품 상세 페이지까지는 도달하지만 실제 구매 전환이 잘 안 된다. 상세 페이지의 설득력이나 구매 경험에 문제가 있다.
7. **광고 지면**: 배너 같은 광고 지면이 잘 보이지 않아 마케팅 활용도가 낮다. 스터디 판매 기간에 공간을 리뉴얼하려 해도 효과적인 지면이 부족하다.
8. **띠배너 변경**: 공간 설정에서 띠배너를 변경하는데, 더 잘 보이는 위치에 노출되었으면 한다.

**Admin/메뉴 설정 및 광고 소재 제작 (재호):**

9. **AI 설명회 메뉴 설정**: AI 설명회 일정을 확인한 뒤, 교육 리드와 함께 메뉴 세팅을 조율한다. 그 후 어드민 페이지에서 메뉴 설정을 직접 수정한다.
10. **스페이스-UI 매핑 불명확**: 원하는 UI 요소를 수정하고 싶은데, 어드민에서 어떤 스페이스를 편집해야 해당 UI가 바뀌는지 알 수 없다. 스페이스와 실제 화면의 대응 관계가 불투명하여 매번 헤맨다.
11. **콘텐츠 수정 제한**: 섹션 제목이나 텍스트를 자유롭게 수정할 수 없다. 원하는 문구로 바꾸고 싶어도 시스템이 허용하지 않는 경우가 있다.
12. **메타 광고 소재 제작**: 이벤트 페이지를 확인하고, UTM 파라미터를 추가하여 메타 광고 소재를 제작한다. 구글시트에서 UTM을 관리하고, 메타 광고 관리자에서 광고를 세팅한다.
13. **상품/쿠폰 관리**: 상품과 쿠폰을 Retool(외부 도구)로 관리한다. 사이트 어드민에서 직접 관리할 수 없어 별도의 외부 도구에 의존한다.
14. **이모지 과다 메뉴**: 모든 메뉴에 이모지가 붙어 있어 시각적으로 혼란스럽다. 메뉴의 계층과 중요도가 이모지로 인해 오히려 불명확해진다.
15. **가입인사 게시판 의문**: "가입인사" 게시판의 존재 의미가 불분명하다. 실질적인 커뮤니티 활성화에 기여하지 못하는 형식적 기능으로 보인다.

**성과 측정:**

16. **데이터 확인**: GA, 믹스패널 등 외부 도구로 콘텐츠 성과를 확인한다. 사이트 자체에서 확인할 수 있는 데이터가 제한적이다.
17. **외부 도구 분산**: 스티비(뉴스레터), 구글시트(UTM 관리), 메타 광고 관리자, 카카오모먼트, Retool 등 다수의 외부 도구에 업무가 분산되어 있다.

**Pain Points:**
- 게시글 가독성 부족으로 콘텐츠 마케팅 효과 저하
- 게시글 → 회원가입 전환 경로가 불명확
- 상품 상세 → 구매 전환율 낮음
- 광고 배너 지면이 비효율적
- 게시판 분류 체계 혼란 (어디에 올릴지 매번 고민)
- 디자인 일관성 부족으로 브랜드 약화
- Admin 스페이스-UI 매핑 불명확 (어떤 걸 수정해야 원하는 UI가 바뀌는지 모름)
- 콘텐츠 수정 자유도 부족 (섹션 제목/텍스트 변경 제한)
- 상품/쿠폰 관리가 Retool 의존 (사이트 어드민에서 불가)
- 이모지 과다로 메뉴 혼란 (시각적 노이즈)
- 가입인사 등 형식적 기능의 존재 의문

### To-Be 시나리오: 리뉴얼 사이트에서의 이상적 경험

**콘텐츠 제작:**

1. **게시글 작성**: 카테고리가 명확하게 정리되어 있어 "AI 소식", "활용 사례", "튜토리얼" 등 직관적으로 맞는 게시판을 선택한다. 리치 텍스트 에디터로 가독성 높은 글을 바로 작성할 수 있다.
2. **SEO 최적화**: 게시글 작성 시 SEO 메타데이터(제목, 설명, 키워드)를 별도로 설정할 수 있다. Open Graph 이미지도 지정 가능하다. 게시글 URL이 검색 엔진과 LLM 대화에 최적화된 구조다.
3. **브랜드 일관성**: shadcn 기반의 일관된 디자인 시스템으로 모든 페이지가 통일된 브랜드 경험을 제공한다.

**Admin UX 개선 (스페이스-UI 매핑):**

4. **시각적 페이지 빌더**: 어드민에서 스페이스를 편집할 때, 실제 사이트 화면의 어느 부분이 바뀌는지 미리보기로 확인할 수 있다. 스페이스와 UI 요소의 대응 관계가 시각적으로 명확하다.
5. **콘텐츠 자유 편집**: 섹션 제목, 텍스트, 메뉴 명칭 등을 마케터가 자유롭게 수정할 수 있다. 시스템 제약 없이 원하는 문구를 바로 적용할 수 있다.
6. **깔끔한 메뉴 디자인**: 이모지 없이도 계층과 중요도가 명확하게 전달되는 메뉴 구조. 필요에 따라 아이콘을 선택적으로 사용할 수 있다.

**캠페인/전환:**

7. **CTA 배치**: 게시글 내 전략적 위치에 회원가입, 뉴스레터 구독, 스터디 안내 CTA를 배치할 수 있다. 마케터가 CTA의 문구와 링크를 직접 관리한다.
8. **전환 최적화**: 게시글 → 회원가입 플로우가 매끄럽다. 글을 읽는 도중 자연스럽게 가입을 유도하는 인라인 CTA, 스크롤 트리거 팝업 등을 마케터가 설정할 수 있다.
9. **상품 상세 페이지**: 스터디 상세 페이지가 랜딩 페이지 수준으로 설득력 있게 구성된다. 커리큘럼, 후기, 수료생 성과 등이 시각적으로 잘 정리되어 있고, 구매 버튼이 항상 접근 가능하다.
10. **광고 지면 활용**: 홈 메인 배너, 사이드바, 인피드 광고 등 다양한 지면을 마케터가 직접 관리하고 교체할 수 있다. 스터디 판매 기간에 맞춰 캠페인 배너를 즉시 변경할 수 있다.

**상품/쿠폰 통합 관리:**

11. **어드민 내 상품/쿠폰 관리**: Retool 없이 사이트 어드민에서 상품 등록, 수정, 쿠폰 발행/관리를 직접 수행할 수 있다. 마케팅 캠페인에 필요한 쿠폰을 즉시 생성하고 배포한다.

**성과 측정:**

12. **내장 애널리틱스**: 사이트 내에서 게시글별 조회수, 전환율, CTA 클릭률을 확인할 수 있다. GA/믹스패널과의 이벤트 연동도 깔끔하게 세팅되어 있다.
13. **UTM 관리 통합**: 광고 소재 생성 시 UTM 파라미터를 사이트 내에서 설정하고 추적할 수 있다. 별도 구글시트 관리가 불필요하다.

**핵심 개선:**
- 가독성 높은 게시글 에디터 + SEO 최적화 기능
- 마케터 주도 CTA 관리 (위치, 문구, 링크)
- 전환 퍼널 최적화 (게시글 → 가입 → 구매)
- 다양한 광고 지면과 마케터 직접 관리 기능
- 일관된 디자인 시스템으로 브랜드 강화
- Admin 스페이스-UI 매핑 명확화 (시각적 페이지 빌더)
- 콘텐츠 자유 편집 (섹션 제목/텍스트 제한 해소)
- 상품/쿠폰 어드민 통합 관리 (Retool 탈피)
- 깔끔한 메뉴 디자인 (이모지 과다 해소)

### 핵심 니즈와 기대

## 페르소나 5: 일반 AI 활용자 (가벼운 참여자)

> 다혜 인터뷰에서 제기된 "AI계의 LinkedIn" 비전과 가벼운 콘텐츠 참여 니즈에서 도출된 페르소나. 현재 사이트에서는 정식 "사례 게시글" 외에 가볍게 참여할 수 있는 포맷이 없어 이 유형의 유저가 사실상 배제되고 있다.

### 프로필

| 항목 | 내용 |
|------|------|
| 이름 | 박민호 |
| 나이 | 35세 |
| 직업 | 대기업 기획자 |
| AI 숙련도 | 중급 (업무에 ChatGPT, Copilot 등을 일상적으로 활용) |
| 동기 | AI 활용 경험을 가볍게 공유하고, 자신만의 AI 활용 프로필을 만들어 전문성을 보여주고 싶음. 그러나 긴 사례글을 쓸 시간과 에너지는 없음 |

**핵심 목표:** AI 활용 경험을 부담 없이 기록하고, "나는 AI를 이렇게 쓰고 있다"를 보여줄 수 있는 셀프 브랜딩 프로필을 갖고 싶다.

### As-Is 시나리오: 현재 사이트에서의 경험

1. **콘텐츠 탐색**: GPTers에 가입은 해두었지만, 주로 다른 사람의 사례 게시글을 읽기만 한다. 본인이 직접 올릴 수 있는 곳을 찾기 어렵다.
2. **참여 장벽**: 사례 게시글은 구조화된 긴 글을 요구하기 때문에, "오늘 ChatGPT로 보고서 초안 5분 만에 뽑았다" 같은 짧은 경험을 공유할 포맷이 없다.
3. **프로필 미흡**: 프로필 페이지에 자신의 AI 활용 이력이나 전문성을 보여줄 수 있는 구조가 없다. LinkedIn처럼 "이 사람이 AI를 어떻게 쓰는 사람인지" 한눈에 알 수 없다.
4. **커뮤니티 참여 포기**: 가벼운 참여 방법이 없어 결국 "읽기만 하는 유저"로 남는다. 가입인사를 올리라고 하지만, 형식적인 인사 외에 의미 있는 첫 참여 경험을 제공하지 못한다.
5. **이탈**: 스터디에 참여할 정도의 시간 투자는 어려워 점차 방문 빈도가 줄어든다.

**Pain Points:**
- 짧은 AI 활용 경험을 공유할 수 있는 가벼운 포맷이 없음
- 프로필에서 AI 활용 전문성을 셀프 브랜딩할 방법이 없음
- "사례 게시글" 외에 참여할 수 있는 콘텐츠 형식이 부재
- 읽기 전용 유저로 고착화 → 커뮤니티 이탈
- 가입인사 등 형식적 기능만 존재

### To-Be 시나리오: 리뉴얼 사이트에서의 이상적 경험

1. **가벼운 포스트**: 트위터/LinkedIn 스타일의 피드형 짧은 포스트를 올릴 수 있다. "오늘 Claude로 기획서 작성 → 30분 절약!" 같은 한두 줄짜리 글도 환영받는다.
2. **자랑글/TIL**: "오늘의 AI 활용" 같은 가벼운 카테고리에서 짧은 자랑글(TIL: Today I Learned)을 올린다. 사진 한 장 + 한두 줄 설명으로 충분하다.
3. **피드 탐색**: 홈 피드에서 다른 사람의 가벼운 포스트를 스크롤하며 "좋아요"나 짧은 댓글로 반응한다. 사례 게시글보다 훨씬 활발한 피드가 형성된다.
4. **AI 셀프 브랜딩 프로필**: 프로필 페이지에 자신이 사용하는 AI 도구, 활용 분야, 올린 포스트, 참여 이력 등이 자동으로 정리된다. "AI계의 LinkedIn" 프로필로서 공유 가능한 링크를 제공한다.
5. **점진적 참여 유도**: 가벼운 포스트를 여러 번 올리면서 자연스럽게 커뮤니티에 익숙해지고, 나중에 스터디 참여나 본격적인 사례글 작성으로 이어진다.

**핵심 개선:**
- 가벼운 피드형 포스트 기능 (진입 장벽 최소화)
- AI 셀프 브랜딩 프로필 (활용 도구, 분야, 이력 표시)
- 가입인사 → 의미 있는 첫 참여 경험으로 전환
- 읽기 전용 유저 → 가벼운 참여자 → 적극 참여자 전환 경로

### 핵심 니즈와 기대

| 니즈 | 기대 |
|------|------|
| 가벼운 참여 | 짧은 포스트로 부담 없이 AI 경험 공유 |
| 셀프 브랜딩 | AI 활용 프로필, "AI계의 LinkedIn" |
| 소속감 | 좋아요, 댓글 등 가벼운 상호작용 |
| 점진적 전환 | 가벼운 참여 → 스터디/사례글로 자연스러운 확장 |
| 발견 | 내 관심 분야의 AI 활용 사례를 피드에서 쉽게 발견 |
