## 공통 정의

### 정렬 기준 (SortTabs)

인사이트, 커뮤니티, 태그 피드에서 공통 적용되는 정렬 기준.

| 정렬 | 기준 | 구현 방식 |
|------|------|---------|
| 최신순 | 게시글 작성 시각 기준 내림차순 | `created_at DESC` |
| 인기순 | **최근 7일 이내** 좋아요 수 기준 내림차순 | `likes WHERE created_at >= now() - 7days, COUNT DESC` |

> **인기순 정책 근거**: 단순 누적 좋아요 수로 정렬하면 오래된 글이 상위를 고정 점유함. 7일 기준으로 제한해 최신 인기 콘텐츠가 노출될 수 있도록 함.
>
> 프로토타입에서는 mock 데이터의 `votes` 필드 내림차순으로 단순 구현됨. 백엔드 연결 시 위 정책으로 교체 필요.

## 공통 레이아웃

### Navbar (`components/navbar.tsx`)
- **로고**: 클릭 시 홈(`/`)으로 이동
- **검색창**: 입력 시 드롭다운으로 인기 태그 + 인기 검색어 표시 → 엔터/클릭 시 `/search?q=검색어`로 이동
- **GNB 메뉴**: 인사이트 / 스터디 / 커뮤니티 / 이벤트 (현재 경로 하이라이트)
- **글쓰기 버튼**: `/write?type=case`로 이동
- **알림 벨**: 드롭다운으로 최근 알림 3개 표시 + 전체보기 → `/messages`
- **프로필 아이콘**: 드롭다운으로 내 프로필 / 내 스터디 / 알림·쪽지 / 설정 / 어드민 / 로그아웃

### Footer (`components/footer.tsx`)
- 단순 텍스트 푸터 (저작권 표시)

## 홈 (/)

### 목적
신규 방문자 진입점. 핵심 콘텐츠 미리보기 + CTA.

### 주요 기능
| 기능 | 설명 |
|------|------|
| Hero 배너 | 스터디 모집 안내 + CTA 버튼 |
| 에디터 픽 | 운영진 추천 게시글 3개 (PostCard) |
| 트렌딩 토픽 | 인기 태그 목록 (pill 형태) → 클릭 시 `/tag/[slug]` |
| 커뮤니티 스냅샷 | 회원수 / 게시글수 / 스터디수 통계 |
| 팔로우 추천 | 활발한 멤버 3명 + 팔로우 버튼 |
| 다가오는 이벤트 | 최신 이벤트 1~2개 미리보기 |

### 인터랙션
- 태그 클릭 → `/tag/[slug]`
- 게시글 클릭 → `/posts/[slug]`
- 팔로우 버튼 → 토글 (팔로잉 상태 전환)

## 인사이트 (/explore/feed)

### 목적
커뮤니티가 작성한 사례/팁/인사이트 글 탐색.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 카테고리 필터 | 가로 스크롤 탭 (전체 / AI활용법 / 프롬프트 / 자동화 등) |
| 정렬 | 최신순 / 인기순 토글 (→ [정렬 기준 참조](#정렬-기준-sorttabs)) |
| 게시글 목록 | PostCard 리스트 (페이지당 5개) |
| 페이지네이션 | 페이지 이동 시 해당 페이지 데이터 표시 |
| 사이드바 (데스크탑) | 추천 토픽 태그 목록 + 에디터 픽 |

### 인터랙션
- 카테고리 선택 → 필터링 + 1페이지 리셋
- 정렬 변경 → 재정렬
- 페이지 이동 → 해당 페이지 데이터

## 커뮤니티 피드 (/community/feed)

### 목적
자유로운 짧은 글 공유 (Threads 스타일).

### 주요 기능
| 기능 | 설명 |
|------|------|
| 인라인 작성 폼 | 피드 상단에 바로 작성 가능 |
| 피드 | FeedPost 카드 (좋아요 / 댓글 / 북마크) |
| 정렬 | 최신순 / 인기순 (→ [정렬 기준 참조](#정렬-기준-sorttabs)) |
| 더 불러오기 | 스크롤 하단에서 추가 포스트 로드 (2배치) |

### 인터랙션
- 인라인 폼 작성 후 게시 → 피드 맨 위에 새 포스트 추가
- 좋아요 → 토글
- 북마크 → 토글
- 더 불러오기 → 추가 3개씩, 2회 후 버튼 숨김

## 게시글 상세 (/posts/[slug])

### 목적
인사이트/사례 게시글 상세 읽기.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 시리즈 표시 | 속한 시리즈명 + 화수 → 클릭 시 시리즈 페이지 |
| 본문 | 마크다운 스타일 콘텐츠 |
| 태그 목록 | 본문 하단 |
| 사이드바 (데스크탑) | 좋아요 / 댓글 / 북마크 / 공유 (sticky) |
| 댓글 트리 | 댓글 + 대댓글 구조, 답글 작성 시 로컬 추가 |
| 임시저장 | 저장 시각 표시 |
| 관련 글 | 하단 3개 추천 |

### 인터랙션
- 좋아요 / 북마크 → 토글
- 공유 → clipboard 복사 + 아이콘 피드백
- 댓글/답글 등록 → 로컬 추가 + "등록됨" 피드백

### 인터랙션
- 뒤로가기 → `router.back()`

## 메시지 (/messages)

### 목적
알림 확인 + 회원 간 쪽지 송수신.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 알림 탭 | 소셜 / 스터디 / 이벤트 / 시스템 알림 |
| 받은 메시지 탭 | 수신 쪽지 목록 |
| 보낸 메시지 탭 | 발신 쪽지 목록 |
| 새 메시지 작성 | 받는사람 / 제목 / 내용 → 전송 후 보낸 탭 이동 |

### 알림 발생 시나리오

#### 소셜 (`NotificationRow` — actor Avatar 표시)
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `like` | 내 게시글에 좋아요 | `/posts/[slug]` |
| `comment` | 내 게시글에 댓글 | `/posts/[slug]` |
| `reply` | 내 댓글에 답글 | `/posts/[slug]` |
| `follow` | 다른 회원이 나를 팔로우 | `/profile/[username]` |

#### 스터디
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `payment` | 수강신청 결제 완료 | `/study/my` |
| `study` | 스터디 시작 안내 | `/study/my` |
| `study` | 세션 시작 리마인더 (당일) | `/study/[slug]/learn` |
| `study` | 과제 마감 임박 (48시간 전) | `/study/[slug]/learn/tasks` |
| `study` | 과제 제출 → 다시보기 권한 부여 | `/study/[slug]/learn` |
| `study` | 수료 판정 완료 | `/study/my/certificates` |
| `study` | 수료증 발급 | `/study/my/certificates` |
| `payment` | 환급 처리 완료 | ⚠️ 페이지 없음 (`/study/my/history` 연결 필요) |

#### 이벤트
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `event` | 이벤트 신청 완료 | `/events/[id]` |
| `event` | 이벤트 시작 리마인더 (전날) | `/events/[id]` |
| `event` | 이벤트 개설 심사 승인/반려 | `/events/[id]` |

#### 시스템
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `welcome` | 회원가입 완료 | `/` |
| `system` | 공지사항 | ⚠️ 페이지 없음 (공지 목록 페이지 미구현) |
| `system` | 에디터픽 선정됨 | `/posts/[slug]` |
| `study` | 새 쪽지 수신 | `/messages` |

### ⚠️ URL 없음 — 개발 시 페이지 연결 필요
| 알림 시나리오 | 필요한 페이지 |
|-------------|-------------|
| 환급 처리 완료 | 환급 내역 페이지 (`/study/my/history` 또는 별도 `/study/my/refunds`) |
| 공지사항 | 공지 목록/상세 페이지 미구현 |

### 인터랙션
- 알림 클릭 → 해당 게시글/프로필로 이동 (href 있는 경우만 Link 렌더링)
- 새 메시지 전송 → 보낸 메시지 목록에 추가 + 보낸 탭으로 전환
- 현재 mock 데이터의 `study` / `event` / `payment` 알림에 href 누락 → 백엔드 연결 시 추가 필요

## 프로필 (/profile/[username])

### 목적
회원의 공개 프로필 + AI 이력서.

### 본인 프로필 vs 타인 프로필 차이

#### 헤더 영역
| 요소 | 본인 (`isOwn = true`) | 타인 (`isOwn = false`) |
|------|----------------------|----------------------|
| 커버 배너 | "커버 변경" 버튼 표시 | 없음 |
| 우측 버튼 | "프로필 수정" + "LinkedIn 공유" | "팔로우 / 팔로잉" 토글 버튼 |
| 쪽지 버튼 | 없음 | "쪽지 보내기" 버튼 표시 |

#### 탭 구성
| 탭 | 본인 | 타인 |
|----|------|------|
| 작성글 | 공개 (전체) | 공개 (전체) |
| 시리즈 | 공개 + **편집 버튼** | 공개, 편집 없음 |
| 이벤트 | 공개 + **수정 / 마감 / 취소 버튼** | 공개, 관리 버튼 없음 |
| 북마크 | **탭 표시됨** | **탭 숨김** (북마크는 비공개) |
| 팔로워 | 공개 | 공개 |
| 팔로잉 | 공개 | 공개 |

#### 사이드바 우측
| 요소 | 본인 | 타인 |
|------|------|------|
| 스터디 이력 | 공개 | 공개 |
| 뱃지 | 공개 | 공개 |

> **현재 구현 상태**: `isOwn`이 `true`로 하드코딩됨. 백엔드 연결 시 로그인한 사용자의 `username`과 URL의 `[username]` 파라미터를 비교하여 분기 처리 필요.
>
> ```typescript
> const isOwn = session?.user?.username === params.username;
> ```

### 인터랙션
- 팔로우 버튼 → 팔로우/팔로잉 토글
- 시리즈 편집 버튼 → 제목/설명 인라인 입력 → 저장/취소
- 이벤트 마감/취소 → 상태 변경 (본인 프로필만)

### 뱃지 시스템

#### 획득 조건 (자동 지급)

| 뱃지 | 획득 조건 | 지급 시점 |
|------|----------|---------|
| AI 입문자 | 첫 번째 스터디 수료 | 수료 확정 즉시 |
| AI ○○ 전문가 | 해당 스터디 수료 (스터디별 1개) | 수료 확정 즉시 |
| 3기 수료 / 5기 수료 | 누적 수료 기수 달성 | 수료 확정 즉시 |
| 베스트 사례 1회 / 3회 / 5회 | 베스트 사례 선정 횟수 달성 | 선정 즉시 |
| 프롬프트 마스터 | 프롬프트 엔지니어링 스터디 수료 | 수료 확정 즉시 |
| 모각AI 리더 | 모각AI 세션 5회 이상 리드 | 달성 즉시 |
| 기여왕 | 월간 기여 포인트 Top 3 진입 | 월 마감 후 지급 |
| 소통왕 | 댓글 100개 이상 작성 | 달성 즉시 |
| 로켓 스타터 | 첫 게시글 작성 | 게시글 발행 즉시 |

> 어드민에서 수동 지급도 가능 (`/admin/badges` — Phase 2)

#### 대표 뱃지 선택 (프로필 헤더 노출)

- 보유 뱃지 중 **최대 3개**를 선택해 프로필 헤더에 노출
- 선택/변경: **설정 > 내 정보 탭** 에서 보유 뱃지 목록 중 체크박스로 선택
- 미선택 시: 최근 획득 순 3개 자동 노출
- 뱃지가 없으면 헤더에서 뱃지 영역 미표시

> **아바타 변경**: 프로필 페이지 아바타에 Camera 버튼 오버레이 (본인만) → 파일 선택으로 변경. 설정 페이지에서는 제공하지 않음.

### 로그인 가드 (Auth Guard)
- `/write`, `/messages`, `/checkout`, `/settings` → 비로그인 시 `/login`으로 리다이렉트 필요
- `/profile/[username]` → 본인 탭(북마크 등)은 로그인 필요
- 좋아요, 북마크, 팔로우 → 비로그인 시 로그인 유도 모달
