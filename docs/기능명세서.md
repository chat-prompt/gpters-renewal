## 공통 정의

### 정렬 기준 (SortTabs)

인사이트, 커뮤니티, 태그 피드에서 공통 적용되는 정렬 기준.

| 정렬 | 기준 | 구현 방식 |
|------|------|---------|
| 최신순 | 게시글 작성 시각 기준 내림차순 | `created_at DESC` |
| 인기순 | **최근 7일 이내** 좋아요 수 기준 내림차순 | `likes WHERE created_at >= now() - 7days, COUNT DESC` |

> **인기순 정책 근거**: 단순 누적 좋아요 수로 정렬하면 오래된 글이 상위를 고정 점유함. 7일 기준으로 제한해 최신 인기 콘텐츠가 노출될 수 있도록 함.
>
> 프로토타입에서는 mock 데이터의 `votes` 필드 내림차순으로 단순 구현됨. 백엔드 연결 시 위 정책으로 교체 필요.

### 인증 시스템 (useAuth)

프로토타입에서 데모용 인증 시스템이 구현되어 있다. Supabase Auth 연결 전까지 클라이언트 상태 기반으로 동작한다.

#### 핵심 구조

| 파일 | 역할 |
|------|------|
| `lib/auth-context.tsx` | AuthContext 정의 + `useAuth()` 훅 export |
| `lib/auth-provider-wrapper.tsx` | "use client" 래퍼 (서버 컴포넌트에서 Provider 사용용) |
| `app/login/page.tsx` | 데모 로그인 페이지 |

#### useAuth() 훅 인터페이스

```typescript
interface AuthContextValue {
  user: AuthUser | null;        // { name, username, role }
  isLoggedIn: boolean;          // user !== null
  isAdmin: boolean;             // role === "admin"
  isOwner: (username: string) => boolean;  // 현재 유저 === 해당 username
  toggle: () => void;           // 데모용 로그인/로그아웃 토글
}

interface AuthUser {
  name: string;       // "홍길동"
  username: string;   // "honggildong"
  role: "member" | "leader" | "admin";
}
```

- **초기 상태**: `user = null` (비로그인)
- **데모 유저**: `{ name: "홍길동", username: "honggildong", role: "admin" }`
- **toggle()**: null ↔ MOCK_USER 전환 (로그인/로그아웃)

#### 인증 패턴 3가지

**패턴 1: 로그인 가드 (페이지 전체 차단)**

비로그인 시 페이지 콘텐츠 대신 로그인 유도 화면을 표시한다.

사용 페이지: `/profile/my`, `/events/[id]/edit`

```tsx
const { isLoggedIn } = useAuth();
const pathname = usePathname();

if (!isLoggedIn) {
  return (
    <div>
      <p>로그인이 필요합니다</p>
      <Link href={`/login?from=${encodeURIComponent(pathname)}`}>로그인</Link>
    </div>
  );
}
```

**패턴 2: 조건부 UI 분기 (로그인/비로그인 다른 버튼)**

같은 위치에 로그인 상태에 따라 다른 UI를 렌더링한다.

사용 페이지: `/events/[id]` (신청하기 vs 로그인 후 참가), (팔로우 vs 로그인 후 팔로우)

```tsx
{isLoggedIn ? (
  <Button onClick={() => setRegistered(true)}>신청하기</Button>
) : (
  <Link href={`/login?from=${encodeURIComponent(pathname)}`}>로그인 후 참가</Link>
)}
```

**패턴 3: 소유자 확인 (isOwner)**

본인 콘텐츠에만 수정/관리 버튼을 표시한다.

사용 페이지: `/events/[id]` (수정 버튼), `/events/[id]/edit` (접근 권한)

```tsx
const { isLoggedIn, isOwner } = useAuth();

{isLoggedIn && isOwner(event.host.username) && (
  <Link href={`/events/${event.id}/edit`}>
    <Button variant="outline" size="sm">수정</Button>
  </Link>
)}
```

#### from 파라미터 (리다이렉트)

로그인 후 원래 페이지로 복귀하기 위해 `from` 쿼리 파라미터를 사용한다.

- **설정**: `<Link href={/login?from=${encodeURIComponent(pathname)}}>`
- **처리** (`/login`): `const from = searchParams.get("from") || "/";` → 로그인 성공 시 `router.replace(from)`
- **이미 로그인**: `/login` 진입 시 `isLoggedIn`이면 즉시 `from`으로 리다이렉트

#### 백엔드 연결 시 변경 사항

| 현재 (프로토타입) | 백엔드 연결 후 |
|---|---|
| `useState` + `toggle()` | Supabase Auth `onAuthStateChange` |
| MOCK_USER 하드코딩 | `supabase.auth.getUser()` |
| `/login` 데모 버튼 | OAuth (Google, Kakao 등) 또는 매직링크 |
| `isOwner()` username 비교 | DB user.id 기반 비교 |
| `isAdmin` role 하드코딩 | DB user_roles 테이블 기반 |

### LMS 레이아웃 (`app/study/my/layout.tsx`)

로그인 필수. `useAuth()` 훅으로 비로그인 시 로그인 유도 화면 표시.

- **데스크톱**: 좌측 고정 사이드바(200px) + 우측 콘텐츠 영역(max-w 860px)
- **모바일**: 하단 고정 탭 네비게이션 (가로 스크롤)
- **사이드바 메뉴**: 홈 / 다시보기 / 게시글 / 수강 이력 / 수료증 (5개 + 구분선 + "나의 학습" 링크)
- **사용자 정보 카드**: 사이드바 하단에 이름 + 기수/스터디명 표시

### 어드민 레이아웃 (`app/admin/layout.tsx`)

- **접근 제어**: `useAuth()`로 로그인 + 관리자 권한 확인. 비인가 시 "관리자 권한이 필요합니다" + 로그인 링크
- **구조**: 좌측 고정 사이드바(w-48) + 우측 콘텐츠(max-w-[860px])
- **사이드바 3개 그룹**:
  - **일반**: 대시보드, 게시글, 회원, 배너, 이벤트, 화이트보드, 텍스트
  - **스터디**: 스터디, 기수, 세션, 상품/쿠폰, 수료/환급
  - **시스템**: 신고, 뱃지, 분류, 리포트
- **반응형**: `hidden md:block` — 모바일에서 사이드바 숨김

## 로그인 (/login)

### 목적
로그인 페이지. 프로토타입에서는 데모 로그인 버튼으로 동작.

### 주요 기능
| 기능 | 설명 |
|------|------|
| GPTers 로고 | 클릭 시 홈(`/`)으로 이동 |
| 데모 로그인 버튼 | 클릭 시 `useAuth().toggle()` 호출 → MOCK_USER("홍길동")로 로그인 |
| from 리다이렉트 | `?from=/path` 쿼리 파라미터로 로그인 후 원래 페이지로 복귀 |
| 이미 로그인 | 진입 시 `isLoggedIn`이면 즉시 `from` URL로 `router.replace` |
| 안내 텍스트 | "실제 로그인은 Supabase Auth 연결 후 활성화됩니다" |

### 인터랙션
- 데모 로그인 클릭 → `toggle()` + `router.replace(from)`
- 이미 로그인 상태로 `/login` 접근 → useEffect에서 즉시 from으로 리다이렉트
- `from` 파라미터 없으면 기본값 `"/"`

### 백엔드 연결 시
- OAuth 로그인 (Google, Kakao 등) 또는 이메일 매직링크로 교체
- Supabase Auth `signIn()` → `onAuthStateChange`로 상태 관리

|------|------|

### 인터랙션
- 카테고리 선택 → 필터링 + 1페이지 리셋

- 페이지 이동 → 해당 페이지 데이터

## 게시글 상세 (/posts/[slug])

### 목적
인사이트/사례 게시글 상세 읽기.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 시리즈 표시 | 속한 시리즈명 + 화수 → 클릭 시 시리즈 페이지 |
| 본문 | 마크다운 스타일 콘텐츠 |
| 태그 목록 | 본문 하단 |
| 사이드바 (데스크탑) | 좋아요 / 댓글 / 북마크 / 공유 (sticky) |
| 댓글 트리 | 댓글 + 대댓글 구조, 답글 작성 시 로컬 추가 |
| 임시저장 | 저장 시각 표시 |
| 관련 글 | 하단 3개 추천 |

### 인터랙션
- 좋아요 / 북마크 → 토글
- 공유 → clipboard 복사 + 아이콘 피드백
- 댓글/답글 등록 → 로컬 추가 + "등록됨" 피드백

## 글쓰기 (/write)

### 목적
인사이트 사례글 또는 커뮤니티 피드 포스트 작성.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 타입 선택 | 사례·인사이트 / 커뮤니티 포스트 탭 전환 |
| 사례글 폼 | 제목 / 카테고리 / 태그 / 본문 (rich 편집 영역) |
| 커뮤니티 폼 | 짧은 텍스트 + 이미지 첨부 |
| 임시저장 | 저장 시각 표시 |
| 발행 | 타입에 따라 적절한 페이지로 이동 |

### 인터랙션
- 뒤로가기 → `router.back()`
- 발행: 사례글 → `/posts/[slug]`, 커뮤니티 → `/community/feed`

## 메시지 (/messages)

### 목적
알림 확인 + 회원 간 쪽지 송수신.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 알림 탭 | 소셜 / 스터디 / 이벤트 / 시스템 알림 |
| 받은 메시지 탭 | 수신 쪽지 목록 |
| 보낸 메시지 탭 | 발신 쪽지 목록 |
| 새 메시지 작성 | 받는사람 / 제목 / 내용 → 전송 후 보낸 탭 이동 |

### 알림 발생 시나리오

#### 소셜 (`NotificationRow` — actor Avatar 표시)
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `like` | 내 게시글에 좋아요 | `/posts/[slug]` |
| `comment` | 내 게시글에 댓글 | `/posts/[slug]` |
| `reply` | 내 댓글에 답글 | `/posts/[slug]` |
| `follow` | 다른 회원이 나를 팔로우 | `/profile/[username]` |

#### 스터디
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `payment` | 수강신청 결제 완료 | `/study/my` |
| `study` | 스터디 시작 안내 | `/study/my` |
| `study` | 세션 시작 리마인더 (당일) | `/study/[slug]/learn` |
| `study` | 과제 마감 임박 (48시간 전) | `/study/[slug]/learn/tasks` |
| `study` | 과제 제출 → 다시보기 권한 부여 | `/study/[slug]/learn` |
| `study` | 수료 판정 완료 | `/study/my/certificates` |
| `study` | 수료증 발급 | `/study/my/certificates` |
| `payment` | 환급 처리 완료 | `/study/my/history` |

#### 이벤트
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `event` | 이벤트 신청 완료 | `/events/[id]` |
| `event` | 이벤트 시작 리마인더 (전날) | `/events/[id]` |
| `event` | 이벤트 개설 심사 승인/반려 | `/events/[id]` |

#### 시스템
| 타입 | 발생 조건 | 클릭 이동 URL |
|------|----------|--------------|
| `welcome` | 회원가입 완료 | `/` |
| `system` | 공지사항 | `/notices` |
| `system` | 에디터픽 선정됨 | `/posts/[slug]` |
| `study` | 새 쪽지 수신 | `/messages` |

### 인터랙션
- 알림 클릭 → 해당 게시글/프로필로 이동 (href 있는 경우만 Link 렌더링)
- 새 메시지 전송 → 보낸 메시지 목록에 추가 + 보낸 탭으로 전환
- 현재 mock 데이터의 `study` / `event` / `payment` 알림에 href 누락 → 백엔드 연결 시 추가 필요

## 이벤트 상세 (/events/[id])

### 주요 기능
| 기능 | 설명 |
|------|------|
| 미리보기 배너 | 비공개 상태(`draft`/`pending_review`/`rejected`) 이벤트 접근 시 상단 안내 배너 표시 |
| 커버 이미지 | 상단 전폭 이미지 (Unsplash placeholder) |
| 카테고리 + 유형 뱃지 | 카테고리명 (텍스트) + 온라인/오프라인 뱃지 |
| 이벤트 정보 | 일시 / 장소 / 주최자(호스트) 표시 |
| 소개 | longDescription 전문 (whitespace-pre-line) |
| 프로그램 | 타임라인 형태 agenda 목록 |
| 연사 | 카드형 연사 목록 — username 있으면 아바타/이름 프로필 링크 + 팔로우 버튼 |
| 참가 신청 카드 (데스크탑) | 우측 sticky — 가격 / 정원 바 / 참석자 아바타 / CTA / 공유 |
| 모바일 sticky CTA | 화면 하단 고정 — 가격 + 남은 자리 + 신청 버튼 |
| 수정 버튼 | 주최자 본인에게만 표시 (isOwner 확인) |

### 인증 분기
| 위치 | 로그인 상태 | 비로그인 상태 |
|------|-----------|-------------|
| 참가 신청 (데스크탑/모바일) | "신청하기" 버튼 → 클릭 시 "신청완료" + attendees +1 | "로그인 후 참가" → `/login?from=현재URL` |
| 연사 팔로우 | "팔로우/팔로잉" 토글 버튼 | "팔로우" → `/login?from=현재URL` (Link) |
| 수정 버튼 | 주최자 본인만 표시 | 미표시 |

### 인터랙션
- 참가 신청 → `registered` state 토글 + attendees +1 + 버튼 disabled
- 연사 팔로우 → `followingSet` Set으로 개별 토글 관리
- 뒤로가기 → `/events` 목록

### 변경 이력 (3/23)
- `hostType` 필드 제거됨 (호스트 유형 구분 UI 없음)
- 연사 팔로우 버튼 + 비로그인 분기 추가
- 주최자 수정 버튼 (isOwner) 추가

## 이벤트 수정 (/events/[id]/edit)

### 목적
이벤트 주최자가 기존 이벤트 정보를 수정하는 페이지.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 접근 제어 | 로그인 + 주최자 본인만 접근 가능 (`isLoggedIn && isOwner(event.hostUsername)`) |
| 비로그인 차단 | "이벤트를 수정하려면 로그인해주세요" + 로그인 버튼 (from 파라미터 포함) |
| 비소유자 차단 | "이벤트 주최자만 수정할 수 있습니다" + "이벤트로 돌아가기" 버튼 |
| 이벤트 미존재 | "이벤트를 찾을 수 없습니다" + 목록 링크 |
| 수정 폼 | EventForm 컴포넌트 (mode="edit", initialData 전달) |
| 반려 사유 표시 | status가 "rejected"인 경우 rejectionNote를 EventForm에 전달 → 상단 안내 배너 |
| 제출 버튼 | status가 "rejected"이면 "재제출", 아닌 경우 "수정 완료" → `/events/[id]`로 이동 |
| 뒤로가기 | ChevronLeft → `/events/[id]` |

### EventForm 필드 (create/edit 공용)
| 필드 | 필수 | 설명 |
|------|------|------|
| 커버 이미지 | 선택 | 드래그앤드롭 업로드 (권장 1200x630px) |
| 제목 | 필수 | 텍스트 입력 |
| 카테고리 | 필수 | 토크/모임/웨비나/네트워킹/워크숍/기타 (버튼 선택) |
| 유형 | 필수 | 온라인/오프라인 (버튼 선택) |
| 날짜 + 시간 | 필수 | date input + time input (시작~종료) |
| 장소 | 필수 | 텍스트 (유형에 따라 placeholder 변경) |
| 정원 | 필수 | 숫자 (2~1000명) |
| 가격 | 필수 | 무료/유료 선택 → 유료 시 금액 입력 |
| 짧은 소개 | 필수 | Textarea (100자 제한, 카운터 표시) |
| 상세 설명 | 선택 | 에디터 목업 (툴바: B/I/H1/H2/UL/OL/Code/Link) |
| 프로그램 | 선택 | AgendaInput (시간+제목 쌍 추가/삭제) |
| 연사 | 선택 | 이름+직함 쌍 추가/삭제 |
| 태그 | 선택 | 추천 태그 10개 중 선택/해제 (토글) |

### 인터랙션
- "수정 완료" / "재제출" 클릭 → `router.push(/events/[id])`
- 뒤로가기 → `/events/[id]`

수강생 대시보드. 이번 주 해야 할 일과 학습 현황을 한눈에 파악.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 이번 주 세션 | 세션 목록 (제목, 시간, 상태 Badge — 종료/오늘/예정) |
| 찐친챌린지 | 진행률 바(주차별 세그먼트), D-day 카운트다운, 참여자 수 |
| 내 스터디 바로가기 | 과제/출석/북마크 요약 stat 카드 + 과제·다시보기·게시글 바로가기 버튼 |
| 활동왕 TOP 10 | 게시글 수 기준 리더보드, 내 순위 하이라이트(bg-accent), 1~3등 색상 구분 |
| 이번 주 할 일 | 액션 아이템 카드 그리드 (사례 게시글 작성, AI토크 참여, 다시보기, 찐친챌린지 인증) |
| 과제 연동 안내 | 과제 제출 → 다시보기 열람 권한 획득 안내 |

### 인터랙션
- 바로가기 버튼 클릭 → `/study/[slug]/learn/tasks`, `/study/[slug]/learn/vod`, `/study/my/posts`
- 액션 아이템 CTA 버튼 → 각 href로 이동 (글쓰기, 커뮤니티, VOD, 게시글)

## 청강 VOD (/study/my/audit)

### 목적
모든 기수/스터디의 VOD를 탐색하고 시청. 과제 미제출 시 잠김 표시.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 3중 필터 | 기수(전체/21기/20기/19기) + 스터디(전체/6개) + 주차(전체/1~4주차) — Select 드롭다운 |
| VOD 그리드 | 카드 그리드 (1~3열 반응형). locked 여부에 따라 VodCard 또는 VodLockCard 렌더링 |
| 과제 연동 안내 | 상단에 AssignmentGuide (과제 제출 → 다시보기 잠금 해제 안내) |
| 빈 상태 | 필터 결과 없을 시 "조건에 맞는 VOD가 없습니다" 메시지 |

### 인터랙션
- 필터 변경 → 클라이언트 사이드 필터링 (state 기반)
- 잠긴 VOD → VodLockCard로 잠금 상태 표시

## 게시글 (/study/my/posts)

### 목적
스터디 수강생들이 작성한 사례 게시글(과제) 목록 탐색. 2-panel 레이아웃으로 목록과 상세를 동시에 확인.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 2중 필터 | 스터디(전체/AI 자동화/프롬프트 ENG/바이브코딩) + 주차(전체/1~4주차) — Select 드롭다운 |
| 내 글만 필터 | 체크박스 토글로 자기 글만 필터링 |
| 결과 카운트 | 필터링된 게시글 수 표시 |
| 2-Panel 레이아웃 | 좌측(45%) 게시글 목록 + 우측(55%) 게시글 상세 미리보기 (데스크톱만) |
| 게시글 목록 | 제목, 스터디+주차 Badge, 작성자, 날짜 표시. 선택 시 bg-accent 하이라이트 |
| 게시글 상세 | 선택된 게시글의 제목, Badge, 작성자, 날짜, 본문 전체 표시 |
| 빈 상태 | "게시글이 없습니다" / "게시글을 선택하세요" |

### 인터랙션
- 필터 변경 → 클라이언트 사이드 필터링
- 게시글 클릭 → 우측 패널에 상세 표시 (state 기반)
- 내 글만 체크박스 → `isMyPost` 필터링

## 스터디 관리 홈 (/study/manage/[slug])

### 목적
스터디장이 자신의 스터디를 운영하는 대시보드. 수강생 현황, 과제 제출률, 미제출자 파악.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 헤더 | 스터디 제목 + 기간 + 현재 주차 + 상태 Badge(진행중) |
| 서브 네비게이션 | 수강생 현황 / VOD 관리 / 공지 관리 버튼 → 각 하위 페이지로 이동 |
| 빠른 설정 | 최종제출 Toggle (ON/OFF), 상세페이지 수정 링크, 커리큘럼 수정 버튼 |
| 스터디 요약 | 4열 grid — 수강생(n/정원), 현재 주차(n/4), 누적 출석률(%), 수료 예측(순조/위험/미달 인원) + 모집률 Progress 바 |
| 주차별 과제 제출 차트 | 바 차트(4주) — 주차별 제출/미제출 비율 시각화 |
| 베스트 발표자 선정 | 수강생 목록에서 클릭으로 선택. 선택된 발표자 하이라이트(border-primary, bg-accent) |
| 미제출자 목록 | 이번 주 미제출자 카드 그리드 — 이름 + 미제출 항목(출석/과제) Badge |

### 인터랙션
- 최종제출 Toggle → 상태 전환
- 베스트 발표자 선택 → 클릭으로 토글 (하나만 선택)
- 서브 네비게이션 → `/study/manage/[slug]/members`, `/vod`, `/notices`

## 프로필 (/profile/[username])

### 목적
회원의 공개 프로필 + AI 이력서.

### 본인 프로필 vs 타인 프로필 차이

#### 헤더 영역
| 요소 | 본인 (`isOwn = true`) | 타인 (`isOwn = false`) |
|------|----------------------|----------------------|
| 커버 배너 | "커버 변경" 버튼 표시 | 없음 |
| 우측 버튼 | "프로필 수정" + "LinkedIn 공유" | "팔로우 / 팔로잉" 토글 버튼 |
| 쪽지 버튼 | 없음 | "쪽지 보내기" 버튼 표시 |

#### 탭 구성
| 탭 | 본인 | 타인 |
|----|------|------|
| 작성글 | 공개 (전체) | 공개 (전체) |
| 시리즈 | 공개 + **편집 버튼** | 공개, 편집 없음 |
| 이벤트 | 공개 + **수정 / 마감 / 취소 버튼** | 공개, 관리 버튼 없음 |
| 북마크 | **탭 표시됨** | **탭 숨김** (북마크는 비공개) |
| 팔로워 | 공개 | 공개 |
| 팔로잉 | 공개 | 공개 |

#### 사이드바 우측
| 요소 | 본인 | 타인 |
|------|------|------|
| 스터디 이력 | 공개 | 공개 |
| 뱃지 | 공개 | 공개 |

> **현재 구현 상태**: `isOwn`이 `true`로 하드코딩됨. 백엔드 연결 시 로그인한 사용자의 `username`과 URL의 `[username]` 파라미터를 비교하여 분기 처리 필요.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 프로필 헤더 | 아바타 / 이름 / 한줄소개 / 가입일 / 외부 링크 / 대표 뱃지 |
| 작성글 탭 | 작성한 게시글 목록 (PostCard) |
| 시리즈 탭 | 연재 시리즈 목록, 본인은 제목/설명 인라인 편집 가능 |
| 이벤트 탭 | 개설한 이벤트 목록 (상태 표시, 본인은 수정/마감/취소 가능) |
| 북마크 탭 | 저장한 게시글 목록 — 본인만 열람 가능 |
| 팔로워/팔로잉 탭 | 팔로우 관계 목록 (팔로우/언팔로우 가능) |
| 우측 사이드바 | 스터디 수료 이력 + 보유 뱃지 전체 목록 |

### 인터랙션
- 팔로우 버튼 → 팔로우/팔로잉 토글
- 시리즈 편집 버튼 → 제목/설명 인라인 입력 → 저장/취소
- 이벤트 마감/취소 → 상태 변경 (본인 프로필만)

### 뱃지 시스템

#### 획득 조건 (자동 지급)

| 뱃지 | 획득 조건 | 지급 시점 |
|------|----------|---------|
| AI 입문자 | 첫 번째 스터디 수료 | 수료 확정 즉시 |
| AI ○○ 전문가 | 해당 스터디 수료 (스터디별 1개) | 수료 확정 즉시 |
| 3기 수료 / 5기 수료 | 누적 수료 기수 달성 | 수료 확정 즉시 |
| 베스트 사례 1회 / 3회 / 5회 | 베스트 사례 선정 횟수 달성 | 선정 즉시 |
| 프롬프트 마스터 | 프롬프트 엔지니어링 스터디 수료 | 수료 확정 즉시 |
| 모각AI 리더 | 모각AI 세션 5회 이상 리드 | 달성 즉시 |
| 기여왕 | 월간 기여 포인트 Top 3 진입 | 월 마감 후 지급 |
| 소통왕 | 댓글 100개 이상 작성 | 달성 즉시 |
| 로켓 스타터 | 첫 게시글 작성 | 게시글 발행 즉시 |

> 어드민에서 수동 지급도 가능 (`/admin/badges`)

#### 대표 뱃지 선택 (프로필 헤더 노출)

- 보유 뱃지 중 **최대 3개**를 선택해 프로필 헤더에 노출
- 선택/변경: **설정 > 내 정보 탭** 에서 보유 뱃지 목록 중 체크박스로 선택
- 미선택 시: 최근 획득 순 3개 자동 노출
- 뱃지가 없으면 헤더에서 뱃지 영역 미표시

> **아바타 변경**: 프로필 페이지 아바타에 Camera 버튼 오버레이 (본인만) → 파일 선택으로 변경. 설정 페이지에서는 제공하지 않음.

## 내 프로필 (/profile/my)

### 목적
로그인한 사용자의 본인 프로필 페이지. `/profile/[username]`과 동일한 ProfileView를 `isOwn=true`로 렌더링한다.

### 주요 기능
| 기능 | 설명 |
|------|------|
| 로그인 가드 | 비로그인 시 "로그인이 필요합니다" + 로그인 버튼 (from 파라미터 포함) |
| ProfileView | `isOwn=true`로 렌더링 — 본인 전용 기능(북마크 탭, 시리즈 편집, 이벤트 관리, 뱃지 설정) 활성화 |

### `/profile/[username]`과의 차이

| 항목 | `/profile/[username]` | `/profile/my` |
|------|----------------------|---------------|
| isOwn 결정 | 하드코딩 또는 URL 비교 | 항상 `true` |
| 로그인 필수 | 아니오 (타인 프로필은 비로그인도 가능) | 예 (로그인 가드) |
| 사용자 데이터 | URL의 username으로 조회 | 현재 로그인 유저 정보 사용 |

### 인터랙션
- 비로그인 → "로그인" 클릭 → `/login?from=/profile/my`
- 로그인 후 → ProfileView의 모든 본인 기능 사용 가능
