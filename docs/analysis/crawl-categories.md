## 계층 구조: Collection > Space > Post

GPTers는 Bettermode의 3단계 계층 구조를 사용:

```
Collection (컬렉션) ─ 사이드바의 접히는 그룹
  └─ Space (스페이스) ─ 실제 게시글이 작성되는 게시판
       └─ Post (게시글) ─ 개별 콘텐츠
```

---

| 게시글 | `/{spaceSlug}/post/{titleSlug}-{uniqueID}` | `/news/post/claude-sonnet-46-released-ShmwtYiNuUkukb0` |

## 공통 UI 패턴 (카테고리 페이지)

### 게시글 목록
- **렌더링**: 완전 CSR (초기 HTML에 게시글 없음)
- **표시 형태**: 카드형 피드 (추정, 동적 로드)
- **정렬**: 최신순(기본), 인기순
- **페이지네이션**: 무한 스크롤 (추정)

### 게시글 메타데이터

| 필드 | 표시 여부 |
|------|-----------|
| 제목 | O |

| 작성일 (상대시간) | O |

## 핵심 발견: 카테고리 분류의 문제점

### 1. 과도한 세분화
- 17개의 주제별 꿀팁 스페이스가 하나의 컬렉션 아래 나열
- 사용자가 적합한 스페이스를 찾기 어려움
- 일부 스페이스는 활동량이 매우 적을 가능성

### 2. 중복/모호한 분류
- `news`(뉴스) vs `newsletter`(뉴스레터) → 차이 불명확
- `spotlight`(베스트 사례) vs `ai-status`(AI 현황) → 활용 사례 내 구분 모호
- `study-community` vs `study-explore` vs `ai-study-post` → 스터디 관련 3개 분리

### 3. 죽은/숨겨진 컬렉션
- "스터디", "아카이브", "주제별 스터디" 3개가 비활성
- 과거 기수별 스페이스(`ai-study-list-16th`)가 독립적으로 존재 → 정리 안됨

### 4. 깊은 탐색 경로
- 사용자 경로: 사이드바 컬렉션 클릭 → 스페이스 선택 → 게시글 선택
- 최소 3클릭 필요, 컬렉션 내 스페이스 목록이 어떻게 표시되는지 불명확
