# GPTers 정보 구조(IA) 개선 제안서

> 작성일: 2026-02-25
> 목적: Next.js + shadcn 기반 사이트 리뉴얼을 위한 정보 구조 재설계 제안
> 핵심 전환: AI 스터디 신청 + 커뮤니티 콘텐츠 소비/참여

---

## 1. 제안 요약

지피터스의 핵심 자산은 **양질의 AI 활용 콘텐츠**와 **활발한 커뮤니티**입니다. 그러나 현재 36개로 과잉 세분화된 게시판, 5개 이상 플랫폼에 걸친 서비스 파편화, CSR 기반의 SEO 부재가 이 자산의 가치를 사용자에게 제대로 전달하지 못하고 있습니다. 본 제안은 **흩어진 콘텐츠를 7개 핵심 카테고리로 재구성**하고, **분산된 서비스를 단일 Next.js 플랫폼으로 통합**하며, **"3클릭 이내 원하는 정보 도달"**이라는 원칙 아래 네비게이션을 재설계하여, AI 스터디 전환율과 커뮤니티 참여도를 동시에 끌어올리는 것을 목표로 합니다.

### Before/After 핵심 비교

| 항목 | Before (현재) | After (제안) |
|------|---------------|--------------|
| **콘텐츠 분류** | 5개 컬렉션 > 36개 스페이스 (3단계) | 7개 카테고리 + 태그 시스템 (2단계) |
| **플랫폼** | Bettermode + Framer + Notion + days.ai + Streamlit (5개+) | Next.js 단일 플랫폼 |
| **렌더링** | CSR (SEO 불가) | SSR/SSG (검색 유입 극대화) |
| **네비게이션** | GNB 4개 + 사이드바 9개 (역할 혼선) | GNB 5개 + 컨텍스트 사이드바 (명확한 역할 분리) |
| **스터디 신청 경로** | 3~4클릭, 경로 다중 분산 | 2클릭 (GNB → 스터디 상세 → 신청) |
| **온보딩** | 규칙 4개 안내만 존재 | 단계별 가이드 + 추천 콘텐츠 + 체크리스트 |
| **검색** | 검색 의존 (구조 탐색 어려움) | 구조 탐색 우선 + 검색 보완 |
| **디자인** | 플랫폼별 상이 (색상, 폰트, 테마 불일치) | 통합 디자인 시스템 (shadcn 기반) |
| **외부 서비스** | 약관(Notion), 개인정보(days.ai), 뉴스레터(Stibee), 신청(Streamlit) | 모두 내부 구현 |
| **스터디 참여자 경험** | 4개 스페이스 분산, 대시보드 없음 | 통합 스터디 허브 + 개인 대시보드 |

## 2. 설계 원칙 (Design Principles)

본 IA 재설계의 근간이 되는 5가지 원칙입니다.

### 원칙 1: 3클릭 이내 도달 (Three-Click Rule)

> 사용자가 원하는 정보에 최대 3번의 클릭으로 도달할 수 있어야 한다.

- 현재: 컬렉션 펼치기 → 스페이스 선택 → 게시글 선택 (3클릭, 그러나 17개 스페이스 탐색 부담)
- 제안: GNB 카테고리 호버/클릭 → 게시글 목록(필터 적용) → 게시글 (2~3클릭, 탐색 부담 최소)

### 원칙 2: 콘텐츠 중심 탐색 (Content-First Browsing)

> 분류 체계가 아닌 콘텐츠 자체가 탐색의 중심이어야 한다.

- 스페이스를 먼저 선택하게 하는 현재 구조 대신, 콘텐츠를 먼저 노출하고 필터/태그로 정제하는 구조
- 홈 피드에서 다양한 카테고리의 콘텐츠를 통합 노출하여 세렌디피티(뜻밖의 발견) 제공

### 원칙 3: 명확한 전환 경로 (Clear Conversion Path)

> AI 스터디 신청이라는 핵심 전환까지의 경로가 자연스럽고 명확해야 한다.

- 모든 페이지에서 스터디로의 자연스러운 유도 (강제가 아닌 맥락적 노출)
- 콘텐츠 소비 → 스터디 관심 → 정보 확인 → 신청의 자연스러운 퍼널

### 원칙 4: 단일 플랫폼 경험 (Unified Platform)

> 사용자가 하나의 사이트 안에서 모든 경험을 완결해야 한다.

- 외부 링크로 이탈하는 순간 브랜드 신뢰와 전환 추적 모두 손실
- 약관, 개인정보, 뉴스레터, 신청 폼 모두 내부 구현

### 원칙 5: 점진적 공개 (Progressive Disclosure)

> 정보는 필요한 시점에 적절한 양만 노출해야 한다.

- 첫 방문: 핵심 가치 + 시작점만 제시
- 탐색 중: 관련 카테고리/태그를 점진적으로 노출
- 깊은 참여: 상세 필터, 고급 검색, 개인 대시보드 제공

## 4. 제안 네비게이션 구조

### 4.1 GNB (Global Navigation Bar)

```
┌─────────────────────────────────────────────────────────────────────┐
│  [GPTers 로고]   커뮤니티   AI 스터디   이벤트   기업교육   시작하기  │
│                                                                     │
│                              [검색]  [알림]  [프로필/로그인]         │
└─────────────────────────────────────────────────────────────────────┘
```

| 메뉴 | 경로 | 역할 | 비고 |
|------|------|------|------|
| **로고** | `/` | 홈 | |
| **커뮤니티** | `/community` | 커뮤니티 게시판 통합 진입 | 호버 시 카테고리 드롭다운 |
| **AI 스터디** | `/study` | 스터디 프로그램 목록 | 핵심 전환 CTA, 시각적 강조 |
| **이벤트** | `/events` | 이벤트 목록 | |
| **기업교육** | `/b2b` | B2B 랜딩 | 내부 페이지로 통합 |
| **시작하기** | `/start` | 신규 사용자 가이드 | 온보딩 경로 |

**커뮤니티 호버 드롭다운:**
```
┌──────────────────────┐
│  전체 보기            │
│  ──────────────────  │
│  AI 활용 팁          │
│  뉴스/트렌드         │
│  활용 사례            │
│  질문과 답변          │
│  자유게시판           │
│  ──────────────────  │
│  📢 공지사항         │
└──────────────────────┘
```

**설계 의도:**
- GNB 메뉴는 최대 5개로 제한하여 인지 부하 최소화 (Miller's Law: 7±2)
- 모든 GNB 메뉴는 내부 페이지로 연결 (외부 링크 제거)
- "AI 스터디"는 시각적으로 강조 (버튼 스타일 또는 색상 차별화)하여 전환 유도
- "스터디장 지원"은 GNB에서 제거하고 `/about/leader`로 이동 (일반 사용자에게 불필요한 메뉴)

## 5. 콘텐츠 분류 체계 재설계

### 5.1 카테고리 통합/재구성 방안

#### 현행: 3단계 계층 (Collection > Space > Post)

```
지피터스 꿀팁 (컬렉션)
  └── AI 글쓰기 (스페이스)
       └── "Claude로 블로그 글 쓰기" (게시글)
```

- 문제: 사용자가 "컬렉션"과 "스페이스"의 개념을 이해해야 함
- 문제: 17개 스페이스 중 하나를 선택하는 인지 부하

#### 제안: 2단계 플랫 구조 (Category + Tags)

```
커뮨니티 > AI 활용 팁 (카테고리)
  └── "Claude로 블로그 글 쓰기" [#글쓰기 #Claude #LLM]  (게시글)
```

- 카테고리는 5개로 한정 (팁/뉴스/사례/Q&A/자유)
- 주제 구분은 태그로 처리 (복수 태그 가능)
- 사용자는 카테고리를 먼저 선택하고, 태그로 정밀 필터링

#### 통합 비율

| 현행 | 제안 | 통합 비율 |
|------|------|-----------|
| 5개 컬렉션 | - (계층 제거) | 100% 감소 |
| 36개 스페이스 | 5개 카테고리 + 태그 | 86% 감소 |
| 3단계 계층 | 2단계 플랫 구조 | 1단계 감소 |

### 5.2 태그 시스템 도입 제안

#### 태그 분류 체계 (Taxonomy)

**1차 태그: 주제 (Topic)**
- `#ChatGPT` `#Claude` `#Gemini` `#LLM` - AI 서비스
- `#개발` `#코딩` `#노코드` - 구현 방식
- `#마케팅` `#글쓰기` `#디자인` - 활용 분야
- `#자동화` `#챗봇` `#MCP` - 기술/도구
- `#데이터` `#분석` `#리서치` - 분석/연구

**2차 태그: 콘텐츠 유형 (Type)**
- `#튜토리얼` `#팁` `#리뷰` `#비교` `#뉴스` `#후기`

**3차 태그: 난이도 (Level)**
- `#입문` `#중급` `#심화`

**자동 태그 + 수동 태그:**
- 글쓰기 시 사용자가 태그를 자유롭게 입력 (최대 5개)
- 인기 태그를 자동완성으로 제안하여 태그 파편화 방지
- 관리자가 공식 태그를 지정하여 일관성 유지

#### 태그 활용 시나리오

```
사용자: "ChatGPT로 마케팅하는 방법을 알고 싶어"

경로 1 (구조 탐색): 커뮤니티 > AI 활용 팁 > #ChatGPT #마케팅 필터
경로 2 (태그 탐색): 인기 태그 "#마케팅" 클릭 > 관련 게시글 전체
경로 3 (검색): "ChatGPT 마케팅" 검색 > 태그 기반 연관 결과 표시
```

### 5.3 검색/필터 개선

#### 현행 검색의 강점과 한계

| 항목 | 현재 | 개선 |
|------|------|------|
| 검색 엔진 | 벡터 검색 + AI 검색 | 유지 + 태그 기반 필터링 추가 |
| 검색 범위 | 전체 | 카테고리별, 태그별 범위 지정 가능 |
| 필터 | 정렬 2개 (최신/인기) | 정렬 + 기간 + 태그 + 카테고리 + 난이도 |
| 검색 결과 | 게시글 목록 | 게시글 + 스터디 + 이벤트 통합 결과 (탭 분리) |

#### 제안하는 필터 UI

```
┌─────────────────────────────────────────────────┐
│  정렬: [최신순 ▼]   기간: [전체 ▼]              │
│                                                  │
│  태그: [#ChatGPT ×] [#마케팅 ×]  [+ 태그 추가]  │
│                                                  │
│  난이도: [전체] [입문] [중급] [심화]              │
└─────────────────────────────────────────────────┘
```

- 외부 서비스 일부(약관, 개인정보) 내부 통합

### Phase 2: 중요 (Important) - 목표: 4~6주

> 스터디 참여자 경험 강화 + 플랫폼 완전 통합

| **스터디장 지원 통합** | 별도 호스팅 → Next.js 내부 구현 | 플랫폼 통합 |

### Phase 3: 개선 (Enhancement) - 목표: 지속적

> 사용자 경험 고도화 + 리텐션 강화 + 데이터 기반 최적화

| 기능 | 상세 | 기대 효과 |
|------|------|-----------|

| **고급 검색** | 시맨틱 검색, 필터 조합 저장, 검색 알림 | 파워 유저 효율성 증대 |

| **A/B 테스팅** | 홈 레이아웃, CTA 문구, 가격 표시 등 | 데이터 기반 전환율 최적화 |

| **다국어 지원** | 영어 지원 (추후) | 글로벌 사용자 확보 |
| **API 공개** | 커뮤니티 데이터 API | 생태계 확장 |
