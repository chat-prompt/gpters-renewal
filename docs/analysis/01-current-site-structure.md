## 1. Executive Summary

GPTers(지피터스)는 지니파이 주식회사가 운영하는 국내 대표 AI 커뮤니티 플랫폼으로, Bettermode(구 Tribe Platform) 기반의 SPA/CSR 방식으로 구축되어 있다. 커뮤니티 게시판, AI 스터디 프로그램, 이벤트, 뉴스레터, B2B 기업교육 등 다양한 서비스를 제공하며, 커뮤니티를 통한 사용자 유입과 유료 스터디/교육 프로그램으로의 전환을 핵심 비즈니스 모델로 삼고 있다. 현재 서비스는 최소 5개 이상의 플랫폼에 걸쳐 분산 운영되고 있으며, 디자인 일관성 및 SEO 측면에서 구조적 개선이 필요한 상태이다.

### 핵심 수치

| 항목 | 수량 |
|------|------|
| 주요 페이지 (메인 네비게이션) | 6개 |
| 컬렉션 (활성) | 5개 |
| 컬렉션 (비활성/숨김) | 3개 |
| 스페이스 (게시판) | 36개 |
| 외부 서비스 연동 | 11개 |
| 하위 도메인 (외부 사이트) | 2개 |
| 수익 전환 경로 | 4개 |

## 3. 정보 아키텍처 (현행 IA)

### 3.1 네비게이션 구조

#### GNB (Global Navigation Bar)

```
[로고] [커뮤니티(/home)] [AI스터디(/ai-study-list)] [스터디장 지원(외부)] [기업교육(외부)]  ···  [검색] [알림] [메시지] [가입] [로그인]
```

- 좌측: 로고 + 핵심 메뉴 4개 (내부 2개 + 외부 2개)
- 우측: 유틸리티 (검색, 알림, 메시지, 인증)
- 외부 링크(`스터디장 지원`, `기업교육`)는 새 탭으로 열림

#### 상단 프로모션 배너 (전 페이지 공통)

- 모든 페이지 최상단에 고정 노출
- 현재: "단 7일, 가장 좋은 혜택! AI 스터디 21기 슈퍼 얼리버드 START"
- 링크: `/ai-study-list`

#### 좌측 사이드바

| 영역 | 항목 |
|------|------|
| 링크 메뉴 (4개) | 홈, 커뮤니티 이벤트, GPTers 사용가이드, 공지사항 |
| 컬렉션 메뉴 (5개) | 지피터스 꿀팁, 서로 도와요!, 활용 사례, AI 캠프, 테스트 |
| 프로모션 위젯 | CTO 워크샵 배너, B2B 기업교육 배너 |
| 푸터 | 지니파이 주식회사 정보, 이용약관(Notion), 개인정보처리방침(days.ai) |

#### 우측 사이드바 (스페이스 페이지)

- "이 게시글도 읽어보세요!" -- 관련 게시글 추천
- "가장 활발한 멤버" -- 활동량 기반 랭킹

### 3.2 콘텐츠 계층

```
Collection (컬렉션)        사이드바의 접히는 그룹 (8개, 활성 5개)
  └── Space (스페이스)     실제 게시글이 작성되는 게시판 (36개)
       └── Post (게시글)   개별 콘텐츠 단위
```

**URL 패턴**

| 유형 | 패턴 | 예시 |
|------|------|------|
| 컬렉션 | `/collection/{collectionID}` | `/collection/BL61SHKVDgF0` |
| 스페이스 | `/{spaceSlug}` | `/ai-writing`, `/question` |
| 게시글 | `/{spaceSlug}/post/{titleSlug}-{uniqueID}` | `/news/post/claude-sonnet-46-released-ShmwtYiNuUkukb0` |

### 3.3 콘텐츠 유형별 분류

| 유형 | 해당 스페이스/페이지 | 특성 |
|------|---------------------|------|
| 주제별 지식 공유 | 꿀팁 컬렉션 17개 스페이스 | 사용자 생성 콘텐츠, 주제별 세분화 |
| Q&A | 질문게시판 | 질의응답 |
| 소셜/커뮤니티 | 가입인사, 멤버 SNS | 소통/네트워킹 |
| 큐레이션 | 베스트 사례, 뉴스 | 운영진 선별 또는 정보성 콘텐츠 |
| 교육/스터디 | AI 스터디 목록, 강의, AX LAB 등 | 유료 프로그램 연결 |
| 운영 | 공지사항, 이벤트, 사용가이드 | 커뮤니티 운영 |

## 5. 사용자 플로우 (Key User Flows)

### 5.1 신규 방문자의 회원가입 및 콘텐츠 탐색

```
[검색/소셜 유입]
    │
    ▼
[메인 홈 (/home)]
    │  ├── 무료 웨비나 섹션 탐색
    │  ├── 새로 올라온 포스트 확인
    │  ├── 많이 읽은 게시글 확인
    │  └── 베스트 사례 확인
    │
    ▼
[관심 게시글 클릭]
    │
    ▼
[게시글 상세 페이지]
    │  (로그인 없이 열람 가능 추정)
    │
    ├── 댓글/좋아요 시도 ──▶ [로그인 요구] ──▶ [회원가입]
    │
    ▼
[가입 완료 후]
    │
    ├── 가입인사 작성 (/hello)
    ├── 사이드바 컬렉션 탐색
    └── 관심 스페이스 알림 구독
```

### 5.2 AI 스터디 발견 및 신청

```
[방문자 진입]
    │
    ├── 경로 A: 상단 프로모션 배너 클릭 ─────────────┐
    ├── 경로 B: GNB "AI스터디" 메뉴 클릭 ────────────┤
    └── 경로 C: 사이드바 AI 캠프 컬렉션 ─────────────┤
                                                      │
                                    ┌─────────────────┘
                                    ▼
                         [AI 스터디 목록 (/ai-study-list)]
                                    │
                                    ▼
                         [스터디 카드 탐색]
                            (스터디명, 커리큘럼, 가격, 기간, 강사 정보)
                                    │
                                    ▼
                         [스터디 신청 CTA 클릭]
                                    │
                                    ▼
                         [결제/신청 프로세스]
                            (상세 프로세스 미확인)
```

### 5.3 커뮤니티 콘텐츠 소비 및 참여

```
[로그인 사용자]
    │
    ▼
[홈 피드 (/home)]
    │
    ├── 새로 올라온 포스트 확인
    │       │
    │       ▼

    │       ├── 좋아요 반응
```

## 부록: 현행 구조의 주요 시사점

### A. 플랫폼 파편화

현재 서비스가 최소 5개 플랫폼(Bettermode, Framer, 별도 호스팅, Notion, days.ai)에 걸쳐 분산되어 있어 사용자 경험의 일관성이 저하되고, 통합 분석 및 전환 추적이 어려운 구조이다.

### B. 카테고리 과도한 세분화

"지피터스 꿀팁" 컬렉션 하위에 17개 스페이스가 존재하며, 일부는 활동량이 적거나 구분이 모호하다(예: `news` vs `newsletter`). 비활성 컬렉션 3개와 과거 기수 스페이스가 정리되지 않고 잔존한다.

### C. SEO 취약성

SPA/CSR 기반으로 대부분의 콘텐츠가 동적 로드되며, /events를 제외한 모든 페이지에 메타 설명이 부재하다. 검색 엔진 인덱싱에 불리한 구조이다.

### D. 디자인 불일치

메인 사이트(#eb5a10, 시스템 폰트, 라이트 테마)와 B2B 사이트(#ff4d00, Inter 폰트, 다크 테마)간 브랜드 일관성이 유지되지 않고 있다.

### E. 온보딩 부재

사용가이드(/welcome)가 커뮤니티 규칙 안내에 한정되어 있으며, 스터디 참여 방법, 게시글 작성법 등 실질적 사용 가이드와 단계별 온보딩 체크리스트가 없다.
