## 페이지별 상세 분석

### 1. 메인 홈 (/home)

**타이틀**: "AI 커뮤니티 - 지피터스"
**역할**: 대시보드형 랜딩 - 주요 콘텐츠 큐레이션

**메인 콘텐츠 섹션 (4개)**:
1. "무료 웨비나" + [전체보기] 링크
2. "새로 올라온 포스트" - 최신 게시글 피드
3. "많이 읽은 게시글" - 인기 게시글
4. "베스트 사례" + [더보기 → /spotlight]

**특징**:
- 홈 페이지에서 4개 섹션이 수직 나열
- 각 섹션은 카드 형태의 게시글 미리보기
- 콘텐츠는 전부 CSR (초기 HTML에 없음)

---

### 2. 뉴스/정보 (/news)

**타이틀**: "지피터스 - 정보와 뉴스"
**역할**: AI 관련 뉴스 및 정보 피드

**특징**:
- 완전 CSR: 정적 HTML에 게시글 콘텐츠 없음
- 게시글 목록이 동적 로드
- 게시글 메타: 제목, 작성자, 작성일(상대시간), 좋아요, 댓글 수
- 정렬 옵션: 최신순(기본), 인기순

---

### 3. 커뮤니티 이벤트 (/events)

**타이틀**: "지피터스 - 커뮤니티 이벤트"
**역할**: 오프라인/온라인 이벤트 관리

**메인 콘텐츠**:
- 2탭 구조: "다가오는 이벤트" / "종료된 이벤트"
- 메타 설명: "지피터스에서 주최하는 다양한 이벤트"

**특징**:
- 유일하게 메타 설명이 포함된 페이지
- 이벤트 카드: 제목, 날짜, 장소, 참가 신청 CTA
- 캘린더 뷰, 카테고리 필터, 날짜 검색 등 고급 기능 없음

---

### 4. 사용가이드 (/welcome)

**타이틀**: "GPTers 사용가이드"
**역할**: 신규 사용자 온보딩 + 커뮤니티 규칙 안내

**4대 이용규칙**:
1. "내가 배운 것을 기꺼이 나눠요!" - 경험 공유 장려
2. 외부 강의 홍보, 비윤리적 주제 금지
3. 나이대 다양성 존중, 상호 배려
4. AI토크(Streamlit) 개설 안내

**특징**:
- "규칙 안내" 중심 → 실제 기능 사용법 안내 부재
- 스터디 참여 방법, 게시글 작성법 등 실질적 가이드 없음
- 온보딩 단계/체크리스트 없음

---

### 5. 공지사항 (/notice)

**타이틀**: "지피터스 - 공지사항"
**역할**: 운영 공지 피드

**특징**:
- 완전 CSR: 정적 HTML에 공지 목록 없음
- 일반 게시글과 동일한 피드 형태
- 중요도/카테고리 분류 없음
- 고정 공지 기능 유무 불확실

---

### 6. 베스트 사례 (/spotlight)

**타이틀**: "지피터스 - 베스트 사례 큐레이션"
**역할**: AI 스터디 베스트 사례 큐레이션

**메인 콘텐츠**:
- 지피터스팀이 선별한 베스트 사례 모음
- 뉴스레터 구독 (Stibee 연동): "신청하기" CTA
- 배너 이미지 포함

**특징**:
- 에디토리얼/큐레이션 콘텐츠
- 자동화된 인기 게시글이 아닌 운영진 수동 선별 추정
- 뉴스레터 구독을 통한 리텐션 전략

## SEO/기술 시사점

1. **SPA/CSR 기반**: 대부분 콘텐츠가 동적 로드 → SEO 취약
2. **메타 설명 부재**: /events 외 나머지 페이지에 메타 설명 없음
3. **프로모션 3중 노출**: 상단 배너 + 사이드바 워크샵 + 사이드바 B2B
4. **외부 서비스 분산**: 핵심 기능(약관, 뉴스레터, 신청폼, 개인정보)이 4개+ 외부 서비스에 산재
