## 2. 역할별 시스템 횡단 동선

### 2.1 비회원 → 일반 회원

```
[비회원]                                    [일반 회원]
사이트만 사용                                사이트만 사용

검색/SNS → 게시글 상세 (100% 공개)           홈 피드 → 탐색 → 글쓰기
              │                               커뮤니티 → 검색 → 북마크
    댓글/투표/스크랩 시도                      AI 이력서 → 메시지
              │
      가입 게이팅 CTA
              │
        회원가입 → 온보딩 (관심분야 선택)
```

- 사이트 내에서만 활동, 어드민/LMS 접근 없음
- 사이트 PRD F1~F5, F9~F11 범위

### 2.2 수강생 (3개 시스템 횡단)

```
사이트                          LMS                        어드민
────────                       ─────                      ────────
스터디 상세 ─── 결제 ──────→ 수강생 대시보드 (/study/my)
                              │
                              ├─ 이번 주 할 일
                              │   ├─ 세션 참여 (Zoom)     ◄── 세션 등록 (운영자)
                              │   ├─ VOD 시청             ◄── VOD 등록 (스터디장)
                              │   └─ 과제 작성 ──────→ 글쓰기 (/write)     ← 사이트
                              │                          │
                              │              사례 게시글 작성 (기수+스터디 태그)
                              │                          │
                              │   ◄── 과제 자동 인식 ─────┘
                              │   ◄── VOD 잠금 해제 ──── 과제 연동 권한 규칙 (어드민)
                              │
                              ├─ 주차별 진도 추적
                              ├─ VOD 다시보기 (내 스터디 / 청강)
                              │
                              └─ (4주 후) 수료 결과 확인  ◄── 수료 판정 (어드민)
                                   │
                                   ├─ 수료증 다운로드     ◄── 수료증 발급 (어드민)
                                   └─ AI 이력서 반영 ──→ 프로필 (/profile)  ← 사이트
```

**핵심**: 수강생은 사이트(글쓰기) ↔ LMS(대시보드) ↔ 어드민(수료) 3개 시스템을 매주 횡단

### 2.3 스터디장 (가장 복잡한 역할)

```
사이트                          LMS                            어드민
────────                       ─────                          ────────
프로필 드롭다운                스터디장 대시보드                 접근 안 함
  └─ "내 스터디" ─────→  (/study/manage/{slug})               (절대로 /admin 경로 불가)
                              │
                              ├─ 빠른 설정
                              │   ├─ 상세페이지 수정 ──→ 에디터 ← 사이트
                              │   └─ 최종제출 토글 (1클릭)
                              │
                              ├─ 수강생 현황 조회
                              │   ├─ 주차별 출석/과제 테이블
                              │   ├─ 미제출자 필터
                              │   └─ 수료 예측
                              │
                              ├─ VOD 등록 (주차별 YouTube 링크)
                              ├─ 공지 작성
                              ├─ 베스트발표자 선정
                              │
                              └─ (수강생으로서)
                                  ├─ 내 스터디 대시보드 (/study/my)
                                  └─ 과제 작성 ──→ 글쓰기 ← 사이트
```

**핵심**: 스터디장은 LMS의 `/study/manage/{slug}` 에서 관리하고, **어드민(`/admin`)에는 접근하지 않음**

```
                              ├─ [기수별] 세션 등록 ──→ LMS 수강생 "할 일" 반영
                              ├─ [기수별] 과제 연동 규칙 ──→ LMS VOD 잠금/해제 규칙
                              ├─ [기수 종료] 수료 판정 ──→ LMS 수료 결과 반영
                              │                         ──→ 사이트 AI 이력서 반영
                              ├─ [기수 종료] 수료증 발급 ──→ LMS 수료증 표시
                              ├─ [기수 종료] 환급 처리
                              ├─ [필요 시] 결제/주문 조회, 환불
                              ├─ [필요 시] 회원 관리 (역할 부여)
                              ├─ [필요 시] 신고/모더레이션 ──→ 사이트 콘텐츠 비공개
                              ├─ [캠페인] 텍스트 수정 ──→ 사이트 문구 반영
```

## 4. 데이터 흐름도 — 생성 → 소비

### 4.1 게시글 (사이트 ↔ LMS ↔ 어드민)

게시글은 3개 시스템을 관통하는 **가장 핵심적인 공유 데이터**.

```
[사용자 작성]                    [시스템 자동]                   [운영자 관리]
사이트 글쓰기                    LMS 과제 인식                   어드민 게시글 관리
(/write?type=case)              (태그 기반 자동)                (/admin/posts)
      │                              │                              │
      │  기수+스터디 태그             │  주차별 과제 집계             │  카테고리 이동
      │  자동 부여 (수강생)          │  과제 연동 권한 판정          │  공개/비공개
      │                              │                              │  일괄 작업
      ▼                              ▼                              ▼
┌──────────────────────────────────────────────────────────────────────┐
│                          게시글 (posts 테이블)                        │
│                                                                      │
│  id, title, body, category, tags[], author, status, votes, created  │
│                                                                      │
│  핵심 태그 구조: ["21기", "바이브코딩", "2주차"]                       │
│  → 이 태그가 커뮤니티 글과 스터디 과제를 연결하는 브릿지                │
└──────────────────────────────────────────────────────────────────────┘
      │                              │                              │
      ▼                              ▼                              ▼
  사이트 피드 노출              LMS 진도 반영                   어드민 통계
  (/explore/feed)              수강생 대시보드                  콘텐츠 리포트
  투표/댓글/북마크             과제 O/X 표시                    게시글 수/카테고리
  AI 이력서 작성글             VOD 잠금 해제 트리거
```

### 4.2 스터디 상태 (어드민 → 사이트 + LMS)

```
어드민 기수 관리                      어드민 스터디 관리
(/admin/cohorts)                    (/admin/studies)
      │                                   │
      │ 기수 상태 변경                      │ 개별 스터디 상태 변경
      │ (준비중→모집중→진행중→완료)         │ (작성중→모집중→진행중→완료)
      │                                   │
      └──────────┬────────────────────────┘
                 │
                 ▼
┌──────────────────────────────────────────────────────────────┐
│                    스터디 (studies 테이블)                      │
│                                                              │
│  id, title, slug, generation, status, leader, period,        │
│  isFinalSubmitted, currentMembers, prices, curriculum        │
└──────────────────────────────────────────────────────────────┘
                 │
        ┌────────┼────────────────────┐
        ▼        ▼                    ▼
    사이트       사이트                LMS
    스터디 목록   스터디 상세           수강생 대시보드
    (/study)    (/study/{slug})      (/study/my)

    상태 뱃지    가격 표시             진행 중 스터디 카드
    모집 중 필터  모집 진행률 바        현재 주차 표시
                Sticky CTA 가격       스터디장 대시보드
```

### 4.3 수료 체인 (어드민 → LMS → 사이트)

## 8. 구현 순서 제안 — 시스템 간 의존성 기반

의존성을 고려한 병렬/순차 개발 순서.

```
Phase 1 (MVP 코어) — 3개 시스템 기반
─────────────────────────────────────────────────
  사이트           어드민              LMS
  ─────           ──────             ─────
  홈               대시보드            (아직 없음)
  콘텐츠 피드       게시글 관리
  게시글 상세       스터디 관리
  스터디 상세       배너 관리
                  ← 사이트와 직접 연결 →

Phase 2 (스터디 전환 퍼널)
─────────────────────────────────────────────────
  사이트           어드민              LMS
  ─────           ──────             ─────
  AI 이력서        상품/쿠폰           수강생 대시보드 ← 핵심
  검색             기수 관리           학습 상세
  커뮤니티                            VOD 다시보기
  ← 결제 플로우 정의 필요 →

Phase 3 (운영 효율화)
─────────────────────────────────────────────────
  사이트           어드민              LMS
  ─────           ──────             ─────
  글쓰기           결제/주문           스터디장 대시보드
  메시지함         수료/환급
  온보딩           세션 관리
                  과제 연동 규칙
                  ← 수료 체인 완성 →

Phase 4 (커뮤니티/분석)
─────────────────────────────────────────────────
  사이트           어드민              LMS
  ─────           ──────             ─────
                  회원 관리            수료증/수료 현황
                  텍스트 관리          리더보드
                  신고/모더레이션       찐친챌린지
                  뱃지 관리
                  리포트
                  DM/알림
```

**핵심 의존성 체인**:
1. 사이트 글쓰기 → LMS 과제 인식 → 어드민 과제 연동 규칙 (Phase 2~3에서 모두 필요)
2. 어드민 상품/가격 → 사이트 스터디 상세 → 결제 → LMS 수강 기록 (Phase 2 내 순차)
3. 어드민 수료 판정 → LMS 수료 표시 → 사이트 AI 이력서 반영 (Phase 3 완성)
