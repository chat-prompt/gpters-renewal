## 1. 개요

### 1.1 배경

GPTers(gpters.org)는 AI 활용에 관심 있는 비개발자 중심의 커뮤니티 사이트다. 현재 Bettermode 플랫폼 위에 운영 중이며, 다음과 같은 구조적 한계에 직면해 있다.

- **카테고리 30개가 혼재**하여 사용자가 콘텐츠를 찾지 못함 (P4, 인터뷰 4명 전원 언급)
- **게시글 → 가입 전환이 약함** (P1, 마케터: "게시글 확인 후 회원가입으로 이어지는 것이 더딤")
- **스터디 상세까지 가도 결제를 안 함** (P11, 마케터: "상품 상세 페이지까지는 가지만 전환이 잘 안 됨")
- **스터디를 어디서 듣는지 모름** (P10, "어디로 들어가야 하나요?" 빈번한 질문)
- **게시판이 과제 제출용으로만 기능** (P6, 가벼운 공유/자랑 불가)
- **프로필이 단순하여 활동이 이력으로 보이지 않음** (P12, "AI계의 LinkedIn" 비전 미달)

Bettermode의 제약(중첩 네비게이션, 커스터마이징 한계, SEO 부족)을 탈피하여 **Next.js 기반 자체 플랫폼**으로 전환한다.

### 1.2 목적

1. 콘텐츠 소비 → 회원가입 → 스터디 참여 → AI 이력서 누적으로 이어지는 **전환 퍼널을 구축**한다.
2. 30개 혼재 카테고리를 **6개 대분류 + 태그 시스템**으로 단순화한다.
3. 가벼운 피드 포스트 + 깊은 사례 게시글 **2트랙 콘텐츠 참여 구조**를 제공한다.
4. 스터디 수료 이력, 작성글, 뱃지가 자동 반영되는 **AI 이력서(셀프브랜딩 프로필)** 를 구현한다.
5. SEO + LLM 검색 최적화로 **외부 유입을 극대화**한다.

### 1.3 대상 사용자

| 사용자 유형 | 설명 | 핵심 니즈 |
|------------|------|----------|
| **비회원 방문자** | 검색/SNS로 게시글에 유입된 사람 | 양질의 AI 활용 콘텐츠를 빠르게 읽고 싶다 |
| **일반 회원** | 가입 후 콘텐츠를 소비하고 참여하는 사람 | 관심사에 맞는 콘텐츠를 찾고, 가볍게 글도 쓰고 싶다 |
| **스터디 참여자** | 유료 스터디를 결제하고 학습 중인 사람 | 스터디에 빠르게 접근하고, 과제를 제출하고 싶다 |
| **스터디 수료자** | 스터디를 완료한 사람 | 내 활동을 포트폴리오로 보여주고 싶다 |

### 1.4 핵심 가치 제안

**"AI 활용 역량이 자동으로 쌓이는 커뮤니티"** -- 콘텐츠를 읽고, 글을 쓰고, 스터디를 수료하면 AI 이력서에 자동 반영되어 셀프브랜딩 자산이 된다.

## 2. 핵심 기능 정의

### F1. 정보 구조 단순화 (D2)

**해결하는 문제**: P3(홈이 혼란), P4(카테고리 30개 혼재), P9(이모지 노이즈), P24(불필요 페이지 미정리)

#### 상세 요구사항

**카테고리 통폐합**
- 기존 30개 카테고리를 6개 대분류로 통합한다: AI 활용법, 프롬프트, 자동화/노코드, 개발/코딩, 비즈니스/마케팅, AI 뉴스
- 각 카테고리는 상호배타적이며, 게시글 작성 시 1개를 필수 선택한다
- 태그 시스템(도구: #ChatGPT, #Claude 등 / 난이도: #입문, #중급, #고급 / 형식: #튜토리얼, #사례 등)으로 교차 필터링을 지원한다
- 태그 입력은 자동완성을 제공하며, 다중 선택이 가능하다

**불필요 페이지 정리**
- 가입인사 게시판 등 사용률이 낮은 페이지를 제거한다
- 기수별 게시판(예: "20기 AI스터디")을 스터디 섹션 태그로 통합한다

**아이콘 정책**
- 메뉴/카테고리의 이모지를 모두 제거한다
- 시스템 아이콘은 lucide-react SVG만 사용하며, 최소한으로 배치한다

#### 수용 기준

- [ ] GNB 기본 메뉴 7개 (탐색, 스터디, 커뮤니티, AI이력서, 메시지, 검색, 로그인). 수강생에게는 "내 스터디" 1개 추가 노출 (최대 8개)
- [ ] 콘텐츠 피드에서 6개 카테고리 탭이 한 줄에 모두 표시된다
- [ ] 태그 필터를 적용하면 해당 태그가 포함된 게시글만 표시된다
- [ ] 사이트 전체에 이모지가 사용되지 않는다
- [ ] 모든 핵심 콘텐츠에 3클릭 이내로 도달 가능하다

### F2. 콘텐츠 참여 2트랙 (D3)

**해결하는 문제**: P6(게시판이 과제 제출용, 가벼운 글을 쓸 수 없음)

#### 상세 요구사항

**피드 포스트 (가벼운 글)**
- 커뮤니티 피드(/community/feed)에서 인라인 폼으로 작성한다 (페이지 이동 없음)
- 짧은 텍스트(최대 500자) + 이미지/스크린샷 첨부(선택) + 링크 첨부(OG 미리보기, 선택)
- 태그 선택 가능(선택)
- 모든 회원이 작성 가능하다 (기존 스터디원 제한 해제)
- 작성 즉시 피드에 노출된다
- 투표(upvote/downvote), 댓글, 공유 기능을 제공한다

**사례 게시글 (깊은 글)**
- 글쓰기 페이지(/write?type=case)에서 리치 텍스트 에디터로 작성한다
- 카테고리 1개 필수 선택 + 태그 다중 선택
- 모든 회원이 작성 가능하다 (기존 스터디원 제한 해제)
- 썸네일 이미지, 본문, 리치 텍스트(헤딩, 코드 블록, 이미지, 테이블 등) 지원

**글쓰기 권한 정책**
- 비회원: 읽기만 가능
- 회원: 피드 포스트 + 사례 게시글 모두 작성 가능
- 스터디 과제 게시글: 해당 스터디 수강생만 작성 가능(기수+스터디 태그 자동 부여)

#### 수용 기준

- [ ] 커뮤니티 피드에서 페이지 이동 없이 포스트를 작성/게시할 수 있다
- [ ] 피드 포스트에 이미지와 링크를 첨부할 수 있으며, 링크는 OG 미리보기가 표시된다
- [ ] 사례 게시글 작성 시 리치 텍스트 에디터가 제공된다
- [ ] 스터디에 참여하지 않은 일반 회원도 글을 작성할 수 있다

### F3. 에디터/글쓰기 경험 개선 (D4)

**해결하는 문제**: P7(임시저장 없음, 글 날아감), P8(버튼 위치 혼란, 댓글 접힘)

#### 상세 요구사항

**자동 저장**
- 사례 게시글 작성 시 30초 간격으로 자동 저장한다
- 수동 임시저장 버튼도 제공한다
- 임시저장 목록(/write/drafts)에서 이전 작성글을 이어쓸 수 있다
- 페이지 이탈 시도 시 "저장하지 않은 변경사항이 있습니다" 경고를 표시한다

**버튼 위치 통일**
- 게시/수정 버튼은 에디터 상단 고정 영역에 배치한다 (하단 스크롤 필요 없음)
- 모바일에서는 하단 Sticky 바에 게시/수정 버튼을 배치한다

**댓글 시스템**
- 댓글은 기본적으로 전체 표시한다 (기존 "1개 더 보기" 클릭 방식 제거)
- 10개 이상일 때만 "전체 댓글 보기" 토글을 제공한다
- 댓글에 투표(upvote/downvote) 기능을 제공한다
- 스레드형 답글(1단계 깊이)을 지원한다

#### 수용 기준

- [ ] 게시글 작성 중 30초마다 자동 저장이 실행된다
- [ ] 임시저장 목록에서 이전 작성글을 불러와 이어쓸 수 있다
- [ ] 브라우저 뒤로가기/탭 닫기 시 저장 경고가 표시된다
- [ ] 게시/수정 버튼이 에디터 상단에 고정 배치되어 있다
- [ ] 댓글 10개 미만일 때 모든 댓글이 펼쳐져 보인다

### F4. 검색/탐색/북마크 강화 (D5)

**해결하는 문제**: P5(검색 부정확, 정렬 기준 불명확, 북마크 실종)

#### 상세 요구사항

**통합 검색**
- 하나의 검색창에서 게시글, 스터디, 사용자를 동시에 검색한다
- 검색 결과는 타입별로 그룹핑하여 표시한다 (스터디를 최상단에, 모집 중 스터디 우선)
- 검색어 하이라이트를 적용한다
- 검색 결과를 카테고리/태그/타입으로 필터링할 수 있다

**정렬 시스템**
- 콘텐츠 피드에 3가지 정렬을 제공한다: 인기순(투표 수 기반), 최신순, 추천순(개인화)
- 각 정렬 기준이 어떤 로직인지 사용자에게 명시한다
- 인기순은 기간 필터(오늘/이번주/이번달/전체)를 추가 제공한다

**북마크**
- 게시글과 피드 포스트에 북마크(스크랩) 기능을 제공한다
- 프로필 드롭다운 또는 AI 이력서 페이지에서 북마크한 콘텐츠 목록을 확인할 수 있다

**투표 시스템**
- Reddit식 upvote/downvote를 게시글과 댓글에 적용한다
- 투표 수는 게시글 목록과 상세 페이지에 표시한다
- 인기순 정렬은 투표 수 + 시간 가중치를 조합한다

#### 수용 기준

- [ ] 검색 결과에 게시글, 스터디, 사용자가 타입별로 그룹핑되어 표시된다
- [ ] 콘텐츠 피드에서 인기순/최신순/추천순 정렬을 전환할 수 있다
- [ ] 게시글에 북마크 버튼이 있고, 북마크한 목록을 확인할 수 있다
- [ ] 게시글과 댓글에 upvote/downvote 투표가 가능하다

### F5. 전환 퍼널 최적화 (D6)

**해결하는 문제**: P1(게시글→가입 전환 약함), P11(스터디 상세→결제 전환 저조)

#### 상세 요구사항

**게시글 → 가입 전환 (요즘IT 벤치마킹)**
- 게시글 본문은 비회원에게 100% 공개한다 (페이월 없음, SEO 극대화)
- 상호작용 게이팅: 댓글/스크랩/투표를 시도하면 "회원가입하고 스크랩해 보세요" CTA를 표시한다
- 본문 중간/하단에 문맥 기반 인라인 CTA를 삽입한다: "이 글이 도움이 되셨나요? GPTers에 가입하면 매주 AI 활용 인사이트를 받을 수 있어요"
- CTA에 2가지 경로를 제공한다: [무료 가입하기] + [뉴스레터만 구독]

**독자 통계 표시 (요즘IT 벤치마킹)**
- 게시글마다 "이 글을 읽은 독자" 직군/관심분야 분포를 표시한다
- 예: "마케터 35%, 개발자 28%, 기획자 20%"
- 비회원에게 "누가 이 글을 읽고 있는지 확인하려면 가입하세요" 유도 가능

**스터디 상세 → 결제 전환**
- 스터디 상세 페이지는 전용 랜딩 레이아웃을 적용한다 (사이드바 제거)
- 구조: 히어로(핵심 정보 + 모집 진행률) → 소개(추천 대상, 얻을 수 있는 것) → 커리큘럼(아코디언) → 스터디장 프로필 → 수료생 후기 → FAQ
- Sticky CTA: 데스크톱은 우측 사이드바에 가격/잔여석/신청 버튼 고정, 모바일은 하단 Sticky 바
- 모집 진행률 표시: "12/20명 모집 (60%)" 프로그레스 바로 긴급성 유발

**사이트 전반 스터디 CTA 노출**
- 홈: "모집 중인 스터디" 섹션에 핵심 정보(기간, 인원, 가격) 포함 카드 노출
- 게시글 하단: "이 주제를 더 깊이 배우고 싶다면?" 관련 스터디 추천 배너
- 콘텐츠 피드 사이: 스터디 프로모션 카드 삽입 (3~5번째 게시글마다)

#### 수용 기준

- [ ] 비회원이 게시글 본문 전체를 읽을 수 있다
- [ ] 비회원이 댓글/스크랩/투표를 시도하면 가입 유도 UI가 표시된다
- [ ] 게시글 본문 중간 또는 하단에 인라인 CTA가 비회원에게만 표시된다
- [ ] 스터디 상세 페이지에 사이드바가 없고, Sticky CTA가 항상 노출된다
- [ ] 스터디 모집 진행률이 프로그레스 바로 표시된다
- [ ] 게시글 하단에 관련 스터디 추천이 표시된다

### F6. 스터디 접근 경로 단순화 (D7)

**해결하는 문제**: P10(스터디를 어디서 듣는지 모름, 4단계 진입)

#### 상세 요구사항

- GNB "스터디" 메뉴를 통해 1클릭으로 스터디 목록에 접근한다
- 로그인 후 프로필 드롭다운 최상단에 "내 스터디" 바로가기를 배치한다
- "내 스터디" 페이지(/study/my)에서 현재 수강 중인 스터디 목록과 LMS 연결을 제공한다
- 스터디 목록 페이지에서 모집 중인 스터디를 최상단에 노출한다
- 스터디 카드에 상태 배지를 표시한다: 모집중(초록), 진행중(파랑), 완료(회색)

#### 수용 기준

- [ ] GNB "스터디" 클릭 1번으로 스터디 목록에 도달한다
- [ ] 프로필 드롭다운 첫 번째 항목이 "내 스터디"이다
- [ ] "내 스터디" 페이지에서 수강 중인 스터디와 LMS 연결을 확인할 수 있다
- [ ] 스터디 카드에 모집중/진행중/완료 상태 배지가 표시된다

### F7. DM/메시지 시스템 (D9)

**해결하는 문제**: P18(DM이 자동 메시지에 묻혀 중요한 메시지를 놓침)

#### 상세 요구사항

- GNB에 메시지 아이콘을 배치하고, 읽지 않은 메시지 수를 표시한다
- 메시지함(/messages)은 3개 탭으로 구성한다:
  - 받은 메시지(inbox): 개인 DM만 표시 (기본 탭)
  - 보낸 메시지(sent): 내가 보낸 메시지
  - 자동 알림(notifications): 시스템 자동 메시지 (가입 환영, 스터디 안내 등)
- 개인 메시지가 자동 알림보다 항상 우선 노출된다
- 이메일 알림 링크 클릭 시 해당 메시지로 정확히 딥링크한다

#### 수용 기준

- [ ] 메시지함에서 개인 DM과 자동 알림이 별도 탭으로 분리되어 있다
- [ ] 받은 메시지 탭이 기본 선택이며, 개인 DM만 표시된다
- [ ] 이메일 알림 링크 클릭 시 해당 메시지 상세로 이동한다

### F8. AI 이력서 (D10)

**해결하는 문제**: P12(활동이 이력/포트폴리오로 보이지 않음)

#### 상세 요구사항

**프로필 헤더**
- 아바타, 이름, 한 줄 소개, 활동 시작일, 기여 포인트를 표시한다
- 관심 분야 태그를 표시한다 (셀프브랜딩 핵심 요소)
- 획득 뱃지를 나열한다
- 외부 링크(GitHub, LinkedIn, 블로그)를 연결한다
- "이 프로필을 LinkedIn에 공유" CTA로 외부 공유를 지원한다

**AI 활용 요약 (자동 생성)**
- 사용자의 게시글/활동 데이터를 분석하여 1~2문장으로 자동 요약한다
- 예: "이 사용자는 주로 Claude와 n8n을 활용하여 마케팅 자동화 분야에서 활동합니다."
- 프로필 방문자가 이 사람의 전문성을 즉시 파악할 수 있게 한다

**활동 요약 숫자**
- 스터디 수료 횟수, 작성 글 수, 받은 투표 수, 댓글 수, 기여 포인트를 한 줄로 표시한다

**탭 구성**
- 개요(기본): AI 스킬 맵(카테고리별 활동량 시각화) + 최근 활동 타임라인
- 스터디 이력: 수료한 스터디 목록, 각 스터디별 기간/스터디장/사례글/프로젝트/뱃지/수료증 링크
- 작성글: 게시한 사례 게시글 + 피드 포스트 목록
- 포트폴리오: 최종 프로젝트, 외부 작업물 링크

**AI 스킬 맵**
- 6개 카테고리별로 활동량(게시글 수, 스터디 수료 횟수)을 막대 그래프로 시각화한다
- 프로필 방문자가 이 사람의 전문 분야 분포를 한눈에 파악한다

**퍼블릭 접근**
- /profile/{username} URL로 외부에서 직접 접근 가능하다
- OG 태그를 설정하여 SNS 공유 시 프로필 요약이 미리보기로 표시된다

#### 수용 기준

- [ ] 프로필 페이지에 AI 활용 자동 요약이 표시된다
- [ ] AI 스킬 맵에 카테고리별 활동량이 시각화된다
- [ ] 스터디 이력 탭에서 수료한 스터디별 상세 정보(기간, 사례글, 수료증)를 확인할 수 있다
- [ ] /profile/{username} URL로 비로그인 사용자도 프로필을 볼 수 있다
- [ ] 프로필 페이지에 "LinkedIn에 공유" 버튼이 있다

### F9. 디자인 시스템 일관 적용 (D11)

**해결하는 문제**: P2(게시글 가독성 나쁨), P19(디자인 일관성 없음)

#### 상세 요구사항

**타이포그래피/가독성**
- 게시글 본문: max-width 720px, line-height 1.8, font-size 17px 기반 가독성 최적화
- @gpters-internal/ui 디자인 토큰의 텍스트 토큰(font-size, line-height, spacing)을 모든 페이지에 일관 적용
- 모바일 반응형 기본 적용 (breakpoint별 폰트/여백 자동 조정)

**디자인 토큰 기반**
- 모든 색상, 간격, radius는 tokens.css의 CSS 변수를 사용한다
- 하드코딩 색상(#fff, rgb 등) 금지, Tailwind 시맨틱 클래스 사용 (bg-primary, text-foreground 등)
- 브랜드 컬러 GPTers Orange(#EF6020)를 primary로 일관 적용

**컴포넌트 일관성**
- 버튼, 카드, 입력 필드, 모달 등 공통 컴포넌트는 shadcn/ui + Radix primitives 기반
- 페이지마다 다른 레이아웃/스타일 문제를 공통 컴포넌트로 해결
- Shadow는 Modal/Dropdown/Toast에만 사용하고, 나머지 컴포넌트에는 사용하지 않는다

#### 수용 기준

- [ ] 게시글 본문의 line-height가 1.8 이상이고, 최대 너비가 720px이다
- [ ] 사이트 전체에 하드코딩 색상 값이 없다
- [ ] 모바일에서 게시글을 불편 없이 읽을 수 있다
- [ ] 모든 페이지의 버튼, 카드, 입력 필드가 동일한 스타일을 따른다

### F10. 프로모션 영역 전략화 (D14)

**해결하는 문제**: P20(광고/프로모션 지면이 잘 안 보임)

#### 상세 요구사항

**홈 히어로 배너**
- 캐러셀 형태의 히어로 배너를 홈 최상단에 배치한다
- 운영자가 어드민에서 배너 이미지, 링크, 순서, 활성 상태를 관리한다
- 스터디 모집 기간에는 해당 스터디 배너를 최우선 노출한다

**콘텐츠 피드 내 프로모션**
- 게시글 목록 3~5번째마다 스터디/이벤트 프로모션 카드를 삽입한다
- 프로모션 카드는 일반 게시글 카드와 시각적으로 구분한다 ("Sponsored" 또는 "추천" 레이블)

**게시글 하단 프로모션**
- 게시글 본문 하단에 "관련 스터디 추천" 배너를 배치한다
- 게시글의 카테고리/태그와 매칭되는 모집 중 스터디를 자동 추천한다

#### 수용 기준

- [ ] 홈 최상단에 히어로 배너 캐러셀이 표시된다
- [ ] 콘텐츠 피드에 프로모션 카드가 일반 게시글 사이에 삽입된다
- [ ] 게시글 하단에 관련 스터디 추천이 표시된다
- [ ] 프로모션 카드에 "추천" 레이블이 표시되어 일반 게시글과 구분된다

### F11. 신규 회원 온보딩 (D16)

**해결하는 문제**: P25(가입 후 안내 없음, "다음에 뭘 하라"는 안내 부재)

#### 상세 요구사항

**가입 직후 온보딩**
- 회원가입 완료 후 관심분야 선택 화면을 표시한다 (6개 카테고리 중 복수 선택)
- 선택한 관심분야 기반으로 맞춤 콘텐츠를 홈 피드에 추천한다
- "다음 단계" 안내: "인기 게시글 읽기" → "첫 글 쓰기" → "스터디 둘러보기"

**스터디 결제 후 온보딩**
- 결제 완료 시 "내 스터디" 페이지로 안내하는 화면을 표시한다
- "내 스터디에서 LMS에 접근할 수 있습니다" 명시적 안내
- 스터디 시작일, 첫 세션 일정, 과제 안내 요약을 보여준다

#### 수용 기준

- [ ] 회원가입 완료 후 관심분야 선택 화면이 표시된다
- [ ] 관심분야 선택 후 홈 피드에 맞춤 콘텐츠가 추천된다
- [ ] 스터디 결제 완료 후 "내 스터디" 안내 화면이 표시된다

## 5. 화면별 스펙

### 5.1 홈 (/)

**목적**: 콘텐츠 디스커버리 시작점. 방문자가 "여기에 내가 찾는 것이 있다"고 즉시 느끼게 한다.

**주요 컴포넌트** (위에서 아래 순서):
1. **히어로 배너 캐러셀**: 스터디 모집, 이벤트 등 프로모션 (어드민에서 관리)
2. **인기 콘텐츠 카드 3~4개**: 투표 수 기반, 썸네일 + 제목 + 카테고리 + 투표수/댓글수 + 작성자/시간
3. **카테고리 탐색 카드 6개**: 각 카테고리 아이콘 + 이름 + 게시글 수, 한 줄 배치
4. **모집 중인 스터디 카드 2~3개**: 기간, 인원(현재/정원), 가격, [상세보기]/[바로신청] CTA
5. **최근 올라온 글 목록 5개**: 카테고리 | 제목 | 투표수/댓글수 | 작성자/시간
6. **비회원 전용 CTA**: "GPTers에 가입하고 AI 활용 인사이트를 받아보세요" + [구글 가입]/[이메일 가입]/[뉴스레터 구독]

**데이터 요구사항**:
- 인기 게시글: 최근 7일 내 투표 수 상위 3~4개
- 카테고리별 게시글 수
- 모집 중(status=recruiting) 스터디 목록
- 최근 게시글 5개
- 활성 배너 목록 (어드민 설정)

**인터랙션**:
- 배너 캐러셀: 자동 슬라이드(5초) + 수동 네비게이션 도트
- 카테고리 카드 클릭 → /explore/category/{slug}
- 게시글 카드 클릭 → /posts/{slug}
- 스터디 카드 [상세보기] → /study/{slug}
- 비회원 CTA는 로그인 사용자에게 표시하지 않음

### 5.2 콘텐츠 피드 (/explore/feed)

**목적**: 관심사/수준 맞춤 콘텐츠 발견. 필터와 정렬로 원하는 콘텐츠를 빠르게 찾는다.

**주요 컴포넌트**:
1. **카테고리 탭 바**: [전체] [AI활용법] [프롬프트] [자동화] [개발] [비즈니스] [뉴스] -- 한 줄, 선택 상태 하이라이트
2. **태그 필터 영역**: 활성 태그 칩(X로 제거) + [+태그 추가] (자동완성)
3. **정렬 버튼 그룹**: [인기순] [최신순] [추천순] + 난이도 드롭다운 [전체/입문/중급/고급]
4. **게시글 목록**: 투표 버튼(좌측) + 제목 + 카테고리/작성자/시간/태그 + 본문 미리보기(2줄) + 댓글수/북마크/공유
5. **무한 스크롤 또는 [더 불러오기] 버튼**

**데이터 요구사항**:
- 카테고리별 게시글 필터링
- 태그 기반 교차 필터링
- 정렬: 인기순(투표+시간 가중치), 최신순(created_at DESC), 추천순(개인화 알고리즘)
- 페이지네이션 또는 커서 기반 무한 스크롤

**인터랙션**:
- 카테고리 탭 클릭 → 해당 카테고리로 필터링 (URL 반영: /explore/category/{slug})
- 태그 칩 X 클릭 → 해당 태그 필터 해제
- 태그 추가 → 자동완성 드롭다운에서 선택
- 투표 버튼 → upvote/downvote (회원만, 비회원은 게이팅)
- 게시글 카드 클릭 → /posts/{slug}

### 5.3 게시글 상세 (/posts/{slug})

**목적**: SEO 랜딩 페이지 겸 콘텐츠 소비 페이지. 가독성 높은 레이아웃으로 읽기 경험을 극대화하고, 비회원 → 가입 전환을 유도한다.

**주요 컴포넌트**:
1. **브레드크럼 + 태그**: 카테고리 > 태그 나열 (내부 링크)
2. **제목 영역**: 게시글 제목 + 작성자 아바타/이름/시간/조회수
3. **Sticky 사이드 액션 바** (데스크톱): 투표 버튼(위/아래) + 투표 수 + 댓글 아이콘(수) + 북마크 + 공유
4. **본문 영역**: max-width 720px, 리치 텍스트 렌더링 (가독성 최적화 타이포그래피)
5. **비회원 인라인 CTA**: 본문 중간/하단에 삽입 (비회원에게만)
6. **독자 통계**: "이 글을 읽은 독자" 직군/관심분야 분포 (요즘IT 벤치마킹)
7. **관련 스터디 추천 배너**: 카테고리/태그 매칭 모집 중 스터디
8. **댓글 영역**: 스레드형 댓글 + 투표 + 전체 표시(10개 미만)
9. **관련 글 추천 카드 3개**: 같은 카테고리/태그 기반

**데이터 요구사항**:
- 게시글 상세 (제목, 본문 HTML, 작성자, 생성일, 투표수, 댓글)
- 작성자 프로필 기본 정보
- 관련 스터디 (카테고리/태그 매칭, 모집 중 우선)
- 관련 게시글 3건
- 댓글 목록 (스레드 구조)
- 독자 통계 (관심분야별 분포)

**인터랙션**:
- 투표 버튼 → upvote/downvote (비회원은 가입 게이팅)
- 북마크 → 스크랩 토글 (비회원은 가입 게이팅)
- 댓글 작성 → 로그인 필요
- 작성자 이름 클릭 → /profile/{username}
- 카테고리/태그 클릭 → /explore/category/{slug} 또는 태그 필터

**SEO 요구사항**:
- SSR/SSG로 초기 HTML에 전체 본문 포함
- JSON-LD Article 스키마
- OG 태그 (제목, 설명, 썸네일)
- 한글 slug 지원
- canonical URL 설정

### 5.4 스터디 상세 (/study/{slug})

**목적**: 스터디 소개 + 결제 전환. 전용 랜딩 레이아웃으로 구매 몰입감을 높인다.

**주요 컴포넌트**:
1. **히어로 섹션**: 스터디명 + 한 줄 소개 + 기간/정원/장소/가격 + 모집 진행률 프로그레스 바 + [스터디 신청하기] CTA
2. **탭 네비게이션**: [소개 | 커리큘럼 | 스터디장 | 후기 | FAQ]
3. **소개 섹션**: "이런 분들에게 추천합니다" 체크리스트 + "얻어갈 수 있는 것" 목록
4. **커리큘럼 섹션**: 주차별 아코디언 (1주차만 기본 펼침)
5. **스터디장 프로필 카드**: 아바타 + 이름 + 경력/소개 + [AI 이력서 보기] 링크
6. **수료생 후기 카드 3개**: 별점 + 한 줄 후기 + 기수 표시
7. **FAQ 아코디언**: 자주 묻는 질문 (코딩 경험, 일정, 수료 기준, 환불 등)
8. **Sticky CTA**: 데스크톱 -- 우측 사이드바에 가격/잔여석/신청 버튼 고정. 모바일 -- 하단 바에 가격/잔여석/신청 버튼 고정

**레이아웃 특이사항**:
- GNB는 유지하되, **사이드바를 제거**하여 콘텐츠에 집중하는 전용 레이아웃 적용
- 본문 영역을 넓게 사용하여 랜딩페이지 몰입감 확보

**데이터 요구사항**:
- 스터디 상세 (제목, 소개, 기간, 정원, 현재 신청자 수, 가격, 커리큘럼, FAQ)
- 스터디장 프로필
- 수료생 후기 목록
- 가격 정책 (슈퍼얼리버드/얼리버드/일반가 -- 마감일 기반 자동 전환)

**인터랙션**:
- 탭 클릭 → 해당 섹션으로 스무스 스크롤 (SPA 내비게이션)
- 커리큘럼 아코디언 토글
- FAQ 아코디언 토글
- [스터디 신청하기] → /checkout/{studyId} (사이트 내 체크아웃 페이지)
- 스터디장 [AI 이력서 보기] → /profile/{username}

### 5.5 AI 이력서 (/profile/{username})

**목적**: "AI계의 LinkedIn" -- 셀프브랜딩 포트폴리오. 스터디 이력, 사례글, 뱃지가 자동 누적되어 외부에 공유 가능한 전문성 페이지.

**주요 컴포넌트**:
1. **프로필 헤더**: 아바타(96x96) + 이름 + @username + 한 줄 소개 + 활동 시작일 + 기여 포인트
2. **관심 분야 태그**: 사용자가 설정한 AI 도구/분야 태그
3. **획득 뱃지 나열**: 수료 뱃지, 베스트 사례, 기여 뱃지 등
4. **외부 링크**: GitHub, LinkedIn, 블로그 등
5. **프로필 액션**: [프로필 수정](본인만) + [LinkedIn에 공유]
6. **AI 활용 요약** (자동 생성): 활동 데이터 기반 1~2문장 자동 요약
7. **활동 요약 숫자**: 스터디 수료 | 작성 글 | 받은 투표 | 댓글 | 포인트
8. **탭**: [개요 | 스터디 이력 | 작성글 | 포트폴리오]
9. **개요 탭**: AI 스킬 맵(카테고리별 활동량 막대그래프) + 최근 활동 타임라인
10. **스터디 이력 탭**: 수료 스터디별 기간/스터디장/사례글/프로젝트/수료증
11. **작성글 탭**: 사례 게시글 + 피드 포스트 목록
12. **포트폴리오 탭**: 최종 프로젝트, 외부 링크

**데이터 요구사항**:
- 사용자 프로필 (이름, 아바타, 소개, 관심분야, 외부 링크)
- 스터디 수료 이력 (스터디 정보, 기간, 사례글, 수료 뱃지)
- 작성 게시글 목록
- 활동 통계 (글 수, 투표 수, 댓글 수, 포인트)
- AI 활용 자동 요약 (게시글/활동 데이터 분석)
- 카테고리별 활동량 (스킬 맵용)

### 5.6 검색 결과 (/search?q={query})

**목적**: 통합 검색. 게시글, 스터디, 사용자를 한 번에 찾는다.

**주요 컴포넌트**:
1. **검색 입력창**: 쿼리 표시 + 검색 버튼
2. **결과 건수**: "'{query}' 검색 결과 (N건)"
3. **타입 필터 탭**: [전체(N)] [게시글(N)] [스터디(N)] [사용자(N)]
4. **카테고리 필터**: 게시글 탭 선택 시 카테고리 필터 노출
5. **스터디 결과 섹션**: 모집 중 스터디 우선 표시, 상태 배지
6. **게시글 결과 섹션**: 제목 + 카테고리/작성자/시간 + 본문 미리보기(검색어 하이라이트) + 투표수
7. **사용자 결과 섹션**: 아바타 + 이름 + 소개 + 활동 요약
8. **페이지네이션**

### 5.7 체크아웃 (/checkout/{studyId})

**목적**: 스터디 결제를 사이트 내에서 완료. 외부 도메인 이탈 없이 PG 연동.

**주요 컴포넌트**:
1. **스터디 요약 카드**: 스터디명 + 기수 + 기간 + 썸네일
2. **가격 섹션**: 현재 적용 가격 (슈퍼얼리버드/얼리버드/일반가) + 정가 대비 할인율 + 마감까지 남은 시간
3. **쿠폰 입력**: 쿠폰 코드 입력 or 보유 쿠폰 목록에서 선택 → 할인 금액 즉시 반영
4. **버디 등록** (선택): 함께 신청하는 버디 이름/전화번호 입력
5. **결제수단 선택**: 카드(법인/개인), 가상계좌, 간편결제(카카오페이 등)
6. **결제 상세**: 정가, 할인, 쿠폰할인, 최종 결제금액, 할부 안내
7. **구매자 정보**: 이름, 이메일, 전화번호 (Supabase Auth 기반 자동 채움, 수정 가능)
8. **약관 동의**: 이용약관 + 개인정보 동의 체크박스
9. **[결제하기] 버튼**: 전체 너비, Primary 색상
10. **← 스터디 페이지로 돌아가기**: 뒤로가기 링크

**레이아웃**: GNB 유지. 사이드바/Footer 없는 집중 레이아웃. 단일 컬럼 중앙 정렬 (max-width 640px).

**데이터 요구사항**:
- 스터디 정보 (제목, 기수, 기간, 가격 정책, 잔여석)
- 사용자 보유 쿠폰 목록
- 사용자 프로필 (이름, 이메일, 전화번호)
- PG 연동 (토스페이먼츠 등)

**인터랙션**:
- 쿠폰 적용 → 결제 금액 실시간 갱신
- 비로그인 상태에서 접근 시 → 로그인 페이지로 리다이렉트 (결제 후 원래 체크아웃으로 복귀)
- 결제 완료 → 결제 완료 온보딩 화면 표시
- 결제 실패 → 에러 메시지 + 재시도

### 5.8 계정 설정 (/settings)

**목적**: 내 정보 관리, 구매내역 조회, 쿠폰 관리를 사이트 내에서 처리. 외부 도메인 이탈 금지.

**주요 컴포넌트**:
1. **좌측 사이드 네비게이션**: 내 정보 | 구매내역 | 쿠폰 | 알림 설정
2. **내 정보 탭** (/settings/profile): 아바타 변경, 이름, 이메일, 전화번호, 비밀번호 변경, 관심분야 수정
3. **구매내역 탭** (/settings/purchases): 결제 이력 목록 (스터디명, 결제일, 금액, 상태), 아코디언으로 상세 펼침 (결제수단, 영수증 조회), 환불 신청 링크
4. **쿠폰 탭** (/settings/coupons): 보유 쿠폰 목록 (코드, 할인 내용, 유효기간, 사용 여부), 쿠폰 코드 등록
5. **알림 설정 탭** (/settings/notifications): 이메일/푸시 알림 On/Off 토글

**레이아웃**: GNB + Footer 유지. 좌측 사이드 네비게이션 + 우측 콘텐츠 영역.

### 5.9 커뮤니티 피드 (/community/feed)

**목적**: 가벼운 포스트 + 자랑글. "과제 제출용" 사례글과 달리, 부담 없이 AI 경험을 공유하는 공간.

**주요 컴포넌트**:
1. **탭 바**: [피드(기본)] [자유게시판] [Q&A] [이벤트]
2. **인라인 포스트 작성 폼**: 아바타 + "무슨 AI 이야기를 나누고 싶으신가요?" + 텍스트 입력 + [이미지][링크][#태그] + [포스트 게시]
3. **포스트 목록**: 작성자/시간 + 본문 텍스트 + 첨부(이미지/링크 미리보기) + 태그 + 투표/댓글/공유

### 6.2 페이지별 구현 우선순위

프로토타입은 정적 목업(백엔드 미연결)으로 시작하며, 핵심 전환 퍼널 순서로 구현한다.

| 우선순위 | 페이지 | 이유 |
|----------|--------|------|
| **P1** | 홈 (/) | 전환 퍼널의 시작점, 모든 동선의 허브 |
| **P1** | 게시글 상세 (/posts/{slug}) | SEO 랜딩 페이지, 비회원→가입 전환의 핵심 |
| **P1** | 콘텐츠 피드 (/explore/feed) | 카테고리 단순화 + 태그 필터의 핵심 검증 |
| **P2** | 스터디 상세 (/study/{slug}) | 스터디→결제 전환의 핵심 |
| **P2** | AI 이력서 (/profile/{username}) | "AI계의 LinkedIn" 비전 핵심 |
| **P2** | 검색 (/search) | 콘텐츠 발견성의 핵심 |
| **P3** | 커뮤니티 피드 (/community/feed) | 2트랙 콘텐츠 참여의 가벼운 트랙 |
| **P3** | 스터디 목록 (/study) | 스터디 탐색 |
| **P3** | 글쓰기 (/write) | 에디터 경험 검증 |
| **P4** | 메시지 (/messages) | DM 시스템 기본 |
| **P4** | 회원가입 + 온보딩 | 관심분야 선택 플로우 |
| **P4** | 내 스터디 (/study/my) | 스터디 접근 경로 |
