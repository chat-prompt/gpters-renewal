# PRD 03: LMS 내재화 + 수강생 대시보드 + 스터디장 권한

> 목적: 외부 LMS 의존을 제거하고, 사이트 안에서 학습-관리-수료까지 완결되는 경험을 설계
> 작성일: 2026-02-25
> 관련 문제: P10 (스터디 진입 4단계), P21 (사이트 = "중간 다리"), P23 (스터디장 경험)
> 관련 개선방향: D7 (스터디 접근 단순화), D15 (LMS 내재화 + 수강생 대시보드 + 스터디장 권한)

## 1. 개요

### 1.1 목적

GPTers AI 스터디의 학습 경험 전체를 사이트 안에 내재화한다.

현재 사이트는 결제와 과제 제출의 "진입점"에 불과하고, 실제 학습(VOD 다시보기, 세션 참여, 진도 확인)은 외부 LMS(`ai-study.gpters.org`)에서 이루어진다. 이로 인해 사이트는 신연권이 표현한 대로 **"중간 다리"** 역할에 그치고 있다.

이 PRD는 사이트를 "중간 다리"에서 **"학습 플랫폼"**으로 전환하기 위한 기능을 정의한다.

> **마이그레이션 성격**: 완전한 신규 설계가 아니라, 기존 LMS(`ai-study.gpters.org`)의 검증된 기능을 새 사이트에 통합하는 작업이다. 기존 LMS에서 이미 VOD 다시보기, 과제 추적, 리더보드, 찐친챌린지, 베스트발표자, 세션 관리, 공지 시스템이 운영 중이므로, 이 기능들의 UX를 개선하여 새 사이트에 자연스럽게 녹여내는 것이 핵심이다.
>
> **프로토타입 범위**: 현재 단계는 정적 목업(Static Mockup)이다. 백엔드 연동 없이, 화면과 사용 흐름을 검증한다.
>
> **기술 구현 참고**: 백엔드/데이터 마이그레이션 세부사항은 `docs/context/lms-migration-reference.md` 참조.

## 2. 현재 문제 상세 분석

### 2.1 스터디 진입 4단계 (P10)

**현재 경로**: 프로필 클릭 → 내 AI스터디 → 로그인 → LMS 페이지 확인

이 4단계 경로는 다음과 같은 문제를 야기한다:

1. **발견 불가**: "내 AI스터디"가 프로필 드롭다운 안에 숨어있어, 신규 수강생이 존재 자체를 모름
2. **외부 이동**: LMS가 별도 도메인(`ai-study.gpters.org`)이라 사이트 이탈 발생
3. **문의 폭주**: 스터디 시작 1주차에 "어디서 들어가요?"가 가장 빈번한 질문
4. **재방문 경로 부재**: 매주 반복 접근해야 하는데, 매번 같은 4단계를 거침

**목표 상태**: 로그인 후 **GNB "내 스터디" 버튼 1클릭**으로 수강생 대시보드 도달 (프로필 드롭다운 의존 제거)

### 2.2 사이트 = "중간 다리" (P21)

현재 사이트에서 처리하는 스터디 관련 작업:
- 스터디 상세페이지 열람 / 결제
- 사례 게시글(과제) 작성
- 수료증 확인

현재 **사이트 밖에서 처리되는** 작업 (LMS/외부 도구):
- VOD 다시보기 열람
- 출석 확인
- 진도 추적 (주차별 어디까지 했는지)
- 세션 참여 (Zoom 링크 접근)
- 수강생 간 소통 (카카오톡)

핵심 학습 활동이 모두 사이트 밖이므로, 수강생은 사이트를 "결제할 때" 와 "과제 쓸 때"만 방문한다. 이는 커뮤니티 형성을 저해하고, 사이트의 핵심 가치를 약화시킨다.

**목표 상태**: VOD 다시보기, 진도 추적, 세션 안내, 수료증이 모두 사이트 안에서 완결

### 2.3 스터디장 경험 (P23)

스터디장은 **사용자**이지 운영자가 아니다. 그러나 현재 다음과 같은 운영자급 네비게이션을 강요받는다:

1. **상세페이지 찾기**: "작성 중 AI스터디" 목록에서 자기 스터디를 찾아야 함. 신규 스터디장은 사이트 구조가 생소해서 위치를 못 찾음
2. **최종제출 토글**: 게시글 → 수정하기 → 여러 내용 스크롤 → "최종제출" 탭 찾기 → ON. 6단계
3. **수강생 현황 파악**: 에어테이블을 직접 확인하거나 운영자에게 문의해야 함

**목표 상태**: 스터디장 전용 대시보드에서 자기 스터디 설정, 수강생 현황, 공지를 한 곳에서 관리

## 3. 핵심 기능 정의

### 3.1 수강생 대시보드 (`/study/my`)

수강생이 로그인 후 자신의 스터디 학습 상태를 한눈에 파악하고, 필요한 행동을 즉시 수행할 수 있는 허브 페이지.

#### 3.1.1 내 스터디 목록

| 항목 | 설명 |
|------|------|
| 진행 중 스터디 | 현재 진행 중인 스터디 카드 (최상단, 강조 표시) |
| 수료 완료 | 수료 확정된 스터디 (수료증 링크 포함) |
| 미수료 | 참여했으나 수료 조건 미달 |
| 수강 이력 전체 | 기수별 참여 이력 (AI 이력서와 연동) |

각 스터디 카드에 표시되는 정보:
- 스터디명, 기수, 기간
- 스터디장 이름
- 현재 주차 (예: "2주차 진행 중")
- 나의 진도율 (프로그레스 바)
- 수료 조건 충족 여부 아이콘

#### 3.1.2 이번 주 할 일 (Action Items)

진행 중 스터디가 있을 때, 대시보드 상단에 이번 주 해야 할 일을 카드로 노출:

| 할 일 유형 | 조건 | 표시 내용 |
|-----------|------|----------|
| **과제 작성** | 이번 주 사례 게시글 미작성 시 | "이번 주 사례글을 작성하세요" + [글쓰기] 버튼. **최우선 노출** |
| **세션 참여** | 이번 주 AI토크/줌 세션 예정 시 | "화요일 7PM AI토크" + [참여하기] 버튼 |
| **세션 다시보기** | 지난 세션의 VOD 업로드 완료 시 | "2주차 AI토크 다시보기 등록됨" + [시청하기] 버튼 |
| **VOD 시청** | 이번 주 VOD 업로드 후 미시청 시 | "2주차 줌 세션 다시보기" + [시청하기] 버튼 |
| **찐친챌린지 인증** | 2~4주차 챌린지 기간 중 | "이번 주 네트워킹 인증하기" + 내 인증 현황 (2/7일) + [인증 방법 보기] |
| **청강 가능** | 과제 연동 권한 획득 시 | "다른 스터디 다시보기 열람 가능" + [둘러보기] 버튼 |

**과제 연동 권한 안내 UX**:

과제 연동 권한은 수강생이 직관적으로 이해하기 어려운 규칙이므로, 단계적으로 안내한다:

1. **권한 획득 조건**: 이번 주 사례 게시글 1개 이상 작성 → 전체 15개 스터디 VOD 열람 가능
2. **리셋 시점**: **매 주차 시작일(월요일 00:00)에 리셋**. 2주차에 글을 썼더라도, 3주차 시작되면 다시 글을 써야 열림
3. **미작성 시 안내**: VOD 영역에 잠금 아이콘 + "이번 주 사례글을 작성하면 모든 스터디 다시보기를 볼 수 있어요" + [글쓰기] 바로가기
4. **작성 완료 시 피드백**: 글쓰기 완료 → 대시보드 복귀 시 "청강 가능" 배지 즉시 표시 + 잠금 해제 시각 변화
5. **잠금 상태 설명**: 잠긴 VOD 카드에 "왜 잠겨있나요?" 링크 → 과제 연동 권한 설명 모달

#### 3.1.3 주차별 진도 추적

4주 스터디의 각 주차별 학습 현황을 시각적으로 표시:

```
1주차          2주차          3주차          4주차
[완료]         [진행중]       [예정]         [예정]

출석: O        출석: -        출석: -        출석: -
과제: O (1/1)  과제: X (0/1)  과제: -        과제: -
VOD:  O        VOD:  신규     VOD:  -        VOD:  -
```

각 주차 카드 클릭 시 해당 주차의 상세 내용 확장:
- 커리큘럼 내용 (스터디장이 작성한 주차별 학습 내용)
- 과제 제출 상태 및 작성한 게시글 링크
- VOD 목록 (AI토크, 줌 발표 세션)
- 출석 상태

#### 3.1.4 VOD 다시보기

현재 외부 LMS에서만 가능한 VOD 열람을 사이트에 내재화:

| 항목 | 설명 |
|------|------|
| VOD 목록 | 주차별 AI토크, 줌 발표 세션 녹화본 목록 |
| 재생 방식 | 임베드 플레이어 (YouTube/Vimeo 등 외부 호스팅 임베드) 또는 자체 호스팅 |
| 접근 권한 | 내 스터디 VOD: 항상 접근 가능. 다른 스터디 VOD: 과제 연동 권한 필요 |
| 청강 | 과제 연동 권한 획득 시, "다른 스터디" 탭에서 전체 15개 스터디 VOD 열람 |
| 시청 기록 | 시청 완료 여부 자동 체크 (출석 데이터와 별도) |

VOD 페이지 구조:
- **내 스터디** 탭: 내가 신청한 스터디의 VOD (항상 접근 가능)
- **다른 스터디** 탭: 전체 15개 스터디의 VOD (과제 연동 권한 필요)
  - 권한 없을 시: 리스트는 보이되 재생 잠금 + 잠금 사유 표시:
    - "이번 주 사례글을 작성하면 열립니다" + [글쓰기 →] 바로가기 버튼
    - "왜 잠겨있나요?" 클릭 시 과제 연동 권한 규칙 설명 모달
  - 권한 획득 시: 잠금 해제 + "청강 가능" 배지 + 시각적 피드백 (잠금 아이콘 → 재생 아이콘)

#### 3.1.5 수료증 조회/다운로드

| 항목 | 설명 |
|------|------|
| 수료 조건 표시 | 출석 N회 이상 + 과제 N개 이상 기준 표시 (기수별 다를 수 있음) |
| 수료 판정 결과 | 수료/미수료 상태 + 근거 데이터 (출석 현황, 과제 제출 현황) |
| 수료증 다운로드 | PDF 또는 이미지 형태. AI 이력서에도 자동 반영 |
| 우수활동자 | 사례 2개 이상 + Zoom 2회 이상 참여 시 우수활동자 마크 |

### 3.2 스터디장 대시보드 (`/study/manage/{study-slug}`)

스터디장이 자기 스터디를 "한 곳에서" 관리할 수 있는 중간관리자 인터페이스.

**설계 원칙**: 스터디장은 사용자이지 운영자가 아니다. 어드민 패널이 아니라, 수강생 대시보드의 확장된 뷰로 설계한다.

#### 3.2.1 접근 경로

- GNB 프로필 드롭다운 → "내 스터디" → 스터디장 권한이 있는 스터디에 "관리" 탭 노출
- 또는 수강생 대시보드에서 스터디장 역할인 스터디 카드에 "스터디 관리" 버튼 노출
- **절대로** 어드민(`/admin`) 경로를 통하지 않음

#### 3.2.2 스터디 설정 관리

| 기능 | 현재 | 개선 |
|------|------|------|
| 상세페이지 수정 | "작성 중 AI스터디"에서 찾기 → 어렵다 | 관리 탭에서 바로 "상세페이지 수정" 링크 |
| 최종제출 토글 | 게시글 → 수정 → 스크롤 → 토글 (6단계) | 관리 탭 상단에 토글 버튼 1개 (1클릭) |
| 커리큘럼 수정 | 상세페이지 안에서 수정 | 주차별 커리큘럼 편집 UI (관리 탭) |
| 스터디 기본정보 | 상세페이지에서 수정 | 기간, 요일/시간, 난이도 등 기본정보 편집 |

#### 3.2.3 수강생 현황 조회

박민수     출석O 과제O  출석O 과제O  출석- 과제-  -           순조

#### 3.2.4 공지 작성

| 기능 | 설명 |

| 스터디 공지 | 자기 스터디 수강생에게만 보이는 공지 작성 |

### 3.3 LMS 핵심 기능

사이트에 내재화해야 하는 LMS 기능과 구현 범위.

#### 3.3.1 VOD 재생

| 항목 | 설명 |
|------|------|
| 호스팅 | **YouTube 비공개 링크 임베드** (현재 LMS에서 검증 완료) |
| 플레이어 | 반응형 iframe (16:9 비율). 모바일 최적화 |
| 접근 제어 | 로그인 + 수강 여부 + 과제 연동 권한 체크 |
| 필터 | 기수별 + 스터디별 + 주차별 필터 |
| 시청 기록 | 재생 시작 시 "시청함" 자동 기록. 수료 판정에는 미사용 (출석과 별도) |

#### 3.3.2 출석 체크

> 현재 LMS에서도 출석 자동 추적은 미구현. 과제 제출이 핵심 지표이고, 출석은 보조 지표.

| 항목 | 설명 |
|------|------|
| 프로토타입 | 출석 칸은 UI상 표시하되 "-" 처리 (데이터 없음). 과제 제출 현황 중심 |
| 추후 | 수동 CSV 업로드 → Zoom Webhook 자동화로 단계적 발전 |
| 수동 출석 | 운영자/스터디장이 직접 출석 체크 가능 (예외 처리용) |
| 출석 기준 | 줌 세션 참여 = 1회 출석. 기수별 최소 출석 횟수 설정 가능 |
| 표시 | 수강생 대시보드에서 주차별 출석 상태 확인 가능 |

#### 3.3.3 과제 제출 현황

기존 사례 게시글 시스템과 연동 (2~4주차만 과제 대상, 1주차는 없음):

| 항목 | 설명 |
|------|------|
| 과제 = 사례 게시글 | 기수+스터디별 태그가 붙은 게시글이 과제로 자동 인식 |
| 자동 트래킹 | 게시글 작성 시 태그 기반으로 주차별/수강생별 자동 집계 |
| 대시보드 표시 | 수강생 본인: "이번 주 과제 제출 완료/미제출". 스터디장: 전체 현황 표 |
| 과제 연동 권한 | 이번 주 게시글 1개 이상 → 전체 VOD 열람 권한 자동 부여. **매 주차 시작일(월요일 00:00)에 리셋** — 이전 주차 과제와 무관하게, 새 주차에서 다시 글을 써야 열림 |

#### 3.3.4 수료 판정 로직

```
수료 조건 (기수별 설정 가능):
  - 출석: 최소 N회 이상 (예: 4회 중 2회 이상)
  - 과제: 최소 N개 이상 (예: 4주 중 2주 이상 제출)

우수활동자 조건:
  - 사례 게시글 2개 이상 작성
  - Zoom 세션 2회 이상 참여

판정 시점: 스터디 종료일(D+28) 이후 운영자가 일괄 판정 실행
  - 자동 집계 데이터 기반으로 수료/미수료 자동 분류
  - 운영자가 예외 케이스 수동 조정 가능
```

#### 3.3.5 수료증 자동 발급

| 항목 | 설명 |
|------|------|
| 트리거 | 운영자가 "수료증 일괄 발급" 실행 (수료 판정 완료 후) |
| 생성 | 수료자 정보 + 스터디 정보로 수료증 자동 생성 |
| 형식 | PDF 다운로드 + 웹 뷰 (공유 가능 URL) |
| 연동 | AI 이력서(프로필)에 수료 이력 자동 반영 |
| 노출 | 수강생 대시보드 → 해당 스터디 카드에 "수료증 보기" 버튼 |

#### 3.3.6 리더보드 (활동왕)

| 항목 | 설명 |
|------|------|
| 집계 기준 | 기수별 사례 게시글 수 (삭제 게시글 제외) |
| 표시 | 수강생 대시보드에 활동왕 Top 10 노출 (홈에는 미노출) |
| 내 순위 강조 | 로그인 사용자가 리더보드에 포함되면 해당 행 하이라이트 + "나" 배지 |
| 동점 처리 | 10위와 같은 점수인 사용자 모두 포함 |
| 프로필 연동 | 사용자 프로필 페이지 링크 |

프로토타입: 정적 목업 데이터로 리더보드 UI 구현.

#### 3.3.7 찐친챌린지

| 항목 | 설명 |
|------|------|
| 챌린지 기간 | 2주차 월요일 00:00 ~ 4주차 목요일 23:59 (자동 계산) |
| 인증 방식 | 카카오톡 네트워킹방 인증 → n8n 자동 수집 → DB 저장 |
| 통계 | 전체 인증 수, 인정 일수(날짜별 1회만), Sender별 통계 |
| UI | 파이차트 (전체 비율) + Top 5 리더보드 + 내 인증 카드 |

**대시보드 통합** (기존 LMS에서는 별도 페이지로 분리되어 학습 흐름과 단절):
- "이번 주 할 일"에 찐친챌린지 인증 카드 추가 (2~4주차 챌린지 기간 중에만 노출)
- 카드에 표시: 내 인증 현황 (예: "3/7일 인증 완료"), [인증 방법 보기] 버튼
- [인증 방법 보기] 클릭 시 안내 모달: "카카오톡 네트워킹방에서 다른 수강생과 대화 인증샷을 올려주세요" + 카톡방 링크
- 대시보드 하단에 찐친챌린지 위젯: 전체 참여 통계 + Top 5 + 내 순위

프로토타입: "이번 주 할 일" 카드 + 찐친챌린지 위젯 목업.

#### 3.3.8 베스트발표자

| 항목 | 설명 |
|------|------|
| 선정 주체 | 스터디장 (자기 스터디만) |

| 수강생 뷰 | 선정된 수강생에게 대시보드 알림 + 프로필에 "베스트발표자" 배지 표시 |

프로토타입: 스터디장 대시보드에 베스트발표자 선정 UI 목업.

#### 3.3.9 세션 관리

| 항목 | 설명 |
|------|------|
| 세션 유형 | 공통 줌 세션 (AI토크, 네트워킹, OT 등) |
| 캘린더 뷰 | 기간 내 모든 공통 세션을 캘린더에 표시 |
| 오늘의 세션 | 오늘 예정된 세션 하이라이트 (Zoom 링크 바로 이동) |
| 데이터 | 날짜시간, 제목(prefix 활용), Zoom URL |

**세션 → VOD 연결** (기존 LMS에서는 세션과 VOD가 별도 데이터로 흐름이 끊김):
- 세션 종료 후 VOD 업로드 시, 해당 세션 카드에 "다시보기" 링크 자동 추가
- "이번 주 할 일"에서: 세션 참여 카드 → (세션 종료 후) → "세션 다시보기 등록됨" 카드로 자동 전환

- VOD 카드에도 원본 세션 정보(제목, 날짜) 표시하여 "이게 그때 그 세션이구나" 인지 가능

프로토타입: 세션 카드 3가지 상태(예정/오늘/다시보기 가능) 목업 + VOD 연결 표시.

프로토타입: 수강생 대시보드 상단 공지 배너 + 공지 목록 페이지 목업.

## 4. 유저 플로우

### 4.1 수강생 플로우: 결제 → 학습 → 수료

```
[스터디 상세페이지]
       │
       ▼
  [ 결제 완료 ]
       │
       ├── 온보딩 안내: "내 스터디에서 학습을 시작하세요"
       │   └── GNB "내 스터디" 버튼 최초 하이라이트 (첫 결제 시 GNB에 새로 등장)
       │
       ▼
  [수강생 대시보드] (/study/my)
       │
       ├── 이번 주 할 일 확인
       │   ├── "1주차 OT 참여하기" → Zoom 링크
       │   ├── "사례 게시글 작성하기" → 글쓰기 페이지 (태그 자동 설정)
       │   └── "AI토크 다시보기" → VOD 플레이어
       │
       ├── 주차별 진도 확인
       │   └── 출석/과제/VOD 상태 한눈에
       │
       ├── VOD 열람
       │   ├── 내 스터디 VOD: 항상 접근 가능
       │   └── 다른 스터디 VOD: 과제 작성 후 잠금 해제
       │
       ├── (4주간 반복)
       │
       ▼
  [스터디 종료]
       │
       ├── 수료 판정 결과 확인
       │   └── "수료 완료" 또는 "미수료 (출석 부족)" 등
       │
       ├── 수료증 다운로드
       │
       └── AI 이력서에 자동 반영
           └── 스터디 이력, 작성 게시글, 수료 배지
```

#### 4.1.1 과제 작성 마이크로 플로우

과제(사례 게시글) 작성은 4주간 매주 반복되는 핵심 행동이다. 어디서 쓰고, 쓴 후 어디로 가고, 완료 후 무엇이 바뀌는지를 명확히 설계:

```
[수강생 대시보드]
       │
       ├── "이번 주 할 일" > [글쓰기] 버튼 클릭
       │   └── 태그 자동 설정 (기수 + 스터디 + 주차)
       │
       ▼
[게시글 작성 페이지] (/write?type=case&studyId={studyId}&week={week})
       │
       ├── 상단 안내: "이 글은 21기 AI자동화 스터디 2주차 과제로 등록됩니다"
       │   └── 태그가 미리 설정되어 있어 수정 불필요
       │
       ├── 글 작성 완료 → [제출] 클릭
       │
       ▼
[작성 완료 피드백]
       │
       ├── 토스트/배너: "과제가 제출되었습니다!"
       │
       ├── 과제 연동 권한 즉시 갱신:
       │   └── "전체 스터디 다시보기가 열렸습니다" 안내
       │
       ├── 다음 행동 유도:
       │   ├── [내 스터디로 돌아가기] → /study/my (기본)
       │   ├── [다른 스터디 다시보기] → /study/my/audit
       │   └── [작성한 글 보기] → /posts/{slug}
       │
       ▼
[수강생 대시보드] (복귀 시)
       │
       ├── "이번 주 할 일"에서 과제 카드 → "제출 완료" 상태로 변경
       ├── "청강 가능" 배지 새로 표시 (과제 연동 권한 획득)
       └── 진도율 자동 갱신
```

## 7. 프로토타입 구현 가이드

### 7.1 구현 우선순위

본 PRD의 기능을 프로토타입(정적 목업)으로 구현할 때의 우선순위:

| 우선순위 | 화면 | 경로 | 설명 |
|---------|------|------|------|
| **P0** | 수강생 대시보드 | `/study/my` | 핵심 허브. P10 해결의 시작점 |
| **P0** | 스터디 학습 상세 | `/study/{slug}/learn` | 주차별 진도, VOD, 과제 통합 뷰 |
| **P1** | VOD 다시보기 | `/study/{slug}/learn/vod` | P21 해결의 핵심. 과제 연동 권한 반영 |
| **P1** | 스터디장 대시보드 | `/study/manage/{slug}` | P23 해결. 최종제출 토글, 수강생 현황 |
| **P2** | 수료 현황/수료증 | 대시보드 내 섹션 | 수료 판정 결과 + 수료증 뷰 |
| **P2** | 결제 후 온보딩 | 결제 완료 페이지 | P25 연계 |

### 7.2 데이터 모델 (프로토타입용 목업 데이터)

프로토타입은 정적 목업이므로 실제 DB 연동은 없으나, 다음 데이터 구조를 기반으로 목업 데이터를 설계한다.

> Airtable 필드 매핑 등 구현 세부사항: `docs/context/lms-migration-reference.md`

```typescript
// 스터디
interface Study {
  id: string;
  title: string;          // "21기 AI 자동화 스터디"
  slug: string;           // "21-ai-automation"
  generation: number;     // 21
  status: 'draft' | 'recruiting' | 'ongoing' | 'completed';
  category: string;       // 비즈니스&마케팅, 개발&자동화, 콘텐츠&지식, AI개발
  dayOfWeek: string;      // 화요일, 수요일, 목요일
  studyTime: string;      // "20:00-22:00"
  period: { start: Date; end: Date };
  leader: User;
  curriculum: WeekContent[];
  currentMembers: number;
  isFinalSubmitted: boolean;
  thumbnailUrl?: string;
  zoomUrl?: string;
  kakaoUrl?: string;      // 카카오톡 오픈채팅방 링크
  isCancelled: boolean;
}

// 주차별 콘텐츠
interface WeekContent {
  week: number;           // 1~4
  title: string;          // "AI 자동화 입문"
  description: string;
  videos: Video[];
  period: { start: Date; end: Date };
}

// VOD
interface Video {
  id: string;
  title: string;
  type: 'ai-talk' | 'zoom-session';
  week: number;
  studyTag: string;       // 빈 값이면 공통 일정
  embedUrl: string;       // YouTube embed URL
  duration: number;       // 분
  uploadedAt: Date;
}

// 수강 기록
interface Enrollment {
  id: string;
  user: User;
  study: Study;
  role: 'student' | 'leader';
  attendance: WeeklyAttendance[];
  assignments: WeeklyAssignment[];
  completionStatus: 'in_progress' | 'completed' | 'failed';
  certificate?: Certificate;
}

// 주차별 출석
interface WeeklyAttendance {
  week: number;
  attended: boolean;
  sessionDate?: Date;
}

// 주차별 과제 (2~4주차만, 1주차는 과제 없음)
interface WeeklyAssignment {
  week: number;
  submitted: boolean;
  postSlug?: string;      // 작성한 게시글 slug
  submittedAt?: Date;
  deadline: Date;         // 주차시작일 + 스터디요일 기반 자동 계산
  isLate: boolean;
}

// 과제 연동 권한
// 규칙: 1주차 공개 / 내 스터디 전체 접근 / 타 스터디는 주차별 과제 제출 시 잠금 해제
interface AssignmentPermission {
  user: User;
  generation: number;
  currentWeek: number;
  hasSubmittedThisWeek: boolean;  // true면 전체 VOD 청강 가능
}

// 수료증
interface Certificate {
  id: string;
  enrollment: Enrollment;
  issuedAt: Date;
  isOutstanding: boolean;  // 우수활동자 여부
  downloadUrl: string;
}

// 리더보드
interface LeaderboardEntry {
  rank: number;
  nickname: string;
  postCount: number;       // 기수별 사례 게시글 수
  profileUrl?: string;
}

// 찐친챌린지
interface ChallengeStats {
  cohort: number;
  period: { start: Date; end: Date };  // 2주차 월~4주차 목
  totalCertifications: number;
  senderStats: { sender: string; count: number; uniqueDays: number; percentage: number }[];
}

// 공지사항
interface Notice {
  id: string;
  title: string;
  content: string;         // 마크다운 지원
  cohort: string;          // "21기"
  category: 'member' | 'leader-guide' | 'zoom-session';
  prefix?: string;         // "필독", "공통세션" 등
  order: number;
  sessionDateTime?: Date;  // 줌세션인 경우
  zoomUrl?: string;
  isHidden: boolean;
}
```

## 8. 현재 운영 프로세스와의 정합성

### 8.1 기존 외부 도구 → 사이트 내재화 매핑

| 현재 (외부 도구) | 내재화 후 (사이트) | 잔여 외부 의존 |
|-----------------|-------------------|---------------|

| 출석 데이터 (미구현) | 프로토타입: UI만. 추후 Zoom 연동 | Zoom 데이터 |
| 수료 판정 (수동) | 어드민 수료 판정 자동화 | 없음 |
| 수료증 (수동 발급) | 자동 PDF 생성 + 대시보드 노출 | 없음 |
| 리더보드 | 대시보드/홈에 활동왕 Top 10 | 없음 |
| 찐친챌린지 | 대시보드 위젯으로 통합 | 카카오톡 + n8n |
| 베스트발표자 | 스터디장 대시보드에 통합 | 없음 |

| 카카오톡 소통 | 카톡방 링크 유지 + 사이트 알림 추가 | 카톡방 유지 |

### 8.2 운영 플레이북 변경점

**준비기 (D-42 ~ D-0)**:
- 스터디장 상세페이지 작성: 기존과 동일하되, 스터디장 대시보드에서 바로 접근
- 최종제출 토글: 스터디장 대시보드에서 1클릭 (기존 6단계 → 1단계)

**모집기 (D-14 ~ D-4)**:
- 변경 없음 (결제/모집은 기존 흐름 유지)

**진행기 (4주)**:
- 수강생: 매주 사이트 대시보드에서 할 일 확인 → 과제 작성 → VOD 시청 (기존: LMS 별도 방문)
- 스터디장: 수강생 현황을 대시보드에서 직접 확인 (기존: 에어테이블 또는 운영자에게 문의)
- 운영자: 어드민에서 전체 스터디 모니터링 (기존과 유사하되 데이터가 사이트에 통합)

**종료 후 (D+28~)**:
- 수료 판정: 어드민에서 자동 집계 기반 일괄 처리 (기존: 에어테이블 수동 필터링)
- 수료증 발급: 자동 생성 + 수강생 대시보드 노출 (기존: 수동 발급 + 개별 전달)
- 환급: 기존 프로세스 유지 (에어테이블 데이터 기반 → 추후 어드민 내재화)
