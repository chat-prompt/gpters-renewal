# 04. 디자인 가이드

> 사이트/어드민/LMS PRD 공통 참조 가이드. 프로토타이핑 시 이 문서를 기준으로 UI를 작성한다.

---

## 1. 개요

### 1.1 목적

이 문서는 GPTers 리뉴얼 프로젝트의 **프로토타이핑 공통 참조 가이드**다. 사이트(프론트엔드), 어드민, LMS 3개 PRD에서 공통으로 사용하며, 새 페이지나 컴포넌트를 만들 때 이 문서의 규칙을 따른다.

### 1.2 디자인 방향: Reddit-inspired

전반적인 UI/UX 톤은 **Reddit (New Reddit, 2024~)**을 레퍼런스로 한다. 콘텐츠 중심 커뮤니티라는 공통점이 있으며, 벤치마킹(docs/report/02-벤치마킹-분석.md §1)에서 도출한 패턴을 적용한다.

**가져오는 핵심 패턴:**

| Reddit 패턴 | GPTers 적용 |
|------------|-----------|
| 투표 기반 큐레이션 (upvote/downvote) | 게시글/댓글 좌측 세로 투표 바. 인기순 정렬의 기본 축 |
| Compact 카드 레이아웃 | 피드에서 썸네일+제목+메타+투표가 한 줄에 밀집. 스캔성 우선 |
| 계층형 댓글 트리 | 들여쓰기 + 접기/펼치기. 대댓글 깊이 제한 없음 |
| Flair = 태그 필터 | 카테고리 대신 컬러 태그. 클릭 시 해당 태그 필터 |
| 가입 게이팅 | 비회원도 콘텐츠 100% 열람. 투표/댓글/북마크 시도 시 로그인 프롬프트 |
| 피드 상단 간편 작성 | 인라인 작성 폼 (클릭 → 확장). 별도 페이지 이동 최소화 |

**가져오지 않는 것:**
- 서브레딧 시스템 (GPTers는 6개 카테고리 + 태그로 충분)
- 다크 모드 (Phase 1 범위 밖)
- Award/코인 시스템 (GPTers는 포인트/뱃지로 대체)

**shadcn 기반 + 커스텀 컴포넌트:**

shadcn/ui에 있는 것은 그대로 사용하고, 없는 Reddit 스타일 컴포넌트는 직접 만든다:

| 컴포넌트 | shadcn 활용 | 직접 구현 |
|---------|-----------|---------|
| Vote (세로 투표 바) | Button 기반 | ArrowUp/ArrowDown + count + 색상 상태 |
| Post Card (compact) | Card 기반 | 투표+썸네일+제목+메타 조합 |
| Comment Thread | — | 재귀 컴포넌트 + 들여쓰기 + collapse |
| Inline Post Form | Input + Button 기반 | 클릭→확장 애니메이션 + 이미지 첨부 |
| Flair Badge | Badge 기반 | 컬러 배경 + 클릭 시 필터 |
| Sticky Action Bar | — | 게시글 상세 좌측 고정 (투표/댓글/북마크/공유) |
| Sort Tabs | Tabs 기반 | 인기/최신/추천 정렬 탭 (아이콘 포함) |

### 1.3 기술 스택

| 항목 | 기술 |
|------|------|
| Framework | Next.js 16 (App Router) |
| Styling | Tailwind CSS v4 (CSS-first, `@theme` 블록) |
| UI 라이브러리 | `@gpters-internal/ui` + shadcn/ui (Radix primitives) |
| 아이콘 | `lucide-react` (SVG only) |
| 인프라 | Supabase (auth, DB, storage) |
| 언어 | TypeScript |

### 1.3 스타일 파일 구조

```
app/globals.css
├── @import '@gpters-internal/ui/tokens.css'          # 디자인 토큰 (색상, 간격, radius 등)
├── @import '@gpters-internal/ui/palettes/gpters.css'  # GPTers Orange 팔레트 오버라이드
├── @import "tailwindcss"                               # Tailwind v4
└── @theme { ... }                                      # CSS 변수 -> Tailwind 테마 매핑
```

**핵심 원칙: Tailwind First** - CSS 변수보다 Tailwind 유틸리티 클래스를 우선 사용한다.

## 2. 디자인 토큰 요약

### 2.1 색상 체계

GPTers의 브랜드 컬러는 **GPTers Orange (#EF6020)**이다. 모든 색상은 Tailwind semantic 클래스를 통해 사용한다.

#### 주요 색상 토큰

| Tailwind 클래스 | 용도 | Light 값 |
|-----------------|------|----------|
| `bg-background` / `text-foreground` | 기본 배경 / 기본 텍스트 | `#fafafa` / `#171717` |
| `bg-primary` / `text-primary-foreground` | 주요 액션 (CTA, 강조) | `#EF6020` / `#ffffff` |
| `bg-secondary` / `text-secondary-foreground` | 보조 배경 | `#f5f5f4` / `#1c1917` |
| `bg-muted` / `text-muted-foreground` | 비활성/보조 텍스트 | `#737373` / `#525252` |
| `bg-accent` / `text-accent-foreground` | 강조 배경 | `#EF6020` / `#ffffff` |
| `bg-destructive` / `text-destructive-foreground` | 위험/삭제 | `#ef4444` / `#ffffff` |
| `border-border` | 기본 테두리 | `#e5e5e5` |
| `border-input` | 입력 필드 테두리 | `#e5e5e5` |
| `ring-ring` | 포커스 링 | `#EF6020` |

#### 색상 사용 규칙

```tsx
// 기본 텍스트
<p className="text-foreground">기본 텍스트</p>
<p className="text-muted-foreground">보조 텍스트</p>

// 배경
<div className="bg-background">기본 배경</div>
<div className="bg-muted">보조 배경 (코드블록, 빈 상태 등)</div>

// 주요 액션
<button className="bg-primary text-primary-foreground">CTA 버튼</button>

// 카테고리/링크 강조
<span className="text-primary">카테고리명</span>

// 위험 액션
<button className="text-destructive">삭제</button>
```

#### 상태 표시 색상 (Badge용)

| 상태 | 클래스 조합 |
|------|------------|
| 활성/모집중 | `bg-primary/10 text-primary` |
| 완료 | `bg-muted text-muted-foreground` |
| 수료 | `text-primary bg-accent px-2 py-0.5 rounded-sm` |

**금지 사항:**
- 하드코딩 색상 (`#333`, `rgb(...)`, `white` 등)
- Tailwind arbitrary value (`bg-[#f5f5f5]`)
- `globals.css` 외부의 새 색상 도입
- 화면당 배경/강조 색상 3개 초과

### 2.2 간격/여백 (Spacing)

Tailwind 4px 기반 스케일을 사용한다.

| 클래스 | 값 | 주요 용도 |
|--------|-----|----------|
| `p-1`, `gap-1` | 4px | 아이콘-텍스트 미세 간격 |
| `p-2`, `gap-2` | 8px | 인라인 요소 간격, 태그 내부 |
| `gap-3` | 12px | 리스트 아이템 간격 |
| `p-4`, `gap-4` | 16px | 컴포넌트 내부 패딩, 카드 리스트 간격 |
| `p-6`, `gap-6` | 24px | 카드 내부 패딩, 네비게이션 간격 |
| `gap-8` | 32px | 사이드바-메인 간격, 섹션 간 탭 아래 |
| `py-8` | 32px | 페이지 상단/하단 여백 |
| `space-y-12`, `mb-12` | 48px | 홈 섹션 간격, 스터디 섹션 간격 |

**페이지 컨테이너 패턴:**
```tsx
<div className="mx-auto max-w-6xl px-4 py-8">
  {/* 페이지 콘텐츠 */}
</div>
```

### 2.3 Border Radius

| Tailwind 클래스 | 값 | 용도 |
|-----------------|-----|------|
| `rounded-sm` | 2px | 태그, 뱃지, 인라인 상태 표시 |
| `rounded-md` | 4px | 버튼, 입력 필드, 정렬 칩 |
| `rounded-lg` | 6px | 카드, 리스트 컨테이너, 모달 |
| `rounded-xl` | 8px | 큰 카드, 다이얼로그 |
| `rounded-2xl` | 12px | 대형 컨테이너 |
| `rounded-full` | 9999px | 아바타, 뱃지 pill, 프로그레스바 |

### 2.4 타이포그래피

| Tailwind 클래스 | 용도 |
|-----------------|------|
| `text-2xl font-bold` | 페이지 메인 제목 (h1) |
| `text-xl font-bold` | 페이지 제목 (admin) |
| `text-lg font-bold` | 섹션 제목 (h2) |
| `font-bold text-foreground` | 서브섹션 제목 (h3) |
| `text-sm font-medium` | 게시글 제목, 리스트 아이템 |
| `text-sm text-muted-foreground` | 본문, 설명 |
| `text-xs text-muted-foreground` | 메타 정보 (시간, 작성자, 카운트) |
| `text-xs text-primary font-medium` | 카테고리 라벨 |

**폰트 패밀리:**
- `font-sans`: Pretendard Variable (한국어 최적화)
- `font-mono`: Fira Code (코드 블록)

## 4. 레이아웃 패턴

### 4.1 기본 레이아웃 (Navbar + Content + Footer)

모든 페이지의 기본 구조. `app/layout.tsx`에서 정의.

```
+------------------------------------------+
|              Navbar (sticky)              |
+------------------------------------------+
|                                          |
|              Main Content                |
|           (max-w-6xl, px-4)              |
|                                          |
+------------------------------------------+
|               Footer                     |
+------------------------------------------+
```

```tsx
<html lang="ko">
  <body className="bg-background text-foreground min-h-screen flex flex-col">
    <Navbar />
    <main className="flex-1">{children}</main>
    <Footer />
  </body>
</html>
```

**Navbar** (`sticky top-0 z-50`):
- 높이: `h-14`
- 하단 보더: `border-b border-border bg-background`
- 컨테이너: `mx-auto max-w-6xl flex items-center justify-between px-4`
- 네비게이션 항목: `text-sm`, 활성 시 `text-foreground font-medium`, 비활성 시 `text-muted-foreground`
- 모바일 숨김: `hidden md:flex`

**Footer**:
- 상단 보더: `border-t border-border bg-muted py-6 mt-12`
- 컨테이너: `mx-auto max-w-6xl px-4 flex flex-col md:flex-row items-center justify-between gap-4`

### 4.2 사이드바 레이아웃 (Admin)

```
+------------------------------------------+
|              Navbar (sticky)              |
+------------------------------------------+
|  Sidebar  |      Main Content            |
|  (w-48)   |      (flex-1)                |
|           |                              |
+------------------------------------------+
|               Footer                     |
+------------------------------------------+
```

```tsx
<div className="mx-auto max-w-6xl px-4 py-8">
  <div className="flex gap-8">
    {/* Sidebar */}
    <aside className="w-48 shrink-0 hidden md:block">
      <h2 className="text-sm font-bold text-foreground mb-4">관리자</h2>
      <nav className="space-y-1">
        {sidebarItems.map((item) => (
          <Link
            key={item.label}
            href={item.href}
            className={`flex items-center gap-2 px-3 py-2 text-sm rounded-md ${
              item.active
                ? "bg-accent text-foreground font-medium"
                : "text-muted-foreground"
            }`}
          >
            <item.icon className="w-4 h-4" />
            {item.label}
          </Link>
        ))}
      </nav>
    </aside>

    {/* Main Content */}
    <div className="flex-1 min-w-0 space-y-6">
      {/* 페이지 콘텐츠 */}
    </div>
  </div>
</div>
```

**사이드바 항목:**
- 활성: `bg-accent text-foreground font-medium`
- 비활성: `text-muted-foreground`
- 아이콘: `w-4 h-4`

### 4.3 Sticky CTA 레이아웃 (스터디 상세, 모바일)

```tsx
<div className="fixed bottom-0 left-0 right-0 border-t border-border bg-background p-4 md:hidden">
  <div className="flex items-center justify-between max-w-6xl mx-auto">
    <div>
      <p className="text-foreground font-bold">150,000원</p>
      <p className="text-xs text-muted-foreground">잔여 8석</p>
    </div>
    <button className="px-6 py-2 bg-primary text-primary-foreground rounded-md font-medium text-sm">
      스터디 신청하기
    </button>
  </div>
</div>
```

### 4.4 Sticky Sidebar (게시글 상세)

```tsx
<aside className="hidden lg:block w-16 shrink-0">
  <div className="sticky top-20 flex flex-col items-center gap-4">
    {/* 투표, 댓글, 북마크, 공유 */}
  </div>
</aside>
```

### 4.5 반응형 전략

| Breakpoint | Tailwind prefix | 값 | 적용 |
|------------|-----------------|-----|------|
| 기본 (모바일) | 없음 | < 640px | 단일 컬럼 |
| `md` | `md:` | >= 768px | 태블릿, 사이드바 표시, 그리드 확장 |
| `lg` | `lg:` | >= 1024px | 데스크톱, sticky sidebar 표시 |

**주요 반응형 패턴:**

```tsx
// 그리드: 모바일 1열 -> 데스크톱 3열
<div className="grid grid-cols-1 md:grid-cols-3 gap-4">

// 사이드바: 모바일 숨김
<aside className="w-48 shrink-0 hidden md:block">

// Sticky sidebar: 모바일/태블릿 숨김
<aside className="hidden lg:block w-16 shrink-0">

// Footer: 모바일 세로 -> 데스크톱 가로
<div className="flex flex-col md:flex-row items-center justify-between gap-4">

// 네비게이션: 모바일 숨김
<nav className="hidden md:flex items-center gap-6">
```

### 4.6 레이아웃 위계 규칙

```
Header (전체 너비, 최상위)
├── Sidebar (좌측, Header 아래)
└── Content (우측, Header 아래)
```

Sidebar가 Header 위에 오는 구조는 금지.

## 5. 아이콘 가이드

### 5.1 사용 규칙

- **lucide-react만 사용** (이모지, 텍스트 문자, 유니코드 기호 절대 금지)
- import 방식: `import { IconName } from "lucide-react";`

### 5.2 크기 규칙

| 크기 | 클래스 | 용도 |
|------|--------|------|
| 12px | `w-3 h-3` | 인라인 메타 정보 (투표, 댓글 카운트) |
| 14px | `w-3.5 h-3.5` | 댓글 내 액션, 소형 버튼 아이콘 |
| 16px | `w-4 h-4` | 사이드바 메뉴, 리스트 아이콘, 탭 쉐브론 |
| 20px | `w-5 h-5` | 네비게이션 아이콘, 투표 버튼, 피드 액션 |

### 5.3 주요 사용 아이콘 목록

| 카테고리 | 아이콘 | import | 용도 |
|----------|--------|--------|------|
| 네비게이션 | 검색 | `Search` | GNB, 검색바 |
| | 메일 | `Mail` | 메시지/알림 |
| | 사용자 | `User` | 프로필/로그인 |
| | 화살표 | `ChevronRight`, `ChevronDown` | 더보기, 아코디언 |
| | 외부링크 | `ExternalLink` | 외부 사이트 이동 |
| 게시글 | 투표 | `ArrowUp`, `ArrowDown` | 투표 |
| | 댓글 | `MessageSquare` | 댓글 수 |
| | 북마크 | `Bookmark` | 저장 |
| | 공유 | `Share2` | 공유 |
| | 글쓰기 | `PenSquare` | 글쓰기 CTA |
| | 닫기 | `X` | 태그 제거, 모달 닫기 |

## 6. Anti-pattern 목록

### 6.1 절대 금지

| 규칙 | 금지 | 올바른 사용 |
|------|------|------------|
| 하드코딩 색상 | `#333`, `rgb(...)`, `white`, `bg-[#f5f5f5]` | `text-foreground`, `bg-background` |
| Shadow | `shadow-md`, `box-shadow: ...` | `border border-border` (Dropdown/Modal/Toast 제외) |
| 이모지/텍스트 아이콘 | `✅`, `❌`, `+`, `×`, `▼` | lucide-react SVG 컴포넌트 |
| 화면당 컴포넌트 7개 초과 | 10개 이상 독립 섹션 | 그룹화, 탭, 페이지 분리 |
| 배경/강조 색상 3개 초과 | 빨+파+초+보+노 | primary + secondary + accent |
| 임의 간격 | `padding: 30px`, `gap: 10px` | Tailwind 4px 스케일 (`p-4`, `gap-4`) |
| 임의 radius | `border-radius: 5px`, `rounded-[10px]` | `rounded-md`, `rounded-lg` 등 토큰 |
| 인터랙션 효과 중첩 | borderLeft + background + fontWeight 동시 변경 | 배경색 변경만 (택 1) |
| 컨테이너 중첩 | wrapper > container > inner > content | 플랫 구조, spacing으로 구분 |
| 불필요한 wrapper div | 스타일 목적의 div 중첩 | 시맨틱 태그 직접 사용 |
| 배경 레이어 중첩 | html + body + div 모두 배경색 | body에서 단일 지정 |
| CSS 변수 직접 사용 | `style={{ color: 'var(--foreground)' }}` | `className="text-foreground"` |

### 6.2 허용되는 예외

| 항목 | 허용 조건 |
|------|----------|
| Shadow | Dropdown, Modal, Toast에 한해 허용 |
| 고정 width | 아이콘 (`w-4`), 아바타 (`w-24`), 사이드바 (`w-48`) |
| style 속성 | 동적 값 (`style={{ width: \`${progress}%\` }}`) |
| `opacity` | 히어로 배너 서브 텍스트 (`opacity-80`), 캐러셀 도트 (`opacity-40`) |

## 7. 프로토타이핑 가이드라인

### 7.1 새 페이지 작성 체크리스트

1. **파일 생성**: `app/{route}/page.tsx` (App Router 라우팅)
2. **컨테이너**: `<div className="mx-auto max-w-6xl px-4 py-8">`
3. **제목**: `<h1 className="text-xl font-bold text-foreground">`
4. **섹션 간격**: `space-y-8` 또는 `space-y-12`
5. **카드/리스트**: `border border-border rounded-lg bg-background`
6. **반응형**: 모바일 우선, `md:` / `lg:` breakpoint 추가

### 7.2 "use client" 사용 기준

| 필요한 경우 | 불필요한 경우 |
|-------------|-------------|
| `useState`, `useEffect` 사용 | 정적 콘텐츠 렌더링 |
| `usePathname` 등 Next.js 클라이언트 훅 | 서버에서 데이터 fetch |
| 이벤트 핸들러 (`onClick`, `onChange`) | Link 컴포넌트만 사용 |
| 탭, 필터, 토글 등 인터랙션 | API 호출 (Server Component) |

### 7.3 파일/컴포넌트 구조 규칙

#### 기본 원칙

```
app/
├── {route}/
│   └── page.tsx              # 페이지 컴포넌트
components/
├── navbar.tsx                 # 전역 GNB
├── footer.tsx                 # 전역 Footer
└── {feature}/                 # 기능별 컴포넌트 (2회 이상 사용 시 분리)
```

- 2회 이상 반복되는 UI는 `components/`로 추출
- 한 페이지에서만 쓰이는 컴포넌트는 해당 `page.tsx`에 인라인 또는 같은 폴더에 배치

#### 7.3.4 추출 시점 판단 기준

프로토타이핑은 속도가 중요하다. 컴포넌트를 언제 파일로 분리할지 판단 기준:

| 상황 | 권장 |
|------|------|
| 한 페이지에서만 사용, 단순한 UI | `page.tsx` 안에 인라인으로 유지 |
| 같은 패턴이 2번째 페이지에서 필요 | `components/ui/`로 추출 |
| 특정 영역의 복합 컴포넌트 (예: 어드민 게시글 테이블 행) | `components/admin/`에 배치 |
| `page.tsx`가 300줄을 초과 | 같은 폴더에 `_components/` 만들어 분리 |
| Props가 3개 이상이고 다른 곳에서도 재사용 | 별도 파일 추출 + Props 인터페이스 정의 |

```
app/explore/feed/
├── page.tsx                    # 피드 페이지 (메인)
└── _components/                # 이 페이지에서만 쓰이는 로컬 컴포넌트
    ├── post-list-item.tsx      # 피드 게시글 아이템
    └── tag-filter.tsx          # 태그 필터 UI
```

> `_components/` 접두사(`_`)는 Next.js App Router에서 라우팅에서 제외되므로 안전하다.

### 7.4 생성 후 자가 검증

```
검증 체크리스트:
- [ ] Tailwind 클래스 사용 (style 속성 최소화)
- [ ] 하드코딩 색상 없음
- [ ] 하드코딩 간격 없음
- [ ] radius는 토큰 사용 (rounded-*)
- [ ] 화면당 컴포넌트 수 <= 7
- [ ] 배경/강조 색상 수 <= 3
- [ ] 이모지/텍스트 아이콘 없음 (SVG만)
- [ ] shadow 없음 (dropdown/modal/toast 제외)
- [ ] Button/Tag/Menu 등 whitespace-nowrap
- [ ] Header > Sidebar 위계
- [ ] 반응형 대응 (고정 width 없음)
- [ ] 불필요한 wrapper div 없음
- [ ] 배경색 레이어 중첩 없음
```
