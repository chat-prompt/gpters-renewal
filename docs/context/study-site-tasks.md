# 스터디 운영에서 사이트가 처리하는 작업 목록

> 3개 문서(about-ai-study.md, study-prep-checklist.md, study-runtime-checklist.md)에서
> **사이트(웹)에서 일어나는 작업만** 추출 정리.
> 에어테이블, 카톡, Slack, Zoom 등 외부 도구 작업은 제외.

---

## 1. 스터디 콘텐츠 관리

### 스터디 상세페이지 (랜딩페이지)
- 스터디장이 상세페이지 작성 → 제출 → 수정 → **최종제출**
- 운영진이 최종제출 상태를 토글로 관리
- 상세페이지에 표시되는 정보: 주제, 요일/시간, 난이도, 커리큘럼, 스터디장 소개
- 스터디 목록 페이지에 노출 여부 제어

### 배너 관리
- CMS에서 사전판매 배너 등록
- 홈 띠배너(히어로 배너) 설정/순서 변경
- 모집 상태에 따른 배너 교체 (사전판매 → 모집중 → 마감)

### 스터디 상태 관리
- 스터디 상태 변경: 작성중 → 모집중 → 진행중 → 완료
- 모집 시작/마감 제어

---

## 2. 수강신청 & 결제

- 수강신청 오픈/마감 처리
- 결제 시스템 연동 (가격 표시, 결제 처리)
- 가격 정책 반영: 슈퍼얼리버드 → 얼리버드 → 일반가 (마감일 기반 자동 전환)
- 할인쿠폰 처리 (스터디장 지인초대 쿠폰, 임직원 쿠폰)
- 버디 등록 폼 (`ai.gpters.org/register_buddy`)
- 신청자 수 실시간 표시 (12/20명)

---

## 3. 사례 게시글 (과제) 시스템

### 게시글 작성
- 수강생이 사이트에 사례 게시글 작성
- 기수+스터디별 태그 부여 (예: `21기 바이브코딩`)
- 주차별 게시글로 분류

### 과제 트래킹
- 기수별/주차별 게시글 작성 현황 자동 수집
- 미제출자 파악 (사이트 데이터 기반)
- 기수당 목표: 게시글 700~800개

### 과제 연동 권한 (핵심 자동화)
- **"이번 주 사례 게시글 1개 이상 작성해야 → 전체 15개 스터디 다시보기 열람 가능"**
- 게시글 작성 여부에 따른 다시보기 페이지 접근 권한 자동 부여/차단
- 주차별 리셋

---

## 4. 다시보기 (VOD)

- AI토크/줌 발표 세션 녹화본 업로드
- 다시보기 페이지 접근 권한 제어 (과제 연동)
- 청강 지원: 자기 스터디 외 다른 스터디 세션도 열람 가능

---

## 5. 수료 & 환급

### 수료 기준 판정
- 출석 데이터 + 과제(게시글) 제출 데이터 집계
- 우수활동자 조건: 사례 2개 이상 + Zoom 2회 이상 참여
- 수료 대상자 자동 필터링

### 수료증
- 수료증 자동 발급
- 사이트에서 수료증 확인/다운로드

### 환급
- 환급 요건 충족 여부 판정 (사이트 데이터 기반)
- 버디 환급: 사례글 2회 작성 시 100% 환급(100원 제외)

---

## 6. 프로필 (AI 이력서)

- 스터디 수료 이력 표시 (기수, 스터디명, 기간)
- 작성한 사례 게시글 모아보기
- 스터디장 프로필 → 스터디 상세페이지에서 링크
- 최종 프로젝트/포트폴리오 연결

---

## 7. 관리자 기능 (운영진용)

### 게시글 관리
- 카테고리 이동 (게시글을 다른 카테고리로 이동)
- 게시글 공개/비공개 전환
- 일괄 작업 (선택 → 이동/삭제)

### 스터디 관리
- 스터디 상태 드롭다운 변경 (작성중/모집중/진행중/완료)
- **최종제출 토글**: 목록에서 원클릭 ON/OFF (기존에는 게시글 → 수정 → 스크롤 → 토글로 6단계)
- 랜딩페이지로 바로 이동 링크

### 배너 관리
- 홈 배너 추가/수정/삭제
- 드래그앤드롭 순서 변경
- 활성/비활성 토글

### 수강생 관리
- 과제 제출 현황 대시보드
- 출석 현황
- 환급 대상자 추출
